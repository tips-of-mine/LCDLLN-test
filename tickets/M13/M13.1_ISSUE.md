# M13.1 — Server core: tick loop + connections + snapshots skeleton

**Status:** Open

## Objectif
Serveur headless avec tick fixe (20/30Hz), gestion connexions, envoi snapshots vides.

## Dépendances

- M00.5

## Livrables
- Server app
- Network transport (UDP/ENet/quic?)
- Tick scheduler

## Structure & chemins (verrouillé)

- **Contenu** : uniquement sous `/game/data/...`

- Chemins relatifs à `paths.content`.
- ❌ Interdit : /assets, chemins absolus.

## Spécification technique
- Serveur authoritative; client envoie inputs; snapshots à 10-20Hz.

## Étapes d'implémentation
1. Choisir transport (UDP + reliability layer ou ENet).
2. Implémenter accept/handshake, clientId.
3. Loop tick: process inputs -> sim -> build snapshot.
4. Client: receive snapshot + stats.
