# M18.4 — Bindless textures/material indices (descriptor indexing)

## Objectif
Mettre en place bindless textures pour réduire bindings materials.

## Dépendances
- M3.3

## Livrables
- Descriptor indexing enable
- Global texture array
- Material uses indices


## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- VK_EXT_descriptor_indexing + robust access.

## Étapes d’implémentation
1. Activer features descriptor indexing.
2. Créer descriptor set global textures[].
3. Modifier shaders materials pour indexer textures.
4. Migrer matériaux vers indices.

## Critères de validation (DoD)
- [ ] Rendu identique, mais bindings par draw fortement réduits.

## Notes / pièges à éviter
- Gérer lifetime textures (streaming) avec slots libres/RC.
