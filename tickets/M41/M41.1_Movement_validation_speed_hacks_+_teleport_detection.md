# M41.1_Movement_validation_speed_hacks_+_teleport_detection

## Objectif
Détecter et bloquer speed hacks et teleports via validation serveur.

## Dépendances
- M30.1
- M26.1

## Livrables
- Server-side movement validation
- Speed limit checks (max speed)
- Teleport detection (delta position threshold)
- Kick/flag cheaters

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Max speed: run=10 m/s, walk=5 m/s (+ margin 20%)
- Teleport threshold: 50m/frame (impossible jump)
- Validation: check delta position vs max speed * dt
- Action: kick after 3 violations, log to DB

## Étapes d'implémentation
1. Server: track last position, last update time
2. On InputCommand: calculate delta position
3. Max allowed delta: maxSpeed * dt * 1.2 (margin)
4. If delta > allowed: increment violation counter
5. If violations >= 3: kick player, log cheat event
6. Reset counter after 60s no violations
7. Exemptions: teleport spells (flag isLegitTeleport)

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD

## Notes / pièges à éviter
- False positives: lag spikes (generous margin)
- Exempt: mounts, blink spells (whitelist)
