# M00.2 — Memory: allocateur, arenas, tags, tracking

**Status:** Open

## Ticket (original)

### Objectif
Avoir une gestion mémoire contrôlée : tags par système, LinearArena, frame arenas, stats (current/peak/count).

### Dépendances
- M00.1

### Livrables
- `/engine/core/memory/MemoryTags.h` (Core/Render/Assets/World/Net/UI/Tools/Temp)
- `SystemAllocator` (aligned alloc/free) + tracking
- `LinearArena` (bump alloc + reset)
- `FrameArena[framesInFlight]` intégré au loop
- `Memory::DumpStats()`

### Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

### Spécification technique
- API alloc générique `alloc(size, align, tag)` / `free(ptr, tag)`.
- Tracking par tag: current bytes, peak bytes, alloc/free count.
- LinearArena: pas de free individuel, reset O(1).
- FrameArena reset au début de la frame correspondante.

### Étapes d'implémentation
1. Définir MemTag enum + stats struct.
2. Implémenter SystemAllocator + tracking (atomic si multi-thread).
3. Implémenter LinearArena (reserve + pointer) + assert overflow.
4. Créer FrameArena[2] ou [3], wiring dans `BeginFrame()`.
5. Implémenter DumpStats tableau par tag.

### Critères de validation (DoD)
- [ ] 10k allocs arena → reset → 10k allocs sans crash/leak.
- [ ] DumpStats reflète allocations par tag (Render vs World).
- [ ] Aucune free individuelle requise dans LinearArena.

### Notes / pièges à éviter
- Ne pas chercher un allocateur général complexe au début; SystemAllocator + arenas suffit.
- Préparer la destruction GPU différée plus tard (liste + fence).
