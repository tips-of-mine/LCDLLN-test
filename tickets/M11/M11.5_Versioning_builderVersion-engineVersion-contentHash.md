# M11.5 — Versioning: builderVersion/engineVersion/contentHash

## Objectif
Ajouter versioning dans chaque output pour invalidation cache et compatibilité.

## Dépendances
- M11.2

## Livrables
- Headers versionnés dans bin/pak/meta
- content hash (xxhash)


## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Refuser de charger si version incompatible (log clair).

## Étapes d’implémentation
1. Définir header commun (magic, version, hash).
2. Calculer hash inputs (layout + referenced assets timestamps/hashes).
3. Écrire header dans chaque fichier.
4. Runtime: valider avant load.

## Critères de validation (DoD)
- [ ] Modifier layout -> hash change; runtime détecte mismatch.

## Notes / pièges à éviter
- Éviter hash des gros fichiers à chaque build (cache).
