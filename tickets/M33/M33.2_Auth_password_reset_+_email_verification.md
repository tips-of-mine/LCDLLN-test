# M33.2_Auth_password_reset_+_email_verification

## Objectif
Reset password via email, vérification email registration.

## Dépendances
- M33.1

## Livrables
- Password reset flow (email link + token)
- Email verification on registration
- SMTP integration (SendGrid, AWS SES, etc.)
- Stockage DB des tokens reset/verification (tables dédiées ou colonnes)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Reset token: UUID, 1h expiry
- Email template: link https://game.com/reset?token=...
- On register: send verification code (6 digits)
- Account locked jusqu'à verification (optionnel)
- DB: tables reset_tokens (account_id, token, expires_at, used_at) et email_verifications (account_id, code, expires_at, verified_at)

## Étapes d'implémentation
1. /forgot-password: generate reset token, store DB (reset_tokens)
2. Send email avec link
3. User clicks link: validate token, prompt new password
4. /reset-password: verify token, hash new password, update DB, mark token used
5. Registration: send 6-digit code, store DB (email_verifications)
6. /verify-email: check code match, activate account
7. Integrate SMTP lib (Nodemailer, etc.)

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants

- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD

## Notes / pièges à éviter
- Token single-use (delete after use)
- Rate limit email sends (anti-spam)
