# M03.3 — Materials: BaseColor/Normal/ORM + loader

## Objectif
Système matériau minimal : textures BaseColor, Normal, ORM, sampler states, binding stable.

## Dépendances
- M03.1

## Livrables
- Material struct (texture handles + params)
- Loader textures (stb_image/ktx) + sRGB/linear flags
- Descriptors set layout pour materials

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- ORM packing: R=AO, G=Roughness, B=Metallic.
- BaseColor sRGB, normal/ORM linear.

## Étapes d’implémentation
1. Définir `Material` (indices textures, tiling, flags).
2. Implémenter chargeur textures + upload GPU.
3. Créer descriptor sets (per-material ou bindless plus tard).
4. Brancher sur geometry pass.

## Critères de validation (DoD)
- [ ] Changer textures modifie rendu (albedo/roughness/metal).
- [ ] Pas d'erreurs sRGB vs linear (visuel cohérent).

## Notes / pièges à éviter
- Prévoir cache texture (hash path).
