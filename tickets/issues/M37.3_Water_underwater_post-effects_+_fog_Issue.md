# M37.3_Water_underwater_post-effects_+_fog

## Objectif
Post-effects underwater (tint blue, fog distance, blur).

## Dépendances
- M37.1
- M08.2

## Livrables
- Underwater detection (camera below water plane)
- Color grading (blue tint)
- Fog (exponential depth-based)
- Blur vignette (optional)

## Spécification technique
- Detect: camera.y < water.y
- Tint: lerp color vers blue (depth-based)
- Fog: exp(-density * depth)
- Blur: slight gaussian (simule diffusion light)

## Étapes d'implémentation
1. Check camera position vs water planes
2. If underwater: enable post-FX
3. Tint shader: color *= vec3(0.5, 0.7, 1.0)
4. Fog: fogFactor = exp(-0.05 * depth), lerp(color, fogColor, fogFactor)
5. Optional blur: apply light gaussian
6. Disable when camera exits water

## Rapport final (M37.3)

### 1) FICHIERS

**Créés :**
- `engine/render/UnderwaterPass.h` — Classe `UnderwaterPass` avec struct `UnderwaterParams` (16 bytes : underwaterFactor, fogDensity, nearZ, farZ). Interface Init/Record/Destroy suivant le pattern TonemapPass. Descriptor set 0 : binding 0 = SceneColorHDR, binding 1 = depth.
- `engine/render/UnderwaterPass.cpp` — Implémentation complète : render pass (DONT_CARE), descriptor set layout (2 samplers), pool, sampler linéaire (couleur) + nearest (depth), pipeline layout avec push constants (16 bytes), pipeline fullscreen triangle sans vertex input. Logs Init/Destroy/Trace conformes aux règles.
- `game/data/shaders/underwater.vert` — Vertex shader fullscreen triangle (même pattern que tonemap.vert).
- `game/data/shaders/underwater.frag` — Fragment shader implémentant : (1) tint bleu `color *= vec3(0.5, 0.7, 1.0)` modulé par `underwaterFactor`, (2) fog exponentiel `fogFactor = 1 - exp(-density * linearDepth)`, (3) blur 5-tap box vignette optionnel, (4) pass-through pur si `underwaterFactor == 0.0`.
- `tickets/issues/M37.3_Water_underwater_post-effects_+_fog_Issue.md`
