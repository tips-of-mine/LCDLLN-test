# M09.10 — Engine : calcul de la distance réelle pour la sélection LOD

## Objectif
Corriger le passage de `0.0f` comme distance à `LodConfig::GetLodLevel` dans
`Engine.cpp`. La distance caméra ↔ objet doit être calculée à partir de la position
caméra disponible dans `RenderState`, pour que le sélecteur LOD introduit en M09.3
soit réellement opérationnel.

## Contexte
Dans la lambda geometry pass de `Engine.cpp` (introduite en M09.3) :
```cpp
const uint32_t lodLevel = m_lodConfig.GetLodLevel(0.0f); // ← distance hardcodée → toujours LOD0
```
Le système LOD est complet (`LodConfig`, `GeometryPass::Record(lodLevel)`, distances
config 0/25/60/150/400 m) mais cette ligne neutralise entièrement la sélection.
La caméra et sa position sont disponibles dans `RenderState` depuis M03.0.
`ChunkDrawDecision::distanceMeters` (M09.5) est également disponible pour les chunks.

## Dépendances
- M09.3 (LodConfig + GeometryPass::Record avec lodLevel)
- M09.5 (ChunkDrawDecision::distanceMeters disponible comme référence)
- M03.0 (Camera + position disponible dans RenderState)

## Livrables
- `engine/Engine.cpp`
  - Dans la lambda geometry pass : calculer `distCam` = distance euclidienne entre
    `camera.position` et le centre approximatif du mesh de test (position origine monde).
  - Passer `distCam` à `m_lodConfig.GetLodLevel(distCam)`.
  - Ajouter un `LOG_DEBUG` au changement de niveau LOD pour faciliter la validation.

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
```cpp
// Mesh de test positionné à l'origine → distance = norme du vecteur caméra
const float dx = camera.position.x;
const float dy = camera.position.y;
const float dz = camera.position.z;
const float distCam = std::sqrt(dx*dx + dy*dy + dz*dz);

const uint32_t lodLevel = m_lodConfig.GetLodLevel(distCam);
```
- Si le mesh de test est à une position non nulle, utiliser sa position réelle.
- Si `m_chunkDrawDecisions` est non vide (M09.5), utiliser `decisions[0].distanceMeters`
  comme alternative pour le chunk le plus proche.
- Documenter en commentaire que ce calcul est provisoire pour le mesh de test unique
  et sera remplacé par une boucle par instance dans un ticket dédié.

## Étapes d'implémentation
1. Localiser la lambda geometry pass dans `Engine.cpp`.
2. Calculer `distCam` à partir de `camera.position`.
3. Remplacer `GetLodLevel(0.0f)` par `GetLodLevel(distCam)`.
4. Ajouter `LOG_DEBUG` pour tracer le niveau LOD sélectionné.

## Notes / pièges à éviter
- Ne pas modifier `LodConfig` ni `GeometryPass` : seul `Engine.cpp` est dans la portée.
- Si le mesh n'a qu'un seul LOD réel, le switch LOD sera inobservable visuellement
  mais le code doit être correct pour accueillir de futurs assets multi-LOD.
- Utiliser `std::sqrt` ici est acceptable (chemin non-hot : calcul unique par frame).
