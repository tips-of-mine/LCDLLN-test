# M21.1 — MySQL schema.sql initial + schema_version

## Objectif
Établir le schéma initial MySQL 8.0 et la table `schema_version` (source of truth).

## Dépendances
- M00.1
- MySQL 8.0

## Livrables
- `db/schema.sql`
- Tables minimales:
  - accounts (auth)
  - sessions (optionnel si persistant)
  - characters (placeholder, max 5 rule)
  - shards (pour M22)
  - schema_version
- Indexes & contraintes (unique email/login)

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- MySQL 8.0 uniquement.
- `schema_version`:
  - `version INT PRIMARY KEY`
  - `applied_at TIMESTAMP`
  - `checksum CHAR(64)` (sha256)
- Transactions: utiliser InnoDB.

## Étapes d’implémentation
1. Définir tables + indexes.
2. Définir `schema_version` version=1.
3. Documenter comment initialiser DB.

## Critères de validation (DoD)
- [ ] schema.sql s’applique sur DB vierge.
- [ ] Indexes/uniques présents.
