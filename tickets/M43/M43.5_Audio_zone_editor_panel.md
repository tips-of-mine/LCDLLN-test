# M43.5 — Éditeur de zones audio (`game/data/audio/zone_audio.json`)

**Priorité :** Haute  
**Périmètre :** `engine/editor/EditorAudioPanel.cpp/.h` · `engine/editor/EditorMode.cpp/.h`  
**Dépendances :** M43.4 (ImGui intégré) · M17.4 (AudioEngine + structures `AudioSoundDefinition`, `AudioBusDefinition`, `ZoneAmbienceDefinition`)

---

## Objectif

Ajouter dans le mode `--editor` un panneau ImGui permettant de **lire, modifier et sauvegarder** le fichier `game/data/audio/zone_audio.json` sans quitter l'application. Les changements sont écrits sur disque à la demande (bouton **Save**).

---

## Contexte

Le fichier `audio/zone_audio.json` contient trois sections :

| Section | Contenu |
|---------|---------|
| `buses` | Liste de buses audio (id, volume 0.0–1.0) |
| `sounds` | Sons disponibles (id, path, bus, minDist, maxDist, loop) |
| `zones` | Liaisons zone → son d'ambiance (zoneId, soundId, position XYZ) |

Actuellement ce fichier est édité manuellement en JSON. Toute erreur de syntaxe crashe le chargement. Un panneau ImGui élimine ce risque et accélère l'itération.

---

## Changements requis

### Nouveaux fichiers

**`engine/editor/EditorAudioPanel.h`**
```cpp
#pragma once
#include "engine/audio/AudioEngine.h"
#include "engine/core/Config.h"
#include <string>
#include <vector>

namespace engine::editor
{
    /// Panneau ImGui d'édition de zone_audio.json.
    class EditorAudioPanel final
    {
    public:
        /// Charge le fichier depuis paths.content/audio/zone_audio.json.
        bool Load(const engine::core::Config& config);

        /// Sauvegarde le fichier sur disque (écrase l'existant).
        bool Save(const engine::core::Config& config);

        /// Dessine le panneau ImGui — appeler entre ImGui::Begin/End.
        void Draw();

        bool IsDirty() const { return m_dirty; }

    private:
        std::vector<engine::audio::AudioBusDefinition>      m_buses;
        std::vector<engine::audio::AudioSoundDefinition>    m_sounds;
        std::vector<engine::audio::ZoneAmbienceDefinition>  m_zones;
        std::string m_loadedPath;
        bool m_dirty = false;

        void DrawBusesSection();
        void DrawSoundsSection();
        void DrawZonesSection();
    };
}
```

**`engine/editor/EditorAudioPanel.cpp`**

- `Load()` : lire `zone_audio.json` via `nlohmann::json`, populer les trois vecteurs. Retourner `false` + `LOG_ERROR` si le fichier est absent ou invalide.
- `Save()` : sérialiser les trois vecteurs en JSON et écrire via `std::ofstream`. Émettre `LOG_INFO("[EditorAudio] Saved zone_audio.json ({} buses, {} sounds, {} zones)", ...)`.
- `Draw()` : trois onglets `ImGui::BeginTabBar("AudioTabs")` :

**Onglet Buses :**
```
[+] Add Bus
────────────────────────────────────────────
 id      volume
[SFX  ] [====|====] 1.00   [X]
[Music] [===|=====] 0.80   [X]
```
- `ImGui::InputText` pour l'id (max 32 chars)
- `ImGui::SliderFloat` pour le volume (0.0–1.0)
- Bouton `[X]` pour supprimer

**Onglet Sounds :**
```
[+] Add Sound
────────────────────────────────────────────
 id                   path                  bus    min   max  loop
[zone_0_ambience   ] [audio/zone_0_...    ] [Music] [8.0] [96.0] [ ]
```
- `ImGui::InputText` pour id, path (max 256 chars)
- `ImGui::Combo` pour bus (liste des ids de buses)
- `ImGui::InputFloat` pour minDistance / maxDistance
- `ImGui::Checkbox` pour loop

**Onglet Zones :**
```
[+] Add Zone
────────────────────────────────────────────
 zoneId  soundId              pos X    pos Y    pos Z
[0     ] [zone_0_ambience   ] [0.0   ] [0.0   ] [0.0   ]
```

Chaque modification positionne `m_dirty = true`.

### `engine/editor/EditorMode.h`

Ajouter :
```cpp
#include "engine/editor/EditorAudioPanel.h"

// dans la classe EditorMode :
engine::editor::EditorAudioPanel m_audioPanel;
```

### `engine/editor/EditorMode.cpp`

- Dans `Init()` : `m_audioPanel.Load(config);`
- Dans le rendu ImGui (après M43.4) : afficher le panneau dans une fenêtre dédiée :
  ```cpp
  if (ImGui::Begin("Audio Editor"))
      m_audioPanel.Draw();
  ImGui::End();
  ```
- Propager `m_audioPanel.IsDirty()` dans le flag `m_dirty` global de l'éditeur
- Associer le raccourci `Ctrl+S` au `m_audioPanel.Save(config)` quand la fenêtre Audio est active

## Format JSON produit

```json
{
  "buses": [
    { "id": "SFX",   "volume": 1.0 },
    { "id": "Music", "volume": 0.8 }
  ],
  "sounds": [
    {
      "id": "zone_0_ambience",
      "path": "audio/zone_0_ambience.wav",
      "bus": "Music",
      "minDistance": 8.0,
      "maxDistance": 96.0,
      "loop": true
    }
  ],
  "zones": [
    { "zoneId": 0, "ambienceSoundId": "zone_0_ambience", "position": [0.0, 0.0, 0.0] }
  ]
}
```

Le JSON sauvegardé doit être **lisible** (indentation 2 espaces, `nlohmann::json::dump(2)`).

## Critères d'acceptation

- [ ] Build Windows OK
- [ ] `--editor` : le panneau "Audio Editor" s'ouvre et affiche les données de `zone_audio.json`
- [ ] Modifier un volume de bus → le champ est mis à jour, indicateur `*dirty*` dans le titre
- [ ] `Ctrl+S` → fichier sauvegardé sur disque, indicateur `*dirty*` disparu
- [ ] Ajouter un son puis sauvegarder → le fichier JSON contient la nouvelle entrée
- [ ] Supprimer un bus → la liste est mise à jour
- [ ] Fichier absent au démarrage → `LOG_ERROR` sans crash, panneau vide éditable
- [ ] Rapport final : fichiers + commandes + résultats + DoD

---

## Interdit

- Ne pas modifier `AudioEngine.cpp/.h` — le panneau opère uniquement sur les fichiers JSON, pas sur le runtime audio
- Ne pas déclencher de rechargement audio live (hors scope — fonctionnalité ultérieure)
- Ne pas créer de dossier racine `/audio` ou `/data`
