# M28.1_Skills_data-driven_definitions_+_cooldowns

## Objectif
Système skills data-driven avec cooldowns, resource costs, casting time.

## Dépendances
- M14.1
- M00.4

 ## Livrables
- `game/data/skills/*.json` schema
- `engine/gameplay/SkillSystem.*`
- Cooldown tracking per player
- Resource costs (mana, rage, energy)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Skill JSON: id, name, cooldown, castTime, cost, range, effects
- Cooldown tracking: timestamp last use
- Resource types: health, mana, energy, rage, combo points
- Validation: check cooldown, resources, range, LOS

## Étapes d'implémentation
1. Définir schema skill.json (voir exemple ci-dessous)
2. Parser et charger skills au boot
3. SkillSystem: UseSkill(playerId, skillId, targetId)
4. Validate: cooldown ready, resources sufficient, in range
5. Apply cost, start cooldown timer, trigger cast

## Exemple skill.json
```json
{
  "id": "fireball",
  "name": "Fireball",
  "cooldown": 4.0,
  "castTime": 1.5,
  "cost": {"mana": 50},
  "range": 30.0,
  "effects": [
    {"type": "damage", "amount": 100, "damageType": "fire"}
  ]
}
```
