# M02.2 — Frame Graph: compilation dépendances + ordre d'exécution

## Objectif
Calculer dépendances read/write entre passes et exécuter en ordre topologique.

## Dépendances
- M02.1

## Livrables
- Graphe de dépendances (writer→reader)
- Topological sort + détection cycles

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Une ressource écrite puis lue crée une dépendance.
- Support multi-writers interdit en MVP (assert), ou définir règles claires.

## Étapes d’implémentation
1. Construire edges passA->passB via lastWriter.
2. Effectuer tri topo (Kahn/DFS).
3. Assertions cycles et multi-writer (MVP).
4. Exécuter dans ordre compilé.

## Critères de validation (DoD)
- [ ] Graph compile pour 3+ passes, ordre correct.
- [ ] Cycle détecté et log fatal.

## Notes / pièges à éviter
- Plus tard: permettre multi-writer via subresources/attachments.
