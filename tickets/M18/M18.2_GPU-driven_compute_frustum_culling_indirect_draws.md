# M18.2 — GPU-driven: compute frustum culling + indirect draws

## Objectif
Réduire coût CPU: GPU culling (compute) + draw indirect.

## Dépendances
- M18.1
- M09.3

## Livrables
- DrawItem buffer
- Visible list append
- Indirect command buffer

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- vkCmdDrawIndexedIndirect (ou indirect count).

## Étapes d’implémentation
1. Définir DrawItem struct GPU (mesh/material/transform/bounds).
2. Compute: frustum test -> write indirect commands.
3. Graphics: consume indirect in geometry pass.
4. Comparer perf vs CPU loop.

## Critères de validation (DoD)
- [ ] CPU render thread time baisse nettement en scène dense.

## Notes / pièges à éviter
- Commencer sans occlusion; ajouter Hi-Z ensuite.
