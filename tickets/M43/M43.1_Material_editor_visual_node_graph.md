# M43.1_Material_editor_visual_node_graph

## Objectif
Éditeur matériau visuel (node graph) pour créer shaders sans code.

## Dépendances
- M3.3
- M12.1

## Livrables
- Node editor UI (ImNodes ou custom)
- Nodes: texture sample, multiply, add, lerp, etc.
- Compile graph -> shader code (GLSL)
- Preview material realtime (sphere preview)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Nodes: inputs/outputs (typed: float, vec3, texture)
- Connections: drag wire entre pins
- Compile: traverse graph, generate GLSL code
- Output: final shader .spv (SPIR-V)

## Étapes d'implémentation
1. UI: canvas pour placer nodes
2. Node types:
   - Input: texture, color, scalar
   - Math: add, multiply, lerp, clamp
   - Output: albedo, normal, roughness, etc.
3. Connect nodes: drag pin -> pin
4. Graph validation: type check, cycles detection
5. Compile:
   - Topological sort nodes
   - Generate code per node
   - Concatenate -> full shader
6. Compile shader (glslangValidator)
7. Preview: render sphere avec shader
