## Ticket content (M35.2) — référence

# M35.2_Vendors_buy sell_items_+_shop_UI

## Objectif
NPCs vendors avec inventory, prix buy/sell, UI shop.

## Dépendances
- M35.1
- M16.3

## Livrables
- Vendor NPC component (shop inventory)
- Buy/sell transactions
- Shop UI panel (vendor items + prices)
- Sell-back mechanic (sell price = 25% buy price)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Vendor inventory: limited stock (optional) ou infinite
- Buy price: defined per item
- Sell price: 25% buy price (default)
- Stock refresh: daily reset (optional)

## Étapes d'implémentation
1. Vendor NPC data: vendorId, items[] (itemId, buyPrice, stock)
2. Interact NPC: open shop UI
3. UI: display vendor items grid, buy button
4. Buy: validate gold, add item inventory, subtract gold
5. Sell: drag item to vendor, offer sell price, confirm
6. Server: validate transactions (anti-dupe)
7. Stock tracking: decrement on buy, reset daily (cron)

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants

- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD

## Notes / pièges à éviter
- Vendor types: general, armor, weapons, potions, etc.
- Reputation discounts (phase 2)
