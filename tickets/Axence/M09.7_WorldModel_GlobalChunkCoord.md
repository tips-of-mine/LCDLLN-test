# M09.7 — WorldModel : GlobalChunkCoord multi-zones (coordonnées globales non clampées)

## Objectif
Remplacer les coordonnées chunk clampées dans `[0, kChunksPerZoneAxis-1]` par des
coordonnées globales signées (`GlobalChunkCoord`) permettant de naviguer dans un monde
composé de plusieurs zones adjacentes sans discontinuité de coordonnées.

## Contexte
`World::WorldToChunkCoord` clamp les indices à `[0, 15]` (kChunksPerZoneAxis = 16).
Un joueur à x = 4200 m (zone 1 voisine) est silencieusement replacé en zone 0 :
les rings Active/Visible/Far autour de lui couvrent des chunks incorrects.
Le système de requests (`GetPendingChunkRequests`) produit alors des données
fausses que n'importe quel scheduler consommant ces requests interprétera mal.

Ce ticket corrige le calcul à la source, en restant dans le périmètre déjà livré
(M09.1 WorldModel, M09.2 ChunkBudgetStats, M09.5 HlodRuntime).

## Dépendances
- M09.1 (WorldModel existant)
- M09.2 (ChunkBudgetStats — consomme ChunkCoord)
- M09.5 (HlodRuntime — consomme ChunkCoord)

## Spécification technique
```
WorldToGlobalChunkCoord(float worldX, float worldZ):
    x = (int32_t)floor(worldX / kChunkSize)   // PAS de clamp
    z = (int32_t)floor(worldZ / kChunkSize)

GlobalToZoneAndLocal(GlobalChunkCoord g, ZoneCoord& zone, LocalChunkCoord& local):
    zone.x  = (int32_t)floor((float)g.x / kChunksPerZoneAxis)
    local.x = g.x - zone.x * kChunksPerZoneAxis   // toujours >= 0
    // idem pour z

ChunkBounds(GlobalChunkCoord g) -> ChunkBounds:
    minX = g.x * kChunkSize
    minZ = g.z * kChunkSize
    maxX = minX + kChunkSize
    maxZ = minZ + kChunkSize
```
- Utiliser `floor` (pas la division entière C++ qui tronque vers zéro pour les négatifs).
- Les rings restent calculés par delta depuis le chunk centre.

## Étapes d'implémentation
1. Définir `GlobalChunkCoord`, `LocalChunkCoord`, `ZoneCoord` dans `WorldModel.h`.
2. Implémenter `WorldToGlobalChunkCoord` (sans clamp).
3. Implémenter `GlobalToZoneAndLocal`.
4. Adapter `ChunkBounds` en world-space absolu.
5. Mettre à jour `World::Update` pour utiliser `GlobalChunkCoord`.
6. Propager `GlobalChunkCoord` dans `HlodRuntime`, `ChunkBudgetStats`, `Engine.cpp`.
7. Ajouter l'alias `using ChunkCoord = GlobalChunkCoord;` pour la compatibilité des call-sites.

## Notes / pièges à éviter
- Division entière C++ : `(-1) / 16 == 0` mais `floor(-1.0/16) == -1`. Toujours utiliser `floor`.
- `ChunkBudgetStats` utilise `ChunkCoord` comme clé de map : vérifier que le hash/comparateur couvre `int32_t`.
- Se limiter à la migration de type dans ce ticket : ne pas ajouter de logique de chargement ou scheduling.
