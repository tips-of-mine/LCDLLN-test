
**Objectif**
Commandes slash (/who, /ignore, /kick, etc.) et outils modération.

**Dépendances**
- M29.1

**Livrables**
- Command parser (/command args)
- Commands: /who, /ignore, /unignore, /report
- Admin commands: /kick, /ban, /mute, /announce
- Ignore list per player (persistent)

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- /who: liste joueurs online (zone ou global)
- /ignore <name>: block messages from player
- /report <name> <reason>: log pour modération
- /kick, /ban, /mute: admin only, log action

**Étapes d'implémentation**
1. Parse input: detect "/" prefix
2. Map commands -> handlers (server-side validation)
3. /who: query online players, return list
4. /ignore: add to blacklist (server persist), filter incoming chat
5. /report: log to DB with timestamp/reporter/target/reason
6. Admin commands: check permissions, apply action, broadcast notification
7. Persist ignore list in player DB

**Critères de validation (DoD)**
- [ ] /who liste joueurs online
- [ ] /ignore bloque messages player
- [ ] /report enregistré en DB
- [ ] /kick (admin) déconnecte player

**Notes / pièges à éviter**
- Permissions système pour admin (role-based)
- Log toutes actions modération (audit trail)
