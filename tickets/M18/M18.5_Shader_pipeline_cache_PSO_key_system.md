# M18.5 — Shader pipeline cache + PSO key system

## Objectif
Éviter stutter: compilation shaders offline/boot + pipeline cache Vulkan + keying.

## Dépendances
- M18.1

## Livrables
- Pipeline cache file
- PSO key hashing
- Warmup list

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Interdire création pipeline en plein gameplay (assert debug).

## Étapes d’implémentation
1. Mettre en place cache Vulkan (vkCreatePipelineCache).
2. Définir PSO key (shaders + states + formats).
3. Pré-créer pipelines au boot (warmup).
4. Sauver cache sur disque.

## Critères de validation (DoD)
- [ ] Première entrée en zone dense sans stutter de compilation.

## Notes / pièges à éviter
- Sur drivers différents, cache peut être invalide -> handle gracefully.
