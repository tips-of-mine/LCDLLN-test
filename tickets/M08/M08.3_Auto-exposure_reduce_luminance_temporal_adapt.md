# M08.3 — Auto-exposure: reduce luminance + temporal adapt

## Objectif
Calculer exposure dynamique (log-average) via compute reduction et lissage temporel.

## Dépendances
- M08.2

## Livrables
- Compute pass reduce luminance -> 1 value
- Persistent buffer exposure
- Param key/speed

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- exposure = lerp(prev, target, 1-exp(-dt*speed)); key=0.18.

## Étapes d’implémentation
1. Compute luminance buffer (downsample) ou direct reduction.
2. Calcul log-average, targetExposure.
3. Adapter temporellement et stocker.
4. Utiliser exposure dans tonemap.
