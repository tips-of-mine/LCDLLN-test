# M08.6 — AutoExposure : histogramme luminance 256 bins (remplacement grille 64×64)

## Objectif
Remplacer la réduction luminance par grille 64×64 (sensible aux outliers) par un
histogramme compute en 256 bins. Utiliser un percentile bas (ex. 10%) pour ignorer
les pixels très sombres (sky occultée) et un percentile haut (90%) pour ignorer
les hauts feux. Résultat : exposition robuste dans toutes les conditions de scène MMORPG.

## Contexte
La grille 64×64 (4 096 samples) calcule une moyenne log directe. Un seul soleil en
surbrillance ou une zone très sombre peuvent fausser l'exposition globale. Les moteurs
AAA utilisent un histogramme sur toute l'image (ou une résolution réduite) avec
sélection par percentile.

## Dépendances
- M08.5 (AutoExposure double-buffering — à appliquer après)
- M08.3 (AutoExposure existant)

## Livrables
- `game/data/shaders/luminance_histogram.comp`
  - Pass 1 : chaque workgroup accumule un histogramme local 256 bins (log-luminance linéarisé).
  - Atomic adds sur un buffer partagé `histogram[256]` (storage buffer).
- `game/data/shaders/luminance_histogram_avg.comp`
  - Pass 2 : lit `histogram[256]`, ignore les bins < percentile_low et > percentile_high,
    calcule la moyenne log-pondérée, écrit `avgLog` dans le buffer exposure.
- `engine/render/AutoExposure.h`
  - Ajouter buffer `m_histogramBuffer[kAESlots]` (256 × uint32, device-local, storage).
  - Ajouter pipeline + layout pour les 2 passes compute.
  - Paramètres config : `exposure.histogram_percentile_low` (défaut 0.10), `exposure.histogram_percentile_high` (défaut 0.90).
- `engine/render/AutoExposure.cpp`
  - `Init` : créer pipelines + buffers histogram VMA (DEVICE_LOCAL, STORAGE).
  - `Record` : dispatch pass 1 (reset histogram + accumulation) puis pass 2 (avg → staging).
  - Supprimer l'ancien compute de grille 64×64.
- `engine/Engine.cpp` / config
  - Exposer `exposure.histogram_percentile_low/high` dans `config.json` (valeurs par défaut).

## Spécification technique
### Pass 1 — Accumulation histogramme
```glsl
// luminance_histogram.comp
// workgroup: 16x16, image réduite à 1/4 (downsample léger)
// Pour chaque pixel : L = dot(rgb, vec3(0.2126, 0.7152, 0.0722))
// bin = clamp(int(log2(L + 1e-5) * scale + offset), 0, 255)
// atomicAdd(histogram[bin], 1u)
```
- Reset `histogram[256]` à 0 en début de pass (compute séparé ou premier workgroup).

### Pass 2 — Moyenne avec percentile
```glsl
// luminance_histogram_avg.comp — 1 workgroup de 256 threads
// Scan préfixe pour trouver les bins low% et high%
// Calcul log-moyenne sur les bins retenus
// Écriture dans staging (float)
```
- `avgLuminance = exp(somme(log_L * count) / total_retained)`

### Update CPU
- Conserver `exposure = lerp(prev, key / avgLuminance, 1 - exp(-dt * speed))`.

## Étapes d'implémentation
1. Écrire `luminance_histogram.comp` (accumulation, reset, atomic).
2. Écrire `luminance_histogram_avg.comp` (scan + percentile + écriture).
3. Créer les pipelines + buffer histogram dans `AutoExposure::Init`.
4. Mettre à jour `AutoExposure::Record` : dispatcher les 2 passes.
5. Supprimer l'ancien shader `luminance_reduce.comp` et son pipeline.
6. Exposer les params percentile dans `config.json`.

## Notes / pièges à éviter
- Mettre une barrière entre pass 1 et pass 2 (`SHADER_WRITE → SHADER_READ` sur le buffer histogram).
- Le reset du histogram doit se faire avant l'accumulation chaque frame (compute séparé ou `memset` via vkCmdFillBuffer).
- Travailler sur une résolution réduite (1/4) pour limiter le coût de la pass 1.
- Ne pas oublier de supprimer le shader `luminance_reduce.comp` du CMakeLists.
