# M23.4 — Logging advanced: rotation + levels config

**Status:** Closed

## Objectif
Configurer logs production-ready: rotation taille/date, niveaux configurables, séparation audit/security vs runtime.

## Dépendances
- M00.1 (logging base)

## Livrables
- spdlog rotating/daily logger
- config:
  - log_level (DEBUG/INFO/WARN/ERROR)
  - rotation_size_mb / retention_days
- loggers séparés:
  - `runtime`
  - `security_audit` (M20.4)
- Tests smoke rotation

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Pas d'info sensible dans runtime logs.
- audit logs: events sécurité, pas de secrets.

## Étapes d'implémentation
1. Implémenter loggers.
2. Ajouter config.
3. Vérifier rotation sur taille/date.

## Critères de validation (DoD)
- [x] Rotation fonctionne.
- [x] Niveaux modifiables via config.
