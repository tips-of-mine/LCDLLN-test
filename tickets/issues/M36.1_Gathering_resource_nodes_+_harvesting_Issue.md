## Ticket content (M36.1) — référence

# M36.1_Gathering_resource_nodes_+_harvesting

## Objectif
Nodes ressources (mining, herbalism, skinning) avec spawn/despawn, harvesting mini-game.

## Dépendances
- M09.1
- M16.3

## Livrables
- Resource node spawner (positions, type, respawn timer)
- Harvesting interaction (progress bar)
- Loot table par node
- Skill level requirement (optional)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Node types: ore_vein, herb, tree, animal_corpse (skinnable)
- Harvesting time: 2-5s (cast bar)
- Respawn: 5-15min per node
- Loot: 1-5 resources, quality variable

## Étapes d'implémentation
1. Define node types (JSON: id, mesh, loot_table, respawn_time)
2. Spawn nodes dans monde (zone config)
3. Interaction: E key proximity check
4. Start harvest: cast bar 3s
5. Interruption: movement/damage cancels
6. On complete: roll loot table, add to inventory
7. Despawn node, start respawn timer
8. Server: validate proximity, prevent teleport-harvesting

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD

## Notes / pièges à éviter
- Phasing: concurrent players voir same node (race condition)
- Shared loot vs individual (individual = moins frustration)
