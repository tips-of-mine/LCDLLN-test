# M14.2 — Aggro + AI states (Idle/Patrol/Aggro/Return)

## Objectif
IA mobs basique + threat table pour cibler joueur.

## Dépendances
- M11.3
- M14.1

## Livrables
- AI state machine
- Threat table
- Leash/return


## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Threat += damage; target = max threat; leash distance configurable.

## Étapes d’implémentation
1. Définir states + transitions.
2. Mettre à jour threat à chaque CombatEvent.
3. Pathfind vers target (navmesh) si dans chunks actifs.
4. Return/reset threat hors leash.

## Critères de validation (DoD)
- [ ] Mob agro, poursuit, attaque, puis reset s'il est kite trop loin.

## Notes / pièges à éviter
- Limiter update rate IA (ex 10Hz) pour perf.
