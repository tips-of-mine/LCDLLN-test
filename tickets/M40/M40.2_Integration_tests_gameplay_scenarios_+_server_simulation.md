# M40.2_Integration_tests_gameplay_scenarios_+_server_simulation

## Objectif
Tests intégration pour scénarios gameplay (login, combat, trade, etc.).

## Dépendances
- M40.1
- M13.1

## Livrables
- Test scenarios: login flow, movement, combat, trade
- Headless server mode (no rendering)
- Bot clients (scripted actions)
- Assertions: state validation (HP, inventory, position)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Headless server: no window, no rendering
- Bot client: connect, send inputs, validate snapshots
- Scenarios: scripted sequences (move, attack, loot)

## Étapes d'implémentation
1. Server: headless mode flag (--headless)
2. Bot client: minimal client (no rendering, scripted input)
3. Test scenario example:
   ```cpp
   TEST_CASE("Combat scenario") {
     Server server(headless=true);
     BotClient player1, player2;
     player1.connect(server);
     player2.connect(server);
     player1.move({10,0,10});
     player1.attack(player2);
     REQUIRE(player2.HP < player2.MaxHP);
   }
   ```
4. Run scenarios: spawn server, bots, simulate, assert
5. Teardown: disconnect, cleanup

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD

## Notes / pièges à éviter
- Deterministic tests (seed RNG)
- Timeout tests (max 10s per scenario)
