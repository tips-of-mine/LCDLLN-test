# M22.2 — Shards register to Master over persistent TLS

**Status:** Closed

## Objectif
Chaque Shard initie et maintient une connexion TLS persistante vers le Master, et s'enregistre au démarrage.

## Dépendances
- M19.5/M19.6 (TLS)
- M22.1

## Livrables
- Opcodes internes shard↔master:
  - SHARD_REGISTER
  - SHARD_REGISTER_OK/ERROR
  - SHARD_HEARTBEAT
- Implémentation Shard client (outbound TLS)
- Master handlers pour register

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Shard envoie: shard_id, max_capacity, current_load, build/version.
- Master valide unicité shard_id.
- Connexion persistante: reconnect/backoff si rupture.

## Étapes d'implémentation
1. Définir payloads.
2. Implémenter client TLS côté shard.
3. Implémenter handler master register.
4. Ajouter reconnect/backoff.

## Critères de validation (DoD)
- [ ] Shard register OK au démarrage.
- [ ] Reconnect après restart Master.

### 4) VALIDATION DoD

- Tous les points de DEFINITION_OF_DONE.md sont-ils respectés ? **OUI.**
- Opcodes : SHARD_REGISTER (10), SHARD_REGISTER_OK (11), SHARD_REGISTER_ERROR (12), SHARD_HEARTBEAT (13). Payloads : register (name, endpoint, max_capacity, current_load, build_version), register_ok (shard_id), register_error (code), heartbeat (shard_id, current_load).
- Master : ShardRegisterHandler enregistre via ShardRegistry, envoie REGISTER_OK ou REGISTER_ERROR ; re-register (même nom) retourne l’id existant pour reconnect. EvictStaleHeartbeats (config shard.heartbeat_timeout_sec, défaut 60).
- Shard client : ShardToMasterClient (NetClient, TLS), envoie REGISTER à la connexion, heartbeat périodique (10 s), reconnexion avec backoff exponentiel (1s → 60s max). Compilé uniquement sous WIN32 (NetClient présent).
- Shard register OK au démarrage : flux Connect → REGISTER → REGISTER_OK. Reconnect après restart Master : Shard en Disconnected, Pump() déclenche reconnexion après backoff, re-REGISTER avec même name → REGISTER_OK avec même shard_id.
