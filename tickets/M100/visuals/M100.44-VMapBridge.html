M100.44 — VMap Bridge Outil offline Runtime server Tests d'intégration
Pipeline de bake
1. Éditeur (M100.35-43)
terrain.bin (érosion appliquée)
mesh_inserts.bin (4 catégories : cave + overhang + arch + dungeon)
dungeons.bin (métadonnées portails)
splat.bin, water.bin
▼ Save Zone Bundle
2. Outil offline lcdlln_vmap_extractor
--zone-bundle path --out vmap_tiles/
• Bake heightmap classique (CMANGOS.05)
• Pour chaque mesh insert dans la tile : ajoute AuxiliaryColliders
• SHA-256 du mesh_inserts.bin embarqué en header
• Écrit .vmap v(N+1) par tile (533 × 533 m)
▼ Deploy (rsync / S3) + UPDATE zone_content_versions
3. Shard server
• Refuse .vmap v(N) → message clair
• Vérifie hash header vs zone_content_versions
• AuxiliaryColliderCache : load .collision.bin partagé
• LOS/GetHeight/Raycast incluent les colliders
• Hot-reload non supporté → cold restart
Extension format .vmap
Header étendu
struct VmapTileHeader {
  char     magic[4];     // "VMAP"
  uint32_t version;      // N+1 (bumpé)
  uint32_t tileX, tileZ;
  byte     meshInsertsBinHash[32]; // ← NOUVEAU SHA-256
  uint32_t auxiliarySectionOffset; // ← NOUVEAU
  // ... sections classiques (heightmap, etc.)
};
Section AUXC (auxiliary colliders)
[AuxiliaryColliders]
  magic   : "AUXC"
  version : 1
  count   : uint32

  for each collider:
    guid             : 16 bytes
    meshPath         : "meshes/caves/cave_medium_01.gltf"
    worldTransform   : float32[16]  // 4x4 row-major
    localAABB        : float32[6]   // min + max
    collisionProxy   : "meshes/caves/cave_medium_01.collision.bin"
    flags            : u8
      bit 0 = blockLOS
      bit 1 = blockMovement
      bit 2 = isDungeonPortalSolid
    dungeonId        : "dungeon_001_ancient_ruin" (or empty)
