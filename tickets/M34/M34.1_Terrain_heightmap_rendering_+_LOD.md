# M34.1_Terrain_heightmap_rendering_+_LOD

## Objectif
Rendu terrain basé heightmap avec LOD géomorphing pour grandes distances.

## Dépendances
- M03.1
- M09.1

## Livrables
- Heightmap loader (R16 ou R32F texture)
- Terrain mesh generation (grid patches)
- LOD system (4-8 levels)
- Géomorphing transitions

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Heightmap: 1025x1025 (power-of-2 + 1)
- Patches: 16x16 quads (257x257 heightmap -> 16x16 patches)
- LOD: distance-based (0-50m: full, 50-200m: half-res, etc.)
- Géomorphing: blend vertices vers LOD parent (smooth)

## Étapes d'implémentation
1. Charger heightmap texture (R16 format)
2. Générer patches mesh (index buffer per LOD)
3. Vertex shader: sample heightmap, displace Y
4. LOD selection: distance camera->patch center
5. Géomorphing: lerp vertex position selon distance transition
6. Frustum culling patches
7. Normal map generation (sobel filter heightmap)
