# M13.4 — Zone transitions: server authoritative + client preload hook

## Objectif
Permettre changement de zone via volume transition, validation serveur, respawn position.

## Dépendances
- M13.3
- M11.2

## Livrables
- Message ZoneChange
- Server zone assignment
- Client clears + requests spawns

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Client ne décide pas la zone; serveur valide.

## Étapes d’implémentation
1. Serveur: détecter entrée volume transition.
2. Envoyer ZoneChange (zoneId + spawn pos).
3. Client: reset streaming, précharger nouvelle zone, reset TAA history.
4. Serveur: recalculer interest set dans nouvelle zone.

## Critères de validation (DoD)
- [ ] Passer zone_001 -> zone_002 sans crash; entités se resynchronisent.

## Notes / pièges à éviter
- Masquer transition par tunnel/brume côté level design.
