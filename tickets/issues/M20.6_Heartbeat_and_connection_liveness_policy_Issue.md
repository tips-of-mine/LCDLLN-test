# M20.6 — Heartbeat & liveness: keepalive policy (client+server)

**Status:** Closed

## Objectif
Implémenter heartbeat applicatif v1 pour détecter clients offline et appliquer les règles d'expiration de session.

## Dépendances
- M19.7 (dispatcher)
- M20.3 (sessions)

## Livrables
- Opcode `HEARTBEAT` (request/response ou ping/pong)
- Client timer heartbeat interval (configurable)
- Serveur:
  - last_seen mis à jour
  - heartbeat timeout => close session + disconnect
- Règle 24h max enforce (même si heartbeats OK)

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Interval par défaut: 30s
- Timeout: 90–120s (3–4 pings ratés)
- Heartbeat doit être lightweight (pas DB hit à chaque ping; update in-memory + flush batch optionnel)

## Étapes d'implémentation
1. Définir payload heartbeat minimal.
2. Ajouter timers côté client.
3. Handler serveur: update last_seen.
4. Watchdog serveur: expire sessions.
5. Tests integration: couper client, vérifier expiration.

## Critères de validation (DoD)
- [ ] Client offline détecté et session close.
- [ ] Heartbeat stable sans surcharge DB.
