# M15.4 — CMakeLists : target `server_app` (binaire serveur headless)

## Contexte / Problème
Depuis M13.1, du code serveur existe sous `engine/server/` (ServerApp, ServerProtocol,
CharacterPersistence, ReplicationTypes, QuestRuntime…) mais **aucun target CMake ne le
compile**. Ces fichiers sont ignorés par le build. Toute validation DoD des tickets M13–M15
côté serveur est donc non testable.

Ce ticket corrige le gap en créant le target `server_app` et son point d'entrée.

## Dépendances
- M13.1 (ServerApp.cpp existe)
- M14.4 (CharacterPersistence.cpp existe)
- M00.1 (Log, Config)
- M00.3 (FileSystem)

## Livrables

### Fichiers à créer
- `engine/server/CMakeLists.txt` — déclaration du target `server_app`
- `engine/server/main_server.cpp` — point d'entrée du binaire serveur

### Fichiers à modifier
- `CMakeLists.txt` (racine) — ajout de `add_subdirectory(engine/server)` après
  `add_subdirectory(tools/zone_builder)`

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique

### Contrainte fondamentale : serveur headless (pas de Vulkan, pas de GLFW)
`engine_core` est liée à Vulkan et GLFW. Le serveur **ne doit pas** lier `engine_core`.
Il compile uniquement les sources nécessaires en les référençant via `${CMAKE_SOURCE_DIR}/`,
sur le modèle de `tools/zone_builder/CMakeLists.txt`.

### Sources à inclure dans `server_app`
Sources propres au serveur (toutes dans `engine/server/`) :
```
engine/server/main_server.cpp
engine/server/ServerApp.cpp
engine/server/ServerProtocol.cpp
engine/server/CharacterPersistence.cpp
```
> ⚠️ Si d'autres `.cpp` ont été créés dans `engine/server/` par des tickets M13–M15
> (ex. `QuestRuntime.cpp`, `SpawnerSystem.cpp`…), les ajouter également.
> L'implémenteur doit lister tous les `.cpp` présents dans `engine/server/` avant de
> rédiger le CMakeLists.

Sources utilitaires moteur partagées (référencées par chemin absolu) :
```
${CMAKE_SOURCE_DIR}/engine/core/Config.cpp
${CMAKE_SOURCE_DIR}/engine/core/Log.cpp
${CMAKE_SOURCE_DIR}/engine/platform/FileSystem.cpp
```

### Contenu de `engine/server/main_server.cpp`
```cpp
// engine/server/main_server.cpp
#include "engine/core/Config.h"
#include "engine/core/Log.h"
#include "engine/server/ServerApp.h"

int main(int argc, char** argv)
{
    engine::core::Log::Init();
    LOG_INFO(Net, "[ServerApp] Starting server...");

    engine::core::Config config;
    if (!config.LoadFromFile("config.json"))
    {
        LOG_ERROR(Net, "[ServerApp] Failed to load config.json");
        return 1;
    }

    engine::server::ServerApp app(config);
    if (!app.Init())
    {
        LOG_ERROR(Net, "[ServerApp] Init failed");
        return 1;
    }

    app.Run();
    app.Shutdown();

    LOG_INFO(Net, "[ServerApp] Clean shutdown");
    return 0;
}
```
> ⚠️ Si `ServerApp` n'expose pas exactement les méthodes `Init()`, `Run()`, `Shutdown()`,
> adapter `main_server.cpp` pour correspondre à l'API réelle existante dans `ServerApp.h`.
> Ne pas modifier `ServerApp.h` dans ce ticket.

### Contenu de `engine/server/CMakeLists.txt`
```cmake
# M15.4 — server_app : binaire serveur headless (pas de Vulkan, pas de GLFW)
cmake_minimum_required(VERSION 3.20)
set(CMAKE_CXX_STANDARD 20)

add_executable(server_app
    main_server.cpp
    ServerApp.cpp
    ServerProtocol.cpp
    CharacterPersistence.cpp
    # Ajouter ici tout autre .cpp présent dans engine/server/
    ${CMAKE_SOURCE_DIR}/engine/core/Config.cpp
    ${CMAKE_SOURCE_DIR}/engine/core/Log.cpp
    ${CMAKE_SOURCE_DIR}/engine/platform/FileSystem.cpp
)

target_include_directories(server_app PRIVATE
    ${CMAKE_SOURCE_DIR}
)

set_target_properties(server_app PROPERTIES
    OUTPUT_NAME "lcdlln_server"
    RUNTIME_OUTPUT_DIRECTORY         "${CMAKE_BINARY_DIR}/pkg/server"
    RUNTIME_OUTPUT_DIRECTORY_RELEASE "${CMAKE_BINARY_DIR}/pkg/server"
    RUNTIME_OUTPUT_DIRECTORY_DEBUG   "${CMAKE_BINARY_DIR}/pkg/server"
)

if(MSVC)
    target_compile_options(server_app PRIVATE /W4 /permissive- /Zc:preprocessor)
else()
    target_compile_options(server_app PRIVATE -Wall -Wextra -Wpedantic)
endif()

# Copie config.json à côté du binaire serveur si présent
if(EXISTS "${CMAKE_SOURCE_DIR}/config.json")
    add_custom_command(TARGET server_app POST_BUILD
        COMMAND ${CMAKE_COMMAND} -E copy_if_different
                "${CMAKE_SOURCE_DIR}/config.json"
                "$<TARGET_FILE_DIR:server_app>/config.json"
        COMMENT "Copie de config.json pour server_app..."
    )
endif()
```

### Modification du `CMakeLists.txt` racine
Ajouter **après** la ligne `add_subdirectory(tools/zone_builder)` :
```cmake
# M15.4: Server app (headless, no Vulkan)
add_subdirectory(engine/server)
```

## Étapes d'implémentation
1. Lister tous les `.cpp` présents dans `engine/server/` (répertoire réel du dépôt).
2. Créer `engine/server/main_server.cpp` avec le contenu spécifié ci-dessus, en adaptant
   l'appel API si nécessaire.
3. Créer `engine/server/CMakeLists.txt` en ajoutant tous les `.cpp` listés à l'étape 1.
4. Modifier le `CMakeLists.txt` racine : ajouter `add_subdirectory(engine/server)`.
5. Builder le projet : `cmake --preset vs2022-x64` puis
   `cmake --build --preset vs2022-x64-release`.
6. Vérifier que `lcdlln_server.exe` est produit dans `build/vs2022-x64/pkg/server/`.
7. Lancer `lcdlln_server.exe` et vérifier qu'il démarre sans crash immédiat.

## Critères de validation (DoD)
- [ ] Build Windows OK (configure + build) via presets existants.
- [ ] `lcdlln_server.exe` présent dans `build/.../pkg/server/`.
- [ ] `lcdlln_server.exe` démarre et affiche les logs de boot, sans crash immédiat.
- [ ] Le build de `lcdlln.exe` (client) n'est pas cassé.
- [ ] Aucun dossier racine non autorisé créé.
- [ ] Rapport final : fichiers créés/modifiés + commandes + résultats + DoD.

## Notes / pièges à éviter
- Ne **pas** lier `engine_core` : elle tire Vulkan et GLFW, inutilisables sans GPU/fenêtre.
- Ne **pas** ajouter `Vulkan::Vulkan`, `glfw` ni `VMA` au `target_link_libraries` de `server_app`.
- Si des headers de `engine/server/` incluent des headers Vulkan indirectement, corriger
  les includes dans ce ticket (inclure un `#ifdef SERVER_BUILD` guard si nécessaire) —
  mais ne pas réécrire la logique métier.
- Le nom de l'exécutable est `lcdlln_server` (pas `server_app`) pour éviter toute
  confusion avec le target CMake.
- Si `Config.cpp` ou `Log.cpp` utilisent eux-mêmes des symboles présents uniquement dans
  `engine_core`, reporter le problème dans le rapport final sans bloquer le ticket.
