
**Objectif**
Système amis avec ajout/suppression, online status, whisper rapide.

**Dépendances**
- M13.1 (server)
- M16.1 (UI)

**Livrables**
- Friends DB table (playerId, friendId, timestamp)
- Friend requests (pending, accepted, declined)
- Online status tracking
- Friends list UI panel

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- Requête: /friend add <name> -> send request
- Accept/decline: /friend accept <name>
- Bilateral relationship (both must accept)
- Online status: online, away, busy, offline
- Whisper shortcut: double-click name -> open chat

**Étapes d'implémentation**
1. DB table: friends (player_id, friend_id, status, created_at)
2. FriendRequest message (requester, target)
3. Server: validate players exist, send request notification
4. Target: accept/decline -> update DB
5. On login: query friends, send online status to friends
6. Track online friends (presence system)
7. UI: list friends, status indicator, whisper button

**Critères de validation (DoD)**
- [ ] Envoyer friend request
- [ ] Accepter request ajoute à liste
- [ ] Online status vert, offline gris
- [ ] Whisper rapide depuis liste

**Notes / pièges à éviter**
- Limit friends (100-200 max)
- Block spam requests (rate limit)
