# M19.9 — Transport hardening v1: limits, throttling, timeouts

**Status:** Closed

## Objectif
Ajouter protections transport-level contre abus (flood, invalid packets, memory blow) avant l'auth (M20).

## Dépendances
- M19.3
- M19.4
- M19.5
- M19.2
- M00.1

## Livrables
- Token bucket packets/sec par connexion
- Compteur decode failures + disconnect policy
- Cap TX queued bytes/connection + disconnect policy
- Handshake timeout (TLS)
- Disconnect reasons standardisées + stats

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Packet rate limit ex: 200 pkt/s burst 400 (configurable).
- Decode failure threshold ex: 5 → disconnect.
- TX queue cap ex: 1–4 MB/conn (configurable).
- Handshake timeout ex: 10s.

## Étapes d'implémentation
1. Ajouter config limits (ini/json selon ton système).
2. Implémenter token bucket par connexion.
3. Compter invalid headers/size; appliquer seuil.
4. Capper TX queue; disconnect si dépassement.
5. Ajouter handshake deadline.

## Critères de validation (DoD)
- [ ] Flood sur une connexion n'impacte pas les autres.
- [ ] Paquets invalides répétés → connexion coupée.
- [ ] Logs indiquent reason + counters.

## Notes / pièges à éviter
- Minimiser contention (stats agrégées).
