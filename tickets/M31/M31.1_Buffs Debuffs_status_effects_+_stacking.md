# M31.1_Buffs Debuffs_status_effects_+_stacking

## Objectif
Système status effects (buffs, debuffs, DoT, HoT) avec durée, stacks, dispel.

## Dépendances
- M28.1
- M14.1

 ## Livrables
- `engine/gameplay/StatusEffect.*`
- Effect types: buff, debuff, DoT, HoT, stun, slow, root
- Stacking rules (stack count, refresh duration)
- Dispel mechanics (magic, curse, poison)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- StatusEffect: id, duration, tickInterval, stacks, maxStacks
- Types: stat modifier (+50 strength), DoT (10 dmg/sec), CC (stun, root)
- Stacking: stack count (1-N), stack type (refresh, extend, independent)
- Dispel types: magic, poison, curse, disease

## Étapes d'implémentation
1. Définir StatusEffect struct (effectId, caster, target, startTime, duration, stacks)
2. Track active effects per entity (list/map)
3. Update: tick effects (apply periodic damage/heal), decrement duration
4. Apply effect: check stacking rules (add stack or refresh)
5. Remove: on expire ou dispel
6. Stat modifiers: recalculate final stats when effects change
7. CC effects: block movement/actions (flag isStunned, isRooted)

## Exemple effect JSON
```json
{
  "id": "poison_dot",
  "name": "Poison",
  "duration": 12.0,
  "tickInterval": 2.0,
  "maxStacks": 5,
  "stackType": "refresh",
  "dispelType": "poison",
  "effects": [
    {"type": "damage", "amount": 10, "damageType": "nature"}
  ]
}
```
