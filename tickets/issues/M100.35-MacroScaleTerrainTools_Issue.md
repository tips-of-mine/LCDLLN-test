## Résumé

Deux livrables couplés dans un **seul ticket / une seule PR / une seule application** (`lcdlln_world_editor.exe`) :

1. **Refactor UI : barre d'outils à boutons-icônes.** Remplace la `BeginTabBar("OutilsTabs")` actuelle de `WorldEditorImGui` par une **barre horizontale fixe** placée en haut du viewport 3D, où **chaque outil disponible est représenté par un bouton-icône cliquable**. ### B. Workflow MountainRangeTool

1. Cliquer le bouton « Mountain Range » dans la toolbar → `m_activeTool = MountainRange`. Curseur ImGui change en croix-de-mire au-dessus du viewport.
2. Clic 1..N sur le terrain → vertices posés. Polyline visualisée en jaune entre les contrôles, vertices marqués par disques jaunes 6 px.
3. Double-clic ou clic sur le premier vertex → ferme la polyline (mode Loop). Sinon, bouton **Enter** ou clic sur le bouton **Apply** dans `ToolPropertiesPanel` → fige la polyline en mode Open.
4. Une fois figée, preview ambre semi-transparente apparaît au-dessus du terrain. Tweak des paramètres (sliders dans `ToolPropertiesPanel`) → preview recalculée (debounce 50 ms).
5. Drag d'un vertex (clic et maintien sur un disque jaune) → repositionne, preview recalculée.

7. **Cancel** dans `ToolPropertiesPanel` → abandon de la polyline en cours.
8. Cliquer **un autre bouton** de la toolbar → confirme implicitement la perte de la polyline en cours (avec confirmation modale si elle a > 1 vertex).
9. Cliquer le **X** (Désélectionner) → `SetActiveTool(None)`.

### E. Algorithme cœur — identique à v1 du ticket

(Inchangé par rapport à la première version, repris ici pour complétude.)

Pour chaque cellule de heightmap dans le bounding box élargi (max `widthMeters` × 1.5) :

1. Pour chaque segment, calculer `(distLat, t)` via `ClosestPointOnSegment`. Garder le segment min `distLat`.
2. Interpoler les paramètres `widthMeters / heightMeters / noiseAmplitude / asymmetry` à `t` (lerp entre les deux vertices du segment retenu).
3. Skip si `distLat > widthMeters_local`.
4. Profil radial : `u = distLat / widthMeters_local`, `w = profile(u) * heightMeters_local`.
5. Asymétrie : `w *= 1 + asymmetry * sign(cross(segDir2D, perpDir2D))`.
6. Bruit : `offset = SimplexNoise2D(xWorld * noiseFrequency, zWorld * noiseFrequency, seed) * noiseAmplitude_local`.
7. Delta : `delta = +(w + offset)` (Mountain) ou `delta = -(w + offset)` (Valley).
8. Push `(chunkCoord, cellIndex, delta)` dans `SparseChunkDeltas`.

Profils :
- `smoothstep(u)` = `smoothstep(1.0, 0.0, clamp(u, 0, 1))`
- `linear(u)` = `clamp(1 - u, 0, 1)`
- `exp(u)` = `exp(-3 * u * u)`

### Structures (extrait `PolylineMacroCore.h`)

```cpp
#pragma once

#include "src/client/world/WorldModel.h"   // engine::world::GlobalChunkCoord
#include "src/shared/math/Math.h"          // Vec2, Vec3

#include <cstdint>
#include <unordered_map>
#include <vector>

namespace engine::editor::world
{
    enum class PolylineMode : uint8_t { Open, Loop };
    enum class FlankProfile : uint8_t { Smoothstep, Linear, Exp };

    /// Un vertex de polyline avec paramètres locaux. Interpolés linéairement
    /// le long du segment vers le vertex suivant.
    /// Effet de bord : aucun.
    struct PolylineVertex
    {
        float worldX           = 0.0f;
        float worldZ           = 0.0f;
        float widthMeters      = 250.0f;
        float heightMeters     = 400.0f;
        float noiseAmplitude   =  30.0f;
        float asymmetry        =   0.0f;   // -1..+1
    };

    struct MacroPolylineParams
    {
        std::vector<PolylineVertex> vertices;
        PolylineMode mode            = PolylineMode::Open;
        FlankProfile profile         = FlankProfile::Smoothstep;
        uint32_t     noiseSeed       = 0;
        float        noiseFrequency  = 0.005f;
    };

    /// (chunkCoord) → (cellIndex → deltaMeters).
    /// cellIndex = z * 257 + x (M100.5 layout 257×257).
    using SparseChunkDeltas =
        std::unordered_map<engine::world::GlobalChunkCoord,
                           std::unordered_map<uint32_t, float>>;

    /// Rasterise les deltas pour la polyline donnée.
    /// \param params  Paramètres de la polyline.
    /// \param invert  false = Mountain (delta positif), true = Valley (delta négatif).
    /// \return        Deltas sparse par chunk. Vide si `vertices.size() < 2`.
    ///
    /// Effet de bord : aucun. Pure function. Thread-safe.
    SparseChunkDeltas RasterizeMacroPolyline(const MacroPolylineParams& params,
                                             bool invert);
}
```

### Couture inter-chunk

La rasterisation calcule les deltas en **espace monde global** (`xWorld`, `zWorld` en mètres). Pour deux chunks adjacents, la colonne de bord partagée reçoit les mêmes deltas. Test `Test_ChunkBorderParity_TwoChunksSameValueOnFrontiers` vérifie ça à 257 cellules de précision.

### Performance attendue

Inchangée vs v1 : ~220 ms single-thread pour polyline de 5 km × 250 m. Si > 500 ms, paralléliser par segment via `engine::core::jobs::JobSystem`.

## Schéma d'interaction utilisateur

```
[click bouton « Mountain Range » dans la toolbar]
  → m_activeTool = MountainRange
  → curseur viewport : croix
  → ToolPropertiesPanel affiche le bloc « Macro polyline »

[click_1 sur le terrain]   → vertex #0 posé, disque jaune
[click_2 sur le terrain]   → vertex #1, polyline jaune entre #0 et #1
...
[double-clic ou Apply]     → polyline figée, preview ambre apparaît
[tweak slider]             → preview recalculée (debounce 50 ms)
[drag vertex disque jaune] → repositionne, preview recalculée

[Apply dans ToolPropertiesPanel]
  → MountainRangeCommand pushée sur CommandStack
  → chunks touchés → dirty → LOD M100.8 enqueue
  → polyline + preview supprimées, outil reste actif

[Ctrl+Z]                   → undo de la dernière commande
[click bouton « Valley Chain »] → bascule, polyline en cours abandonnée
                                  (avec confirmation modale si vertices > 1)
[click bouton X (Désélectionner)] → m_activeTool = None
```

### Outils macros

- [ ] Une polyline de 3 vertices génère une crête continue entre les vertices avec largeur correcte mesurée perpendiculairement (tolérance ±2 %).
- [ ] Mode `Loop` connecte le dernier vertex au premier sans discontinuité.
- [ ] La rasterisation sur 2 chunks adjacents produit des valeurs strictement identiques sur les 257 cellules de frontière partagée.
- [ ] `asymmetry = +1.0` produit un flanc abrupt d'un côté, doux de l'autre.
- [ ] Le bruit simplex est déterministe (même seed = mêmes deltas, parité octet-à-octet).
- [ ] Undo d'une `MountainRangeCommand` restaure toutes les heightmaps touchées octet-à-octet.
- [ ] Après commit, les chunks touchés sont `dirty` et `EnqueueLodRegenForDirtyChunks` a été appelé.
- [ ] La preview ne modifie pas `TerrainChunk` tant qu'`Apply` n'a pas été cliqué.
- [ ] `Cancel` ou changer d'outil avec confirmation supprime la preview sans pousser de commande.
- [ ] Polyline avec moins de 2 vertices : silencieusement ignorée (pas de commande, pas de log error).
