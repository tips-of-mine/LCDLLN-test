# M14.1 — Combat serveur: validation + damage + events

## Objectif
Implémenter combat minimal authoritative : AttackRequest, validation, dégâts, CombatEvent.

## Dépendances
- M13.3

## Livrables
- AttackRequest msg
- Server combat system
- CombatEvent broadcast


## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- MVP hit=always; range check; cooldown simple.

## Étapes d’implémentation
1. Ajouter composants Stats/Combat aux entités.
2. Handler AttackRequest: vérifier cible + range + cooldown.
3. Appliquer damage, HP<=0 -> death state.
4. Broadcast CombatEvent aux clients intéressés.

## Critères de validation (DoD)
- [ ] Un joueur peut attaquer un mob et voir HP descendre côté client.

## Notes / pièges à éviter
- Pas de prédiction nécessaire au MVP.
