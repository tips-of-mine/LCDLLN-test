
**Objectif**
Outils UI avancés pour raiders: DPS meter, threat meter, combat log détaillé.

**Dépendances**
- M16.2 (combat UI)
- M32.2 (party)

**Livrables**
- DPS meter: track damage per player, update realtime
- Threat meter: show aggro % per player vs target
- Combat log: scrollable text log (damage, heals, buffs, etc.)
- Export logs (CSV optional)

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- DPS: sum damage per player / fight duration
- Threat: server sends threat values, display bars
- Combat log: events stream (damage, heal, buff, death)

**Étapes d'implémentation**
1. DPS meter:
   - Track CombatEvent (damage) per player
   - Calculate DPS: total damage / elapsed time
   - UI: bar chart (top 5 players)
   - Reset on combat end
2. Threat meter:
   - Server: broadcast ThreatUpdate(targetId, playerId, threat)
   - Client: display bars (100% = highest threat)
   - Color code: red=pulling aggro, yellow=close, green=safe
3. Combat log:
   - Subscribe to all combat events
   - Format: "[12:34:56] Player hits Target for 150 (Fire)"
   - Scrollable window (ring buffer 500 lines)
   - Filters: damage, healing, buffs, etc.
4. Export: save log to CSV file

**Critères de validation (DoD)**
- [ ] DPS meter affiche damage ranks
- [ ] Threat bars update realtime
- [ ] Combat log scroll fonctionne
- [ ] Export CSV génère fichier

**Notes / pièges à éviter**
- Performance: aggregate events (don't spam UI)
- Privacy: option hide DPS from others
