# M36.2_Crafting_recipes_+_profession_skills_+_UI

## Objectif
Système crafting avec recettes, professions (blacksmithing, alchemy, etc.), skill leveling.

Dépendances
- M36.1
- M16.3

## Livrables
- Profession system (learn profession, skill 1-300)
- Recipes data-driven (requirements, output, skill gain)
- Crafting UI (recipe list, materials, craft button)
- Skill-up on craft (diminishing returns)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Professions: primary (2 max), secondary (unlimited)
- Skill levels: 1-300 (Apprentice, Journeyman, Expert, Artisan, Master)
- Recipes: ingredients[], tool, skill_required, skill_up_chance
- Crafting time: 1-5s, quality tiers (optional)

## Étapes d'implémentation
1. DB: player_professions (player_id, profession_id, skill_level)
2. Learn profession: assign to player (max 2 primary)
3. Recipes: JSON (id, profession, skill_req, ingredients, output, skill_up)
4. UI: list known recipes, filter by skill
5. Craft: validate ingredients in inventory, skill level
6. Craft process: cast bar, consume ingredients, create item
7. Skill-up: roll chance (100% if skill < recipe level, 0% if far above)
8. Skill breakpoints: unlock new recipes (skill 50, 100, 150, etc.)

## Exemple recipe JSON
```json
{
  "id": "iron_sword",
  "profession": "blacksmithing",
  "skill_required": 50,
  "ingredients": [
    {"item": "iron_bar", "quantity": 8},
    {"item": "leather_strap", "quantity": 2}
  ],
  "tool": "blacksmith_hammer",
  "output": {"item": "iron_sword", "quantity": 1},
  "craft_time": 3.0,
  "skill_up_chance": 1.0
}
```
