# M16.5 — Theme/skin switch: theme.json + style.qss integration

## Objectif
Appliquer thèmes UI par race via chargement theme.json + style.qss, hot-reload optionnel.

## Dépendances
- M16.1

## Livrables
- Theme manager
- Apply QSS
- Asset refs (fonts/icons/colors)


## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Widgets utilisent classes logiques (hud/panel/button/tooltip).

## Étapes d’implémentation
1. Définir mapping theme.json -> variables (colors/fonts).
2. Charger et appliquer style.qss.
3. Hook changement race/thème.
4. Option: hot reload fichier.

## Critères de validation (DoD)
- [ ] Changer thème modifie UI sans casser la logique.

## Notes / pièges à éviter
- Assurer fallback si fichier manquant.
