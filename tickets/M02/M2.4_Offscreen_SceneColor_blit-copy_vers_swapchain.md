# M2.4 — Offscreen SceneColor + blit/copy vers swapchain

## Objectif
Rendre vers une image offscreen (HDR/ LDR) puis copier vers swapchain pour préparer postprocess.

## Dépendances
- M2.3
- M1.3

## Livrables
- FG ressource `SceneColor` (offscreen)
- Pass Present: copy/blit vers swapchain
- Resize gère SceneColor


## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Swapchain reste la dernière étape.
- SceneColor format configurable (HDR plus tard).

## Étapes d’implémentation
1. Créer SceneColor suivant extent swapchain.
2. Pass Clear/Draw écrit SceneColor.
3. Pass Present: transition SceneColor->TRANSFER_SRC, swapchain->TRANSFER_DST, copy, swapchain->PRESENT.
4. OnResize: rebuild graph resources.

## Critères de validation (DoD)
- [ ] Écran identique mais via offscreen.
- [ ] Resize reconstruit SceneColor sans crash.

## Notes / pièges à éviter
- Blit nécessite formats compatibles; sinon utiliser copy + full-screen pass plus tard.
