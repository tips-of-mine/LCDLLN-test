# M25.3 — Network buffer pooling + alloc optimization

**Status:** Closed

## Objectif
Réduire allocations/frees dans hotpath réseau (RX/TX) via pooling de buffers.

## Dépendances
- M00.2 (memory arenas) optionnel
- M19.4

## Livrables
- Pool de buffers (fixed sizes) pour RX/TX
- Reuse sur disconnect
- Metrics: alloc saved

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Pool thread-local ou sharded pour limiter locks.
- Taille buffers: 4KB/16KB (aligné max packet).

## Étapes d’implémentation
1. Définir buffer classes.
2. Implémenter pool.
3. Instrumenter gains.

## Critères de validation (DoD)
- [ ] CPU time et alloc count baissent sous charge.
