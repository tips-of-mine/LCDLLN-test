# M25.1_Animation_skeletal_loading_+_playback_engine

## Objectif
Charger animations skelettiques depuis glTF et implémenter playback runtime avec interpolation de poses.

## Dépendances
- M03.3
- M00.3

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Code jeu** : uniquement sous `/game/...`
- **Contenu** (textures/meshes/json/audio/shaders) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- **Interdit** : créer des dossiers racine du type `/assets`, `/textures`, `/data`, `/shaders`, etc.
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json). Pas de chemins absolus.

## Livrables
- `engine/animation/Skeleton.*` (joints hierarchy)
- `engine/animation/AnimationClip.*` (keyframes, channels)
- `engine/animation/AnimationPlayer.*` (playback, sampling)
- Loader glTF animations

## Spécification technique
- Support rotations (quaternions), translations, scales
- Interpolation linéaire (LERP/SLERP)
- Sampling à 30-60 Hz
- Time normalisé [0..duration]

## Étapes d'implémentation
1. Définir structures Skeleton (joints, parenté, bind pose)
2. Parser glTF animations (accessors, samplers, channels)
3. Implémenter AnimationPlayer: update(dt), sample pose
4. Interpoler keyframes (linear pour MVP)
5. Output pose = array transforms local-space

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD
