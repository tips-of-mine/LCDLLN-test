# M100.39 — Thermal & Wind Erosion

## Résumé

Deux mécanismes d'érosion complémentaires de l'hydraulique (M100.38), regroupés dans **un seul outil et un seul ticket** parce que leurs algorithmes sont simples et symétriques (relaxation locale du terrain par règle physique) et qu'ils s'utilisent souvent ensemble pour finaliser un terrain naturaliste :

1. **Thermal erosion** (érosion thermique / talus) — quand une pente entre deux cellules adjacentes dépasse l'**angle de repos** (~ 30-45°), du matériau « s'effondre » de la cellule haute vers la cellule basse. Itéré sur N passes jusqu'à convergence ou nombre de passes fixé. Adoucit les crêtes anguleuses, crée des pentes naturelles à profil constant, simule éboulements et alluvions de pied de versant.
2. **Wind erosion** (érosion éolienne) — un vent dominant transporte du sable depuis les **faces exposées** vers les **faces sous le vent**. Implémenté en particle-system (similaire à M100.38) mais avec direction de transport fixe + critère d'exposition.

Les deux modes partagent l'outil `ThermalWindErosionTool` (un seul bouton toolbar avec radio Thermal / Wind / Both), réutilisent le pattern `SparseChunkDeltas`, et produisent une seule `ICommand` réversible.

L'érosion thermique est **rapide** (~500 ms pour une zone complète). L'érosion éolienne est **moyenne** (~5-15 s, comme l'hydraulique).

**Aucun nouvel exécutable n'est créé.** Code dans `engine_core`, binaire éditeur unique `world_editor_app` (`lcdlln_world_editor.exe`).

- `ConsolidatedHeightGrid` (lecture-seule consolidée) déjà extrait par M100.38 si M100.36 ne l'avait pas fait. M100.39 le consomme tel quel.
- `BilinearGradientSample` (helpers gradient + distribute bilinéaire) → réutilisé pour le mode Wind. Le mode Thermal n'en a pas besoin (opère cellule à cellule).
- Pattern modale de progression ImGui → factoré dans `src/world_editor/ui/SimulationProgressModal.h/.cpp` (helper réutilisable, créé dans ce ticket si pas déjà fait par un précédent).
- `SparseChunkDeltas` (M100.35) → réutilisé pour les deux modes.

#### Partie C — Algorithme Thermal Erosion

Pour chaque passe (max `numPasses`, early-exit si convergence) :

```
deltasThisPass = SparseChunkDeltas vide
totalTransferred = 0

Pour chaque cellule (x, z) du grid consolidé :
  h_self = heights[x, z]
  totalExcess = 0
  excessPerNeighbor = []

  Pour chaque (dx, dz) dans les 8 voisins :
    h_n = heights[x+dx, z+dz]
    dist = sqrt(dx² + dz²) * cellSizeMeters
    slope = (h_self - h_n) / dist
    Si slope > talusSlope (= tan(talusAngleDeg)) ET slope > minActivationSlope :
      excess = (slope - talusSlope) * dist * forcePerPass * 0.5
      excessPerNeighbor.push((x+dx, z+dz, excess))
      totalExcess += excess

  Si excessPerNeighbor non vide :
    factor = min(1.0, h_self / totalExcess)   // anti-runaway
    Pour chaque (nx, nz, excess) dans excessPerNeighbor :
      delta = excess * factor
      deltasThisPass.add((x, z), -delta)
      deltasThisPass.add((nx, nz), +delta)
      totalTransferred += delta

// Appliquer cette passe au grid local (lecture-seule extérieure)
ApplyDeltasToLocalCopy(heights, deltasThisPass)
deltasAccumulated.merge(deltasThisPass)

Si totalTransferred < convergenceThreshold :
  break  // convergence atteinte
```

**Important** : le grid `heights` utilisé en lecture par les passes successives est une **copie locale** mise à jour passe par passe, pas le `TerrainChunk` du document. Seul le résultat final est committé via `ICommand`.

Convergence : seuil `convergenceThreshold = numCells * 1e-6 m` (sur une zone 26M cells = 26 m de transfert total, arbitraire mais raisonnable).

#### Partie D — Algorithme Wind Erosion

Idée : particle system analogue à M100.38 mais avec **direction de transport quasi-fixe** (vent dominant) au lieu du gradient gravitationnel.

```
windDir = normalize(vec2(cos(windAngleRad), sin(windAngleRad)))

Pour chaque particule (parallélisable) :
  pos = position initiale (uniform random dans la zone)
  sand = 0.0
  steps = 0

  Tant que steps < maxLifetime :
    h_pos = BilinearSampleHeights(pos)
    Si h_pos < seaLevelMeters ET stopUnderSeaLevel : break

    // Calcule l'exposition au vent en arrière de la position courante
    upwindPos = pos - windDir * exposureRadiusMeters
    h_upwind = BilinearSampleHeights(upwindPos)

    exposure = h_pos - h_upwind   // positif si la position est plus haute que l'amont

    capacity = max(0, exposure) * windStrength * sandCapacityFactor

    Si sand > capacity :
      // Déposition : on relâche du sable (face sous le vent / abritée)
      amount_deposit = (sand - capacity) * depositionRate
      DistributeBilinearDelta(pos, +amount_deposit)
      sand -= amount_deposit
    Sinon :
      // Érosion : on emporte du sable (face exposée)
      amount_erode = min((capacity - sand) * erosionRate, h_pos * 0.05)
      Si restrictToSandSplat ET splatSandValue(pos) < 0.3 : amount_erode = 0
      DistributeBilinearDelta(pos, -amount_erode)
      sand += amount_erode

    // Avance dans la direction du vent (avec petit jitter pour ne pas tracer ligne)
    pos += windDir * cellSize + jitter
    steps += 1
```

Variante optionnelle : `restrictToSandSplat = true` → l'érosion ne s'applique qu'aux cellules dont la splat layer « Sand » a un poids ≥ 0.3 (sable existant). Évite que le vent érode de la roche.

Multi-chunk : identique à M100.38 (DistributeBilinearDelta gère la couture).

#### Partie E — Preview interactive

**Thermal** : un overlay 2D affiche en orange épais les **arêtes du grid** (entre cellules adjacentes) dont la pente initiale dépasse `talusAngleDeg`. Recalculé à chaque modification d'angle (debounce 200 ms). Avant simulate, c'est juste un indicateur visuel ; après simulate, l'overlay rouge/vert (érosion/déposition) recouvre comme pour M100.38.

**Wind** : un overlay 2D affiche :
- Une **flèche bleue** indiquant la direction du vent dans un coin du viewport.
- Une grille sparse de **petites flèches** ('quivers') montrant le vent sur le terrain (densité 1 / 100 m).
- Après simulate : cellules teintées **sable doré** pour la déposition, **gris-rocher** pour l'érosion.

Toggle indépendants : « Afficher arêtes au-delà du talus », « Afficher quivers vent », « Afficher delta post-sim ».

#### Partie F — Multi-chunk

Identique à M100.36/38 : `SparseChunkDeltas` en espace monde global, couture inter-chunks préservée par construction. Tests dédiés.

#### Partie G — Parallélisation

- **Thermal** : chaque passe est intrinsèquement séquentielle (les cellules en aval lisent l'état post-modif des cellules en amont), mais **on peut paralléliser par bandes** de la zone (chaque worker traite une bande de chunks, avec double-buffering grid local). Seuil par défaut : grid ≥ 4M cells (~zone complète).
- **Wind** : particules indépendantes, même parallélisation que M100.38 (réduction par thread). Seuil par défaut : `numParticles ≥ 20000`.
- Config : `editor.world.thermal_erosion.parallel_threshold` et `editor.world.wind_erosion.parallel_threshold`.

#### Partie H — Commit unique

Mode **Both** : Thermal est appliqué en premier dans une simulation interne, son résultat sert d'**input** au Wind (sur la même copie locale du grid), et les deltas cumulés sont commités en **une seule** `ThermalWindErosionCommand`.

Mode **Thermal seul** ou **Wind seul** : commit simple sur la commande, l'autre delta est vide.

Undo restaure octet-à-octet pour tous les modes.

#### Partie I — Tests

Suite `thermal_wind_erosion_tests` couvrant convergence thermal, conservation de masse (mass-conservation = somme deltas = 0 globalement), wind direction strictement respectée, exposition correctement calculée, restrictToSandSplat respecté, déterminisme, parallélisation byte-equal, undo complet.

## Spécification fonctionnelle

### Workflow Thermal seul

1. Cliquer **Thermal/Wind** dans la toolbar.
2. Sélectionner radio « Thermal seul ».
3. Ajuster `talusAngleDeg` (35° = pierre/gravier typique, 25° = sable, 45° = rocher dur).
4. Cliquer **Simulate** → modale rapide (~500 ms).
5. Preview rouge/vert apparaît.
6. **Apply** → commit, chunks dirty, LOD regen.

### Workflow Wind seul

1. Sélectionner radio « Wind seul ».
2. Configurer direction du vent (combo 8 directions cardinales ou angle libre).
3. Cliquer **Simulate** → modale ~5-15 s.
4. Preview avec teinte sable + arrows.
5. **Apply**.

### Workflow Both

1. Sélectionner radio « Both ».
2. Configurer paramètres des deux sous-modes.
3. Cliquer **Simulate** → modale en 2 phases (thermal puis wind, indicateur de phase courante).
4. Preview cumulée.
5. **Apply** → une seule commande.

### Heuristiques recommandées (tooltips dans l'UI)

| Paramètre | Défaut | Sens physique |
|---|---|---|
| `talusAngleDeg` | 35° | Pierraille / éboulis typique. Sable = 30-32°, roche dure = 40-45°. |
| `forcePerPass` | 0.3 | 1.0 = transfert immédiat de tout le surplus en une passe (instable visuellement). 0.3 = 3-4 passes pour atteindre 99 % du transfert final. |
| `numPasses` | 40 | Avec `force=0.3`, convergence typique en 25-30 passes. |
| `windAngleDeg` | 180° (N→S) | Vent dominant typique en climat tempéré du nord. |
| `windStrength` | 0.5 | Vent modéré. > 0.8 = tempête, < 0.2 = brise. |
| `numParticles` | 30 000 | Suffisant pour zone 10 km. 100k+ pour qualité finale. |
| `exposureRadiusMeters` | 30 m | Distance de recherche d'obstacle upwind. |

```
  → modale "Wind erosion (phase 2/2)... 18 000 / 30 000 particules"
  → modale ferme, preview rouge/vert + sable apparaît

```

## Critères d'acceptation

### Thermal

- [ ] Sur une heightmap plate, thermal ne produit aucun delta (`deltas.empty()`).
- [ ] Sur un cône vertical (angle >> talus) initial, après 40 passes : la pente max du résultat est ≤ `talusAngleDeg + 2°` (tolérance d'approximation discrète).
- [ ] **Conservation de masse** : `Σ(deltas) ≈ 0` (somme algébrique de tous les deltas thermal = 0 ± `numCells * 1e-7 m`). Le matériau qui s'effondre d'une cellule arrive intégralement chez le voisin.
- [ ] Convergence détectée si `totalTransferred < threshold` ; le `passesExecuted` peut être < `numPasses` dans ce cas.
- [ ] `preserveSteepSlopes = true` avec seuil 60° : aucune pente initiale > 60° n'est touchée (ce qui empêche d'aplanir des falaises voulues).

### Wind

- [ ] Sur une heightmap plate, wind produit deltas négligeables (< `numParticles * 1e-4 m` total).
- [ ] Sur une dune isolée, après simulation : la **face exposée au vent** perd du matériau (deltas négatifs), la **face sous le vent** gagne (deltas positifs).
- [ ] Le vent suit strictement `windAngleDeg` : la trajectoire moyenne des particules est colinéaire à `windDir` à `±10°` près.
- [ ] `restrictToSandSplat = true` : cellules à splat sand < 0.3 ne sont pas érodées (delta négatif = 0 strict), mais peuvent toujours recevoir de la déposition.
- [ ] **Déterminisme** : même `rngSeed` + mêmes params → résultats byte-equal.

### Both

- [ ] En mode Both, thermal est appliqué en premier sur une copie locale du grid ; wind utilise ce grid post-thermal comme input.
- [ ] Une seule `ThermalWindErosionCommand` est poussée, contenant les deltas cumulés.
- [ ] Undo restaure octet-à-octet vs pré-Apply.

### Multi-chunk

- [ ] Thermal et wind : deltas aux frontières inter-chunks identiques bit-à-bit.

### Parallélisation

- [ ] Mode parallèle thermal : résultat byte-equal vs single-thread (avec même split de bandes).
- [ ] Mode parallèle wind : résultat byte-equal vs single-thread (même seed).
- [ ] Configs `editor.world.thermal_erosion.parallel_threshold` et `editor.world.wind_erosion.parallel_threshold` fonctionnent comme attendu.

### Preview & UI

- [ ] L'overlay orange "arêtes > talus" est correctement aligné avec la heightmap (chaque arête de 2 cellules adjacentes au-delà de l'angle).
- [ ] La flèche du vent dans le viewport pivote en temps réel quand `windAngleDeg` change.
- [ ] Les quivers du vent sont espacés à ~100 m, non couvrants du viewport (densité raisonnable).

### Commit & Undo

- [ ] Apply en mode Thermal seul ne touche pas `windDeltas` (vide).
- [ ] Apply en mode Wind seul ne touche pas `thermalDeltas` (vide).
- [ ] Apply en mode Both touche les deux.
- [ ] LOD regen enqueué pour tous les chunks touchés (union thermal + wind).

### Invariant terrain visible

- [ ] Activation outil + simulate ne mute pas `Camera::position`, `TerrainRenderer::IsFrustumCullEnabled()`, `m_noUserTextures`.
- [ ] Preview overlay est rendue via `ImGui::GetForegroundDrawList()`, pas de render pass propriétaire.
- [ ] Outil utilisable quand terrain en fallback orange (test : forcer `m_noUserTextures = 1`).
