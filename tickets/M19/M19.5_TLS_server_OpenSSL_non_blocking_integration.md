# M19.5 — TLS server (OpenSSL) non-blocking integration

## Objectif
Ajouter TLS obligatoire côté serveur via OpenSSL, intégré à epoll/non-blocking, avec logs d’erreurs exploitables.

## Dépendances
- M19.4
- M00.1

## Livrables
- Initialisation OpenSSL + `SSL_CTX` serveur
- Chargement cert/key via config
- Handshake TLS non-bloquant par connexion
- Read/Write via SSL (SSL_read/SSL_write) intégrés au loop
- Politique TLS min (TLS 1.2+)

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Par connexion:
  - `SSL* ssl`
  - état handshake: in_progress/ready/failed
- Non-blocking:
  - gérer `SSL_ERROR_WANT_READ/WRITE` via intérêts EPOLLIN/OUT
- TLS obligatoire: pas de mode plain sur le port TLS

## Étapes d’implémentation
1. Init OpenSSL, créer `SSL_CTX`.
2. Charger cert/key (config).
3. À accept: créer `SSL*`, set fd, démarrer handshake.
4. Dans epoll loop: progresser handshake selon WANT_READ/WRITE.
5. Une fois ready: read/write via SSL.
6. Shutdown: SSL_shutdown + free + close fd.

## Critères de validation (DoD)
- [ ] Connexion TLS client↔serveur OK.
- [ ] Erreurs handshake loggées clairement.
- [ ] Le serveur reste stable sous handshakes ratés.

## Notes / pièges à éviter
- Ne pas bloquer sur handshake.
- Bien gérer close_notify / HUP.
