# M04.3 — Lighting: shadow sampling PCF + bias

**Status:** Closed

## Ticket (original)

### Objectif
Intégrer shadow maps dans lighting pass avec sélection cascade et PCF 3x3.

### Dépendances
- M04.2
- M03.2

### Livrables
- Shader: cascade selection + project + sample compare
- Paramètres bias configurables

### Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

### Spécification technique
- Cascade selection via depth view-space vs split distances.
- PCF 3x3 (9 taps) en UV space, sampler compare.

### Étapes d'implémentation
1. Calculer cascade index.
2. Projeter posWS en light clip, UV + depth.
3. Appliquer bias (const + slope).
4. Faire PCF 3x3 et multiplier light direct.

### Critères de validation (DoD)
- [ ] Ombres visibles et cohérentes, acne limité.
- [ ] Réglage bias via config/cvar.

### Notes / pièges à éviter
- Traiter UV hors [0,1] -> shadow=1 (no occlusion).
