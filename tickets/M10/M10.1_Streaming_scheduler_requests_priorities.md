# M10.1 — Streaming scheduler: requests + priorities

## Objectif
Créer scheduler de streaming avec priorités (ring, distance, direction).

## Dépendances
- M00.3
- M09.1

## Livrables
- Queues request/io/cpu/gpuUpload
- Priority function

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Active > Visible > Far; devant joueur > derrière.

## Étapes d’implémentation
1. Définir `ChunkRequest` (chunkId, targetState, priority).
2. Implémenter priority score.
3. Mettre une file priority (heap) ou buckets.
4. Producer: World update push requests.

## Critères de validation (DoD)
- [ ] En avançant, chunks devant se chargent avant ceux derrière.

## Notes / pièges à éviter
- Éviter starvation: aging/boost après X sec.
