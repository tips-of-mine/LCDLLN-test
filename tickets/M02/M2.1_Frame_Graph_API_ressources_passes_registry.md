# M2.1 — Frame Graph: API ressources + passes + registry

## Objectif
Mettre en place un Frame Graph minimal : création images/buffers logiques, passes avec read/write, registry pour handles Vulkan.

## Dépendances
- M1.4
- M0.2

## Livrables
- `FrameGraph.*` (addPass, createImage/createBuffer)
- `PassBuilder` (declare reads/writes)
- `Registry` (resolve FG id -> VkImage/VkBuffer/View)


## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Ressources transientes par défaut (lifetime frame).
- Pass = setup + execute(cmd, registry).
- Nommage des ressources/passes pour debug.

## Étapes d’implémentation
1. Définir IDs (uint32) et descriptors (ImageDesc/BufferDesc).
2. Implémenter PassBuilder read/write.
3. Implémenter Registry simple (map id->resource).
4. Exécuter passes dans l'ordre d'ajout (temporaire).

## Critères de validation (DoD)
- [ ] Créer une image SceneColor + une pass clear + une pass copy/present.
- [ ] Registry retourne des handles valides pendant la frame.

## Notes / pièges à éviter
- Commencer simple: pas d'aliasing mémoire au tout début.
