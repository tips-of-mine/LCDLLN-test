# M05.1 — IBL: BRDF LUT (compute) generation

## Objectif
Générer BRDF LUT 2D (R16G16) pour split-sum specular GGX.

## Dépendances
- M03.2

## Livrables
- Texture BRDF_LUT (256x256 RG16F)
- Compute shader intégration GGX

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Génération au boot (runtime) acceptable; offline plus tard.

## Étapes d’implémentation
1. Créer image BRDF_LUT + view + sampler.
2. Compute dispatch 256x256, écrire RG.
3. Barrières: GENERAL->SHADER_READ_ONLY.
4. Smoke test: LUT non nulle.

## Critères de validation (DoD)
- [ ] BRDF LUT générée une fois, réutilisée dans lighting.

## Notes / pièges à éviter
- Limiter samples (ex 1024) pour temps raisonnable; cache disque plus tard.
