# M23.2 — Prometheus /metrics: core counters

**Status:** Closed

## Objectif
Exposer `/metrics` compatible Prometheus avec compteurs essentiels réseau, sessions, shards, DB latence.

## Dépendances
- M19.11 (counters)
- M20 (sessions)
- M22 (shards)

## Livrables
- `/metrics` (text format)
- Metrics:
  - connections_active
  - handshake_fail_total
  - bytes_in_total / bytes_out_total
  - auth_success_total / auth_fail_total
  - sessions_active
  - shards_online
  - db_query_latency_ms (histogram simple)

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Agrégation thread-safe.
- Ne pas exposer données sensibles.

## Étapes d'implémentation
1. Implémenter exporter prometheus.
2. Connecter aux compteurs existants.
3. Vérifier format via Prometheus local.

## Critères de validation (DoD)
- [x] Prometheus scrape OK.
- [x] Valeurs cohérentes sous charge.
