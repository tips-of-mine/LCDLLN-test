# M00.5 — Game Loop: update/render, double-buffer render state

**Status:** Open

## Ticket (original)

### Objectif
Boucle de jeu stable avec séparation Update/Render, double-buffer de l'état rendu et hooks frame.

### Dépendances
- M00.1
- M00.2
- M00.4

### Livrables
- `Engine::Run()` + `BeginFrame/Update/Render/EndFrame`
- Double-buffer `RenderState[2]` (producer Update / consumer Render)
- Cadre pour fixed tick (option) et frame pacing

### Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

### Spécification technique
- Update écrit dans RenderState A pendant que Render lit B; swap à chaque frame.
- BeginFrame reset FrameArena + collecte input.
- Support vsync on/off via config.

### Étapes d'implémentation
1. Définir structure `RenderState` minimal (camera, view/proj, drawlist placeholder).
2. Implémenter loop (poll input → update → build render state → render → present placeholder).
3. Ajouter compteur FPS dans log toutes les 1s.
4. Préparer hooks `OnResize`, `OnQuit`.

### Critères de validation (DoD)
- [ ] Loop stable (pas de drift), input caméra fluide.
- [ ] Swap RenderState sans data race (atomic index).
- [ ] FrameArena reset par frame.

### Notes / pièges à éviter
- Éviter de faire du rendu dans Update (préparer seulement).
