## Ticket content (M10.5) — référence

# M10.5 — Chunk packaging: geo/tex/instances/nav split

## Objectif
Structurer fichiers chunk en paquets séparés pour charger progressivement.

## Dépendances
- M11.2

## Livrables
- `.pak` format simple
- chunk.meta + bin files

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Séparer `geo.pak`, `tex.pak`, `instances.bin`, `navmesh.bin`, `probes.bin`.

## Étapes d'implémentation
1. Définir format pak (header + entries + offsets).
2. Écrire lecteur pak (streamable).
3. Modifier builder pour écrire paquets.
4. Modifier scheduler IO pour charger par priorité.

## Critères de validation (DoD)
- [ ] Chunk visible affiche HLOD/geo avant textures HD.

## Notes / pièges à éviter
- Utiliser zstd/lz4 pour compression; garder header lisible.
