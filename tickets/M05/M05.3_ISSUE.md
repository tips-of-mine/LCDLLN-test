# M05.3 — IBL: prefiltered specular cubemap (mips)

**Status:** Closed

## Ticket (original)

### Objectif
Générer cubemap specular préfiltrée (GGX) avec mipmaps = roughness.

### Dépendances
- M05.1
- M03.3

### Livrables
- Prefiltered cubemap (256-512) RGBA16F + mips
- Compute shader GGX importance sampling

### Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

### Spécification technique
- Roughness -> mip = r*(mipCount-1).

### Étapes d'implémentation
1. Créer cubemap avec mips.
2. Pour chaque mip, dispatch compute avec roughness correspondant.
3. Écrire face/mip.
4. Transition en sampled.

### Critères de validation (DoD)
- [ ] Reflets deviennent plus flous quand roughness augmente.

### Notes / pièges à éviter
- Prévoir temps de build; possibilité de pré-bake offline.
