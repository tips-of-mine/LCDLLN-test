
**Objectif**
Auction house (hôtel des ventes) avec listings, enchères, buyout immédiat.

**Dépendances**
- M35.1
- M16.1

**Livrables**
- AH database (listings, bids, history)
- Post auction: duration, start bid, buyout
- Search/filter UI
- Bid system avec auto-outbid notification
- Mail delivery items/gold

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- Listing: item, seller, duration (12h/24h/48h), start bid, buyout
- Bid: increment min 5%, track highest bidder
- Buyout: instant purchase, end auction
- Expiry: items/gold mailed to seller/buyer
- AH cut: 5% fee (taken from seller)

**Étapes d'implémentation**
1. DB: auction_listings (id, seller_id, item_id, start_bid, buyout, current_bid, high_bidder, expires_at)
2. UI: create listing (set prices, duration)
3. Post listing: escrow item (remove from inventory)
4. Search UI: filters (item type, price range, sort)
5. Bid: validate amount > current_bid, update high_bidder
6. Outbid notification: mail previous bidder (return gold)
7. Buyout: end auction, transfer item to buyer, gold to seller
8. Expiry cron: check expired auctions, mail results
9. AH cut: subtract 5% from seller proceeds

**Critères de validation (DoD)**
- [ ] Post item en AH avec buyout
- [ ] Search trouve item
- [ ] Bid augmente prix
- [ ] Buyout achète immédiatement
- [ ] Expiry mail items/gold

**Notes / pièges à éviter**
- Prevent bid own auction
- Anti-sniping: extend timer si bid last 5min (optional)
- Index DB par item_id, price pour search performance
