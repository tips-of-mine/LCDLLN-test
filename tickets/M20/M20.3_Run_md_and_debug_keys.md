> **Type**: Ticket (Engineering)  
> **Priority**: High  
> **Owner**: Codex  
> **Status**: TODO  
> **Created**: 2026-02-08

## Objectif
Écrire une documentation minimaliste et utile pour exécuter le projet et activer les modes debug.

Objectif:
- Un nouveau contributeur doit pouvoir compiler et lancer la démo en suivant RUN.md.
- Décrire les arguments CLI les plus importants et où se trouvent les assets.
- Définir une convention stable pour les toggles debug (préfixes `render.*`, `world.*`, etc.).

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Code jeu** : uniquement sous `/game/...`
- **Contenu** (textures/meshes/json/audio/shaders) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- **Interdit** : créer des dossiers racine du type `/assets`, `/textures`, `/data`, `/shaders`, etc.
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json). Pas de chemins absolus.


## Livrables
- `RUN.md` à la racine
- Mise à jour `README.md` (si présent) avec le chemin vers RUN.md
- Liste des flags CLI supportés (au moins les principaux)

## Validation
- RUN.md contient: prérequis, build, run, dépannage (common errors)
- Les chemins d’assets sont documentés (paths.content)

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD
