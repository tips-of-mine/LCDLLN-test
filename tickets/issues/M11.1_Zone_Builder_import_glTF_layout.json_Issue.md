# M11.1 — Zone Builder: import glTF + layout.json

## Objectif
Outil CLI qui lit glTF (assets) + layout.json (placements) pour construire une zone.

## Dépendances
- M00.4

## Livrables
- `tools/zone_builder`
- Parser layout.json
- Importer glTF (tinygltf)

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- layout.json versionné; positions en mètres; GUID instances.

## Étapes d’implémentation
1. Créer exécutable tool, parsing args.
2. Charger layout.json, valider schema minimal.
3. Charger glTF references (meshes/materials).
4. Émettre liste instances + assets.

## Critères de validation (DoD)
- [ ] Tool charge une zone simple et liste instances/meshes.

## Notes / pièges à éviter
- Ne pas faire FBX au début; glTF d'abord.
