# M100.50 — Quick Start Wizard (Phase 12 — Accessibilité)

## Résumé

Assistant graphique en **5 étapes guidées** qui produit une zone jouable de A à Z sans qu'aucune connaissance préalable de l'éditeur ne soit requise. Le mappeur choisit :

1. Un **climat** (tempéré, aride, polaire, tropical) → influence palette splat + paramètres érosion.
2. Un **relief dominant** (plaines, collines, montagnes, escarpé) → multiplicateurs amplitude des macros.
3. Un **thème côtier** (intérieur, côte modérée, côte spectaculaire) → influence Coastline.
4. Un **point d'intérêt initial** (aucun, grotte, ruine antique, donjon) → place un mesh insert au centre.
5. Une **prévisualisation 3D** statique de la zone à générer + customisation seed.

À la dernière étape, l'utilisateur clique « Générer la zone » et l'éditeur exécute la séquence d'opérations dérivée des choix, en réutilisant l'infrastructure `ZonePresetExecutor` de M100.46. Résultat : une zone unique, cohérente avec les choix, prête à être enrichie ou exportée.

Différences clés vs M100.46 Zone Presets Library :
- **M100.46** : choisir parmi 8 zones pré-faites (Forêt tempérée, etc.).
- **M100.50** : composer une zone par combinaison de paramètres (climat × relief × côte × POI), ce qui donne ~80 combinaisons possibles.

Le wizard est accessible depuis :
- `Fichier → Nouvelle zone… → Mode guidé`
- Bouton « 🪄 Assistant » dans la modal de création de zone (en alternative au choix de preset).
- Étape automatique à la fin du tutoriel premier lancement (M100.49) : « Voulez-vous essayer l'assistant pour créer une zone personnalisée ? ».

**Aucun nouvel exécutable.** Code dans `engine_core`.

## Contexte critique à respecter

### 1. Le wizard COMPOSE, le preset library FOURNIT

Architecture :
- M100.46 a 8 fichiers JSON pré-faits, prêts à l'emploi.
- M100.50 a un **template paramétré** : une template-string d'opérations avec placeholders qui sont remplis selon les choix utilisateur.
- Le template est UN seul fichier `wizard_template.json` qui décrit toutes les combinaisons.

Ainsi, le wizard ne duplique pas les 8 presets ; il génère une zone unique à chaque combinaison de choix.

### 2. Préview 3D statique, pas live

L'étape 5 affiche une **image statique** (PNG ou rendu offline pré-calculé) qui représente le résultat attendu pour la combinaison choisie. Pas de rendu 3D live (coûteux). Si le mappeur veut voir le vrai résultat, il génère.

Le système choisit l'image à afficher dans une bibliothèque `game/data/editor/wizard/previews/<climat>_<relief>_<côte>.png`. MVP : ~12 images de référence pour les combinaisons les plus courantes (4 climats × 3 reliefs simplifiés). Pour les autres combinaisons, fallback sur l'image la plus proche + texte « Aperçu approximatif ».

### 3. Skippable à toute étape

Boutons toujours présents : **Précédent**, **Suivant**, **Générer maintenant** (à partir de l'étape 2, avec les choix par défaut pour les étapes suivantes).

### 4. Pas de saved progress entre sessions

Si l'utilisateur quitte au milieu, les choix sont perdus. Le wizard est rapide (< 2 minutes) donc pas critique. Phase ultérieure possible : sauvegarder les wizard choices comme un preset perso.

### 5. Décoration : section vide identique à M100.46

Le wizard génère des zones avec `decoration: []` vide pour le MVP, à enrichir par Phase 13.

### 6. Invariant terrain visible

Le wizard est une succession de modaux ImGui. Pas de touch viewport pendant les étapes 1-4. À l'étape 5, preview = image statique. À la génération, exécution via M100.46 (qui respecte déjà l'invariant).

### 7. Une seule application

Pas de nouvel exécutable hors la suite `quick_start_wizard_tests`.

### 8. Arbo cmangos-style

- `src/world_editor/wizard/` (nouveau).
- `game/data/editor/wizard/` (template + previews).

Notations :
- `{{variable}}` : substitué par la valeur du choix utilisateur.

- `"if"` : opération exécutée seulement si la condition est vraie.

#### Partie B — `WizardTemplateResolver`

Service qui prend le template + les choix + le seed et produit une liste d'opérations équivalente à un `ZonePreset` standard, consommable par `ZonePresetExecutor`.

```cpp
namespace engine::editor::world::wizard
{
    struct WizardChoices
    {
        std::string climate;     // "temperate", "arid", ...
        std::string relief;
        std::string coast;
        std::string poi;
        uint32_t    seed = 42;
    };

    class WizardTemplateResolver
    {
    public:
        /// Charge le template au boot.
        void LoadFromContentPath();

        /// Résout le template avec les choix + seed et produit
        /// un ZonePreset prêt à être passé au ZonePresetExecutor.
        zone_presets::ZonePreset Resolve(const WizardChoices& choices) const;
    };
}
```

Résolution étape par étape :
1. Pour chaque opération du template :
   - Évaluer la condition `if` → skip si false.
   - Substituer les `{{var}}` par les valeurs des choix.
   - Évaluer les `$auto_xxx` via une fonction d'auto-génération.
2. Le tool preset référencé (ex : `temperate_hills`) doit exister dans M100.45. Si pas trouvé, fallback sur le preset par défaut de l'outil.
3. Renvoyer un `ZonePreset` complet avec `decoration: []`.

#### Partie C — Fonctions d'auto-génération

`AutoGenerators.h/.cpp` :

```cpp
namespace engine::editor::world::wizard
{
    /// Génère une polyline de chaîne montagneuse adaptée au relief choisi.
    /// hills : 3 points, sinuosité légère, hauteur modérée.
    /// mountains : 5 points, plus dramatique.
    /// steep : 3 points, escarpés.
    /// plains : pas appelé (skipped via if condition).
    std::vector<engine::math::Vec2> AutoChainForRelief(
        const std::string& relief, uint32_t seed);

    /// Sea level adapté au thème côtier.
    /// inland: appelé seulement si l'op ne dépend pas, retourne -inf.
    /// moderate: 50m. dramatic: 30m.
    float AutoSeaLevelForCoast(const std::string& coast);

    /// 3 sources de rivière placées aléatoirement dans les zones hautes
    /// déterminé par le relief.
    std::vector<engine::math::Vec2> AutoRiverSources(uint32_t seed,
                                                      const std::string& relief);

    /// Position centrale de la zone, légèrement décalée pour ne pas
    /// systématiquement coller au centre.
    engine::math::Vec3 AutoCenterOffset(uint32_t seed);
}
```

Toutes les fonctions sont **déterministes** étant donné le seed.

##### D.7 — Génération

Click « Générer la zone » → :
1. `WizardTemplateResolver.Resolve(choices)` → `ZonePreset`.
2. Lance `ZonePresetExecutor.Execute(preset, ...)` avec une modale de progression (réutilisée de M100.46).
3. À la fin, exécute automatiquement la validation (M100.48) en background.
4. Affiche un toast : « Zone créée ! 2 warnings détectés (ouvrir la validation). »
