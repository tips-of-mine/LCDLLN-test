# M32.4_Guild_tabard_+_emblem_+_bank_(optionnel)

## Objectif
Tabard guild customizable, emblem visuel, guild bank partagée.

## Dépendances
- M32.3
- M16.3

## Livrables
- Tabard item (équipable chest)
- Emblem designer (couleurs, symbole)
- Guild bank (shared storage)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Tabard: texture overlay sur chest armor
- Emblem: background color, symbol, border
- Guild bank: 6-8 tabs, 98 slots each
- Permissions per rank per tab (view, deposit, withdraw)

## Étapes d'implémentation
1. Tabard item: equipment slot, apply texture shader
2. Emblem data: store in guild table (JSON colors/symbol)
3. Render emblem: procedural texture ou atlas symbols
4. Guild bank: separate inventory storage (shared)
5. Permissions: rank can access tabs 0-N
6. Log withdrawals (audit trail)
7. UI: bank panel similar inventory

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD

## Notes / pièges à éviter
- Emblem cache client-side (pas re-download each frame)
- Gold bank separate from items
