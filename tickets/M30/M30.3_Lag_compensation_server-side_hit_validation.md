
**Objectif**
Rewind simulation serveur pour valider hits client (compensate latency).

**Dépendances**
- M30.1
- M28.1 (skills)

**Livrables**
- Server: store historical positions (ring buffer)
- Skill use: rewind à timestamp client
- Hit validation à t-latency

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- Buffer 1s historical positions per entity
- SkillUseRequest: include client timestamp
- Server: rewind world à (now - RTT/2)
- Validate hit at rewound positions
- Apply effects à current state

**Étapes d'implémentation**
1. Server: track positions ring buffer per entity (60Hz, 1s = 60 snapshots)
2. Client: send SkillUseRequest(skillId, targetId, clientTimestamp)
3. Server: calculate rewind time = clientTimestamp - RTT/2
4. Fetch historical positions à rewind time
5. Check hit (range, LOS) using rewound positions
6. If valid: apply effects à current state
7. Broadcast result (hit/miss)

**Critères de validation (DoD)**
- [ ] Hit validation "fair" même avec 100ms latency
- [ ] Joueur voit hit si target était visible client-side
- [ ] Pas d'exploits (rewind limité à 500ms max)

**Notes / pièges à éviter**
- Rewind trop loin = abuse (limit 500ms)
- LOS check important (pas hit à travers murs)
