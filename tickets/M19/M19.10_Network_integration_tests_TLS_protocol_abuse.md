# M19.10 — Network integration tests: TLS + protocol + abuse cases

## Objectif
Mettre en place des tests d’intégration automatisés couvrant TLS, framing protocole, erreurs, et abus.

## Dépendances
- M19.5
- M19.6
- M19.7
- M19.8
- M19.9

## Livrables
- Binaire client headless test
- Harness:
  - start server (port ephemeral)
  - connect TLS
  - run scénarios
- Scénarios:
  - handshake OK
  - fingerprint mismatch (client refuse)
  - ping/pong framing
  - oversize reject
  - unknown opcode → ERROR
  - flood → disconnect offender

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Tests déterministes.
- Pas de dépendance au rendu.
- Exécuter en CI Linux.

## Étapes d’implémentation
1. Écrire test runner.
2. Lancer serveur sur port libre.
3. Exécuter scénarios.
4. Exit non-zero si fail.

## Critères de validation (DoD)
- [ ] Tests passent en local + CI.
- [ ] Pas de flaky tests.
