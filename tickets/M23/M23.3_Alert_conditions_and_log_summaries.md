# M23.3 — Alert conditions (rules) + log summaries

## Objectif
Définir conditions d’alerte (documentation + hooks logs) pour incidents critiques (shard down, DB inaccessible, auth spike).

## Dépendances
- M23.2

## Livrables
- `docs/alerts.md`:
  - shard heartbeat timeout
  - db down
  - high auth failures rate
  - high disconnect rate
- Logs résumés périodiques (toutes 60s) pour état cluster

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Alerte “down” basée sur métriques (Prometheus rule) + logs côté serveur.
- Ne pas spammer; summary throttled.

## Étapes d’implémentation
1. Définir règles.
2. Implémenter summary periodic logs.
3. Vérifier avec scénarios (kill shard, stop DB).

## Critères de validation (DoD)
- [ ] Docs utilisables pour config Alertmanager.
- [ ] Summary logs lisibles.
