# M32.3_Guild_system_creation_+_roster_+_ranks

## Objectif
Système guild/clan avec création, roster, grades, permissions, bank (optionnel).

## Dépendances
- M13.1
- M14.4

## Livrables
- Guild DB tables (guilds, guild_members, ranks)
- Create guild (cost gold, unique name)
- Invite/kick/promote
- Ranks with permissions
- Guild chat channel (M29)
- Guild bank (optionnel MVP)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Creation cost: 1000 gold, name unique
- Max 500 members (configurable)
- Ranks: Guild Master, Officer, Member, Recruit (customizable)
- Permissions: invite, kick, promote, withdraw bank, edit MOTD
- MOTD (message of the day)

## Étapes d'implémentation
1. DB: guilds (id, name, motd, created_at)
2. DB: guild_members (guild_id, player_id, rank, joined_at)
3. DB: guild_ranks (guild_id, rank_name, permissions_bitfield)
4. /guild create <name>: deduct gold, insert DB
5. /ginvite <name>: validate permissions, send invite
6. /gpromote, /gkick: check permissions
7. Guild chat: route messages to all online members
8. UI: roster panel, rank icons, online count

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD

## Notes / pièges à éviter
- Guild name validation (profanity, length 3-20)
- Disband guild: requires vote ou GM only
