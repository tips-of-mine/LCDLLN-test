# M02.3 — Frame Graph: barrières Vulkan (MVP) + tracking layouts

## Objectif
Insérer les barrières nécessaires entre passes (layout transitions + access masks).

## Dépendances
- M02.2
- M01.4

## Livrables
- Tracking dernier usage par ressource (layout/stage/access)
- Barrières `vkCmdPipelineBarrier2` (si sync2) ou legacy

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- MVP acceptable: barrière conservative entre passes.
- Objectif: transitions correctes pour COLOR_ATTACHMENT, SAMPLED, TRANSFER.

## Étapes d’implémentation
1. Définir états d'usage (ColorWrite, DepthWrite, SampledRead, TransferSrc/Dst).
2. Mapper usage -> layout + stage/access.
3. Avant chaque pass, insérer barrières pour ressources utilisées.
4. Tester avec SceneColor offscreen + copy.
