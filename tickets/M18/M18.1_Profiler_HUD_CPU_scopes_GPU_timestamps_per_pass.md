# M18.1 — Profiler HUD: CPU scopes + GPU timestamps per pass

## Objectif
Profiler intégré: scopes CPU multi-thread + GPU timestamps (frame graph passes) + HUD overlay.

## Dépendances
- M02.2
- M00.3

## Livrables
- PROFILE_SCOPE macros
- GPU query pool
- Debug overlay

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Afficher ms total + breakdown top passes; export json optionnel.

## Étapes d’implémentation
1. Implémenter CPU profiler (ring buffer events).
2. Implémenter GPU timestamps (vkCmdWriteTimestamp).
3. Associer passes frame graph à scopes GPU.
4. UI overlay pour lecture.

## Critères de validation (DoD)
- [ ] Voir coût GBuffer/Shadows/SSAO/etc en ms.

## Notes / pièges à éviter
- Attention au nombre de queries; recycler.
