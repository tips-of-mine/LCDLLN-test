
**Objectif**
Emotes textuelles (/dance, /laugh) et chat bubbles au-dessus personnages.

**Dépendances**
- M29.1
- M25.3 (animations pour emotes)

**Livrables**
- Emote commands (/dance, /wave, etc.)
- Animation playback pour emotes
- 3D chat bubbles (texte au-dessus tête)

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- Emotes: /dance, /wave, /laugh, /cry, /sit, etc.
- Animation play once (ou loop si /sit)
- Chat bubbles: durée 3-5s, fade out
- Billboard texte face caméra

**Étapes d'implémentation**
1. Emote command: trigger animation state (M25.3)
2. Broadcast EmoteEvent(playerId, emoteId) aux clients proches
3. Clients: play animation correspondante
4. Chat bubble: on say/yell, spawn billboard text au-dessus tête
5. Render text 3D (world space UI ou shader)
6. Auto-destroy après timeout

**Critères de validation (DoD)**
- [ ] /dance joue animation dance
- [ ] Say message affiche bubble 3D
- [ ] Bubble visible par autres joueurs
- [ ] Fade out après 5s

**Notes / pièges à éviter**
- Limiter bubbles actives (max 10 visible)
- Culling bubbles hors screen
