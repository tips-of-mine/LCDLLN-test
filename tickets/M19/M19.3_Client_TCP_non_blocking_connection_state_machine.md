# M19.3 — Client TCP non-blocking + connection state machine

## Objectif
Implémenter la couche réseau client TCP non-bloquante + framing protocole v1, indépendante du gameplay.

## Dépendances
- M19.2
- M00.1
- M00.3

## Livrables
- `NetClient` (thread réseau dédié)
  - connect avec timeout (10s)
  - read loop (partial reads) + framing (size header)
  - write queue non-bloquante
  - disconnect propre + raisons
- API events:
  - connected/disconnected
  - packet received
- Counters: bytes in/out, packets in/out

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- RX:
  - buffer accumulate
  - parse header quand >= header_size
  - attendre `size` bytes
  - déliver packet complet
- TX:
  - file d’attente par connexion
  - flush en non-bloquant
- Timeouts (hooks):
  - connect 10s
  - response warning 5s (M19.7)
  - heartbeat timeout 90–120s (M20)

## Étapes d’implémentation
1. Connect non-bloquant (socket + deadline).
2. Thread réseau: boucle poll/read/write.
3. Framing: utiliser `PacketView` pour valider header et taille.
4. Route packets vers queue thread-safe vers main.
5. Ajouter instrumentation logs + counters.

## Critères de validation (DoD)
- [ ] Connexion à serveur dummy + échange ping/pong stable.
- [ ] RX gère partial packets.
- [ ] Aucun blocage du thread main.

## Notes / pièges à éviter
- Éviter allocations par paquet (réutiliser buffers).
- Ne pas exécuter callbacks utilisateur dans le thread réseau.
