# M09.5 — Runtime HLOD switching + culling

**Status:** Closed

## Ticket (original)

### Objectif
Au-delà d’un seuil, rendre HLOD à la place des props individuels + culling frustum/distance.

### Dépendances
- M09.4
- M10.4

### Livrables
- Switch props->HLOD selon distance
- Culling CPU stable

### Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu : sous `/game/data/...`
- Outils offline : sous `/tools/...`
- Chemins relatifs à `paths.content`.

### Spécification technique
- Seuil HLOD ~200m (base).

### Étapes d'implémentation
1. Dans build draw lists, choisir HLOD vs instances.
2. Culling bounding volumes.
3. Debug: overlay HLOD state.

### Critères de validation (DoD)
- [ ] Drawcalls chutent nettement quand on s’éloigne.

### Notes / pièges à éviter
- Occlusion culling plus tard (M18).

### Détail implémentation

- **HlodRuntime** : `GetHlodDistanceThreshold()` (config), `UseHlodForDistance(distance)`, `VisibleInFrustum(frustum, aabbMin, aabbMax)` (délègue à `Frustum::TestAABB`), `WithinDrawDistance(cameraPos, point, maxDistance)`.
- **Geometry pass** : centre de l’objet depuis la matrice modèle, distance caméra–objet, AABB (centre ± 2), culling frustum + distance ; si visible, `UseHlodForDistance(dist)` détermine HLOD ou instances, incrément de `m_hlodDrawsThisFrame` ou `m_instanceDrawsThisFrame`, puis draw + `RecordDraw`. Si non visible, aucun draw (culling).
- **Overlay debug** : si `debug.hlod_overlay` est true, log toutes les 60 frames : `HLOD overlay: hlod_draws=N instance_draws=M`.
