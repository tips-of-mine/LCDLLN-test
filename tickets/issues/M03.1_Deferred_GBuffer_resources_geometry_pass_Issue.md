# M03.1 — Deferred: GBuffer resources + geometry pass

## Objectif
Créer GBuffer (A/B/C + Depth) et remplir via geometry pass (opaques) dans frame graph.

## Dépendances
- M02.4

## Livrables
- FG images: GBufferA, GBufferB, GBufferC, Depth
- Pipeline geometry (VS/PS) + render pass compatible

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Formats recommandés: A=R8G8B8A8_SRGB, B=A2B10G10R10_UNORM ou R16G16B16A16_SFLOAT, C=R8G8B8A8_UNORM, Depth=D32_SFLOAT.
- Écrire albedo/normal/ORM; depth test enabled.

## Étapes d'implémentation
1. Définir descriptors GBuffer dans FG.
2. Créer render pass/FB pour multiple color attachments + depth (ou dynamic rendering).
3. Écrire shaders geometry (outputs).
4. Brancher un mesh test (cube/sponza).

## Critères de validation (DoD)
- [ ] GBuffer remplit correctement (debug view possible via renderdoc).
- [ ] Pas d'erreurs validation.

## Notes / pièges à éviter
- Commencer avec une seule mesh + material simple.
