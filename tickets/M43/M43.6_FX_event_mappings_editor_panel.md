# M43.6 — Éditeur FX et event mappings (`game/data/fx/`)

**Priorité :** Haute  
**Périmètre :** `engine/editor/EditorFxPanel.cpp/.h` · `engine/editor/EditorMode.cpp/.h`  
**Dépendances :** M43.4 (ImGui intégré) · M17.1 (FXManager + structures `FXDefinition`, `FXEventMapping`)

---

## Objectif

Ajouter dans le mode `--editor` un panneau ImGui permettant de **lire, modifier et sauvegarder** les deux fichiers FX :
- `game/data/fx/effects.json` — catalogue des effets visuels composites
- `game/data/fx/event_mappings.json` — liaisons événement gameplay → effet FX

---

## Contexte

| Fichier | Structures C++ | Contenu |
|---------|----------------|---------|
| `effects.json` | `FXDefinition` | id, particleId, decalId, soundId, scale, durationSeconds |
| `event_mappings.json` | `FXEventMapping` | eventType → fxId |

Ces deux fichiers sont couplés : un mapping référence un fxId qui doit exister dans `effects.json`. L'éditeur doit valider cette cohérence en temps réel.

---

## Changements requis

### Nouveaux fichiers

**`engine/editor/EditorFxPanel.h`**
```cpp
#pragma once
#include "engine/client/FXManager.h"
#include "engine/core/Config.h"
#include <string>
#include <vector>

namespace engine::editor
{
    class EditorFxPanel final
    {
    public:
        bool Load(const engine::core::Config& config);
        bool Save(const engine::core::Config& config);
        void Draw();
        bool IsDirty() const { return m_dirty; }

    private:
        std::vector<engine::client::FXDefinition>   m_effects;
        std::vector<engine::client::FXEventMapping> m_mappings;
        std::string m_effectsPath;
        std::string m_mappingsPath;
        bool m_dirty = false;

        void DrawEffectsSection();
        void DrawMappingsSection();
        bool ValidateMappings() const; // vérifie que tous les fxId de m_mappings existent dans m_effects
    };
}
```

**`engine/editor/EditorFxPanel.cpp`**

`Load()` :
- Lire `fx/effects.json` → populer `m_effects`
- Lire `fx/event_mappings.json` → populer `m_mappings`
- `LOG_INFO("[EditorFX] Loaded {} effects, {} mappings", ...)`

`Save()` :
- Sérialiser `m_effects` → `fx/effects.json`
- Sérialiser `m_mappings` → `fx/event_mappings.json`
- `LOG_INFO("[EditorFX] Saved effects.json + event_mappings.json")`

`Draw()` — deux onglets `ImGui::BeginTabBar("FxTabs")` :

**Onglet Effects :**
```
[+] Add Effect
────────────────────────────────────────────────────────────────────
 id                    particleId            decalId        soundId          scale  duration(s)
[combat_hit_default ] [particles/hit_spark] [decals/hit_mk] [sounds/hit_01 ] [1.0 ] [0.35]   [X]
```
- `ImGui::InputText` pour id, particleId, decalId, soundId (max 128 chars)
- `ImGui::InputFloat` pour scale (0.01–10.0) et durationSeconds (0.0–60.0)
- Bouton `[X]` pour supprimer (avec confirmation : popup "Delete effect 'id' ?")

**Onglet Event Mappings :**
```
[+] Add Mapping
Validation: ✓ All mappings resolved
────────────────────────────────────────────────────────
 eventType              fxId
[CombatEvent         ] [Combo: combat_hit_default ▼]   [X]
```
- `ImGui::InputText` pour eventType
- `ImGui::Combo` pour fxId : liste déroulante des ids de `m_effects` (plus robuste que texte libre)
- Indicateur de validation en temps réel :
  - `✓ All mappings resolved` (vert) si tous les fxIds existent
  - `⚠ N unresolved fxId(s)` (orange) sinon, avec liste des fxIds inconnus

Chaque modification positionne `m_dirty = true`.

### `engine/editor/EditorMode.h`

Ajouter :
```cpp
#include "engine/editor/EditorFxPanel.h"
engine::editor::EditorFxPanel m_fxPanel;
```

### `engine/editor/EditorMode.cpp`

- Dans `Init()` : `m_fxPanel.Load(config);`
- Rendu ImGui :
  ```cpp
  if (ImGui::Begin("FX Editor"))
      m_fxPanel.Draw();
  ImGui::End();
  ```
- Propager `m_fxPanel.IsDirty()` dans `m_dirty` global
- `Ctrl+S` → `m_fxPanel.Save(config)` quand la fenêtre FX est active

## Format JSON produit

**`effects.json`**
```json
{
  "effects": [
    {
      "id": "combat_hit_default",
      "particleId": "particles/hit_spark",
      "decalId": "decals/hit_mark",
      "soundId": "sounds/hit_01",
      "scale": 1.0,
      "durationSeconds": 0.35
    }
  ]
}
```

**`event_mappings.json`**
```json
{
  "mappings": [
    { "eventType": "CombatEvent", "fxId": "combat_hit_default" }
  ]
}
```

JSON sauvegardé avec indentation 2 espaces.

## Critères d'acceptation

- [ ] Build Windows OK
- [ ] `--editor` : le panneau "FX Editor" s'ouvre et affiche les effets et mappings existants
- [ ] Ajouter un effet → apparaît dans la liste et dans le Combo des mappings
- [ ] Créer un mapping avec un fxId inexistant → indicateur `⚠` orange en temps réel
- [ ] Supprimer un effet référencé par un mapping → indicateur `⚠` immédiatement
- [ ] `Ctrl+S` → les deux fichiers JSON sont sauvegardés, `*dirty*` disparu
- [ ] Fichier absent → `LOG_ERROR` sans crash, panneau vide éditable
- [ ] Rapport final : fichiers + commandes + résultats + DoD

---

## Interdit

- Ne pas modifier `FXManager.cpp/.h`
- Ne pas déclencher de rechargement FX live (hors scope)
- Ne pas fusionner les deux fichiers en un seul (la séparation effets/mappings est intentionnelle)
