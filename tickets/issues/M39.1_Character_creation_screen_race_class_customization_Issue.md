# Issue M39.1 — Character creation screen: race/class customization

**Status:** Closed

## Ticket source

```
# M39.1_Character_creation_screen_race class customization

## Objectif
Écran création personnage avec sélection race, classe, apparence.

## Dépendances
- M16.1
- M33.1

## Livrables
- UI screens: race selection, class selection, customization
- 3D preview character (rotation)
- Customization sliders (face, hair, colors)
- Name input + validation (unique, profanity)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Races: 4-8 (humans, elves, orcs, etc.)
- Classes: 4-10 (warrior, mage, rogue, etc.)
- Customization: face type, hair style, skin color, hair color, eye color
- Preview: rotate character 360°, apply changes realtime

## Étapes d'implémentation
1. UI flow: login -> character select -> creation (si aucun char)
2. Race selection: grid icons, tooltip racials
3. Class selection: grid icons, tooltip abilities
4. Customization panel: sliders/dropdowns
5. 3D preview: load character model, apply customization
6. Rotate camera: mouse drag
7. Name input: validate length (3-12), check DB unique
8. Create: send CharacterCreateRequest(name, race, class, customization)
9. Server: validate, insert DB, return success

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD

## Notes / pièges à éviter
- Profanity filter nom (blacklist words)
- Limit characters per account (5-10)
```
