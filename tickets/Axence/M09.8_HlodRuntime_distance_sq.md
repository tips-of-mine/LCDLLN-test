# M09.8 — HlodRuntime : suppression du sqrt sur le hot path de BuildChunkDrawList

## Objectif
Éliminer l'appel `std::sqrt` dans `BuildChunkDrawList` pour la décision HLOD/instances,
en comparant directement les carrés de distances. Le sqrt est inutile pour une
comparaison de seuil et pénalise un hot path appelé sur l'ensemble des chunks actifs
à chaque frame.

## Contexte
Dans `HlodRuntime.cpp`, héritage de M09.5 :
```cpp
float distanceMeters = std::sqrt(dx*dx + dy*dy + dz*dz);  // ← inutile pour la décision
bool useHlod = hlod.ShouldUseHlod(distanceMeters);        // compare à m_thresholdMeters
```
`ShouldUseHlod` compare `distanceMeters > m_thresholdMeters`. Il suffit de comparer
les carrés pour éviter le `sqrt`. La racine carrée ne reste nécessaire que pour
`ChunkDrawDecision::distanceMeters` (debug/overlay) et uniquement pour les chunks
non culled — soit une fraction des chunks traités.

## Dépendances
- M09.5 (HlodRuntime + BuildChunkDrawList existants)

## Livrables
- `engine/world/HlodRuntime.h`
  - Membre privé `float m_thresholdSq = 0.0f` (précalculé dans `Init`).
  - Méthode `bool ShouldUseHlodSq(float distSq) const` (compare `distSq > m_thresholdSq`).
- `engine/world/HlodRuntime.cpp`
  - `Init` : `m_thresholdSq = m_thresholdMeters * m_thresholdMeters`.
  - `BuildChunkDrawList` :
    1. Calculer `distSq = dx²+dy²+dz²`.
    2. `dec.useHlod = hlod.ShouldUseHlodSq(distSq)`.
    3. `dec.distanceMeters = dec.culled ? 0.0f : std::sqrt(distSq)` (sqrt limité aux chunks visibles).
  - Conserver `ShouldUseHlod(float distanceMeters)` inchangé (compatibilité call-sites existants).

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
```cpp
// Avant (M09.5)
float dist = std::sqrt(dx*dx + dy*dy + dz*dz);
bool hlod   = ShouldUseHlod(dist);

// Après (M09.8)
float distSq = dx*dx + dy*dy + dz*dz;
bool hlod     = ShouldUseHlodSq(distSq);                             // pas de sqrt
dec.distanceMeters = dec.culled ? 0.0f : std::sqrt(distSq);          // sqrt uniquement si visible
```
- `IsVisible` utilise déjà `distSq <= maxSq` correctement : ne pas le modifier.
- `m_thresholdSq` doit être recalculé si `m_thresholdMeters` change (actuellement fixé au boot).

## Étapes d'implémentation
1. Ajouter `m_thresholdSq` et `ShouldUseHlodSq` dans `HlodRuntime.h`.
2. Calculer `m_thresholdSq` dans `Init`.
3. Mettre à jour `BuildChunkDrawList` : utiliser `distSq` + sqrt conditionnel.
4. Vérifier que le comportement (seuil identique) est conservé par rapport à M09.5.

## Notes / pièges à éviter
- Ne pas supprimer `ShouldUseHlod(float)` : compatible avec des call-sites qui disposent déjà d'une distance calculée.
- `dec.distanceMeters = 0.0f` pour les chunks culled est acceptable (valeur non affichée dans l'overlay).
- Ne pas inliner `m_thresholdSq` comme constexpr : il dépend de la config runtime (`world.hlod_distance_threshold_m`).
