
**Objectif**
Charger animations skelettiques depuis glTF et implémenter playback runtime avec interpolation de poses.

**Dépendances**
- M3.3 (mesh loading)
- M0.3 (jobs)

**Livrables**
- `engine/animation/Skeleton.*` (joints hierarchy)
- `engine/animation/AnimationClip.*` (keyframes, channels)
- `engine/animation/AnimationPlayer.*` (playback, sampling)
- Loader glTF animations

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- Support rotations (quaternions), translations, scales
- Interpolation linéaire (LERP/SLERP)
- Sampling à 30-60 Hz
- Time normalisé [0..duration]

**Étapes d'implémentation**
1. Définir structures Skeleton (joints, parenté, bind pose)
2. Parser glTF animations (accessors, samplers, channels)
3. Implémenter AnimationPlayer: update(dt), sample pose
4. Interpoler keyframes (linear pour MVP)
5. Output pose = array transforms local-space

**Critères de validation (DoD)**
- [ ] Charger animation idle/walk depuis glTF
- [ ] Playback loop fluide à 60 FPS
- [ ] Interpolation sans artefacts visibles

**Notes / pièges à éviter**
- Quaternion SLERP coûteux: optimiser plus tard
- Prévoir compression animations (quantization) phase 2
