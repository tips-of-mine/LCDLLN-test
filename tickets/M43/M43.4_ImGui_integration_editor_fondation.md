# M43.4 — ImGui : intégration dans le mode `--editor` (fondation)

**Priorité :** Haute  
**Périmètre :** `engine/editor/EditorMode.cpp/.h` · `engine/Engine.cpp` · `CMakeLists.txt` · `vcpkg.json`  
**Dépendances :** M12.1 (EditorMode existant) · M01.4 (command buffers Vulkan)

---

## Objectif

Intégrer **Dear ImGui** (backend Vulkan + GLFW) dans le mode `--editor` pour remplacer le shell window-title actuel par des panneaux overlay interactifs rendus dans la fenêtre principale.

Sans ce ticket, aucun éditeur de contenu (M43.5 à M43.8) ne peut être implémenté.

---

## Contexte

L'`EditorMode` actuel utilise le titre de la fenêtre GLFW comme seule interface (`window.SetTitle(...)`). Tous les sous-éditeurs prévus (audio, FX, loot, thème, quêtes, matériaux, particules) nécessitent une UI interactive. Dear ImGui avec le backend `imgui_impl_vulkan` + `imgui_impl_glfw` est la solution retenue — elle s'intègre directement dans le command buffer existant sans passe de rendu supplémentaire complexe.

---

## Changements requis

### `vcpkg.json`

Ajouter la dépendance :
```json
"imgui[vulkan-binding,glfw-binding]"
```

### `CMakeLists.txt`

- `find_package(imgui CONFIG REQUIRED)` après les autres `find_package`
- Lier `imgui::imgui` à `engine_core`

### `engine/editor/EditorMode.h`

Ajouter les includes ImGui (en forward declaration dans le header, implémentation dans `.cpp`) :
```cpp
// Forward declare — ne pas inclure imgui.h dans le header
struct ImGuiContext;
```

Ajouter dans la classe `EditorMode` :
```cpp
/// Contexte ImGui — initialisé par ImGuiBegin, détruit par ImGuiShutdown.
ImGuiContext* m_imguiContext = nullptr;

/// Initialise ImGui (backend Vulkan + GLFW). Appeler après Init().
bool ImGuiInit(GLFWwindow* window, VkInstance instance, VkPhysicalDevice physDev,
               VkDevice device, uint32_t graphicsQueueFamily, VkQueue graphicsQueue,
               VkRenderPass renderPass, uint32_t imageCount);

/// Shutdown ImGui. Appeler avant Shutdown().
void ImGuiShutdown(VkDevice device);

/// Début de frame ImGui — appeler en début de Render(), avant vkBeginCommandBuffer.
void ImGuiNewFrame();

/// Fin de frame ImGui — enregistre les draw calls dans cmd.
void ImGuiRender(VkCommandBuffer cmd);
```

### `engine/editor/EditorMode.cpp`

Implémenter `ImGuiInit` :
```cpp
bool EditorMode::ImGuiInit(GLFWwindow* window, VkInstance instance, VkPhysicalDevice physDev,
                            VkDevice device, uint32_t graphicsQueueFamily, VkQueue graphicsQueue,
                            VkRenderPass renderPass, uint32_t imageCount)
{
    IMGUI_CHECKVERSION();
    m_imguiContext = ImGui::CreateContext();
    ImGui::SetCurrentContext(m_imguiContext);
    ImGuiIO& io = ImGui::GetIO();
    io.ConfigFlags |= ImGuiConfigFlags_NavEnableKeyboard;

    ImGui::StyleColorsDark();

    ImGui_ImplGlfw_InitForVulkan(window, true);

    ImGui_ImplVulkan_InitInfo initInfo{};
    initInfo.Instance       = instance;
    initInfo.PhysicalDevice = physDev;
    initInfo.Device         = device;
    initInfo.QueueFamily    = graphicsQueueFamily;
    initInfo.Queue          = graphicsQueue;
    initInfo.RenderPass     = renderPass;
    initInfo.MinImageCount  = imageCount;
    initInfo.ImageCount     = imageCount;
    ImGui_ImplVulkan_Init(&initInfo);

    LOG_INFO(Core, "[EditorImGui] Dear ImGui {} initialized (Vulkan + GLFW backend)",
             IMGUI_VERSION);
    return true;
}
```

Implémenter `ImGuiShutdown`, `ImGuiNewFrame`, `ImGuiRender` selon la documentation Dear ImGui.

### `engine/Engine.cpp`

- Après `m_pipeline->Init(...)`, si `m_editorEnabled`, appeler :
  ```cpp
  m_editorMode->ImGuiInit(m_glfwWindowForVk, m_vkInstance.GetInstance(),
      m_vkDeviceContext.GetPhysicalDevice(), m_vkDeviceContext.GetDevice(),
      m_vkDeviceContext.GetGraphicsQueueFamily(), m_vkDeviceContext.GetGraphicsQueue(),
      <renderPassFromPipeline>, m_vkSwapchain.GetImageCount());
  ```
- Dans `Render()`, si `m_editorEnabled` :
  - Avant `vkBeginCommandBuffer` : `m_editorMode->ImGuiNewFrame()`
  - Après `m_frameGraph.execute(...)` et avant `vkEndCommandBuffer` : `m_editorMode->ImGuiRender(cmd)`
- Dans `Run()` shutdown, si `m_editorEnabled` : `m_editorMode->ImGuiShutdown(device)`

### Panneau de base — "Editor Hub"

Une fois ImGui opérationnel, remplacer `RefreshShell` par un panneau ImGui minimal affiché en overlay en haut à gauche :

```
[LCDLLN Editor] fps=60 *dirty*
[Scene]    obj: 0  | [Inspector]  (none)
[Assets]   mesh:1 tex:1
```

Le panneau doit être non-modal (`ImGui::Begin("EditorHub", nullptr, ImGuiWindowFlags_NoDecoration | ImGuiWindowFlags_AlwaysAutoResize)`).

---

## Règle de rendu ImGui dans la render pass

ImGui doit être rendu **dans la render pass finale** (celle qui écrit dans le swapchain / backbuffer), pas dans une passe intermédiaire du FrameGraph. Récupérer le `VkRenderPass` de la passe `CopyPresent` ou de la passe finale.

---

## Critères d'acceptation

- [ ] Build Windows OK, aucune erreur de linker
- [ ] `--editor` : le panneau "Editor Hub" s'affiche en overlay dans la fenêtre principale
- [ ] Aucun panneau ImGui visible sans `--editor`
- [ ] `fps`, `dirty`, compteurs mesh/tex sont corrects dans le panneau
- [ ] Aucune régression de rendu sur les passes existantes (SceneColor, TAA, Bloom…)
- [ ] `LOG_INFO` "[EditorImGui] Dear ImGui X.Y.Z initialized" visible au boot avec `--editor`
- [ ] Rapport final : fichiers + commandes + résultats + DoD

---

## Interdit

- Ne pas lier ImGui à `engine_core` sans guard `#ifdef EDITOR` — ImGui ne doit pas entrer dans le build Release sans `--editor`
- Ne pas créer de passe de rendu ImGui séparée dans le FrameGraph
- Ne pas implémenter de sous-éditeur (audio, fx, loot…) dans ce ticket — c'est le rôle de M43.5–M43.8
