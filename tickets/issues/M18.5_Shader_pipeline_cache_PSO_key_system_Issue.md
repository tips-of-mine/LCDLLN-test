**Status:** Closed

## Objectif
Éviter stutter: compilation shaders offline/boot + pipeline cache Vulkan + keying.

## Dépendances
- M18.1

## Livrables
- Pipeline cache file
- PSO key hashing

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Interdire création pipeline en plein gameplay (assert debug).

## Étapes d'implémentation
1. Mettre en place cache Vulkan (vkCreatePipelineCache).
2. Définir PSO key (shaders + states + formats).
3. Pré-créer pipelines au boot (warmup).
4. Sauver cache sur disque.

## Critères de validation (DoD)
- [ ] Première entrée en zone dense sans stutter de compilation.

## Notes / pièges à éviter
- Sur drivers différents, cache peut être invalide -> handle gracefully.

### Implémentation réalisée

- **Pipeline cache Vulkan :** `PipelineCache::Init` charge un fichier optionnel (`paths.content` + `/cache/pipeline_cache.bin`), crée `VkPipelineCache` (ou cache vide si fichier invalide/absent). `Destroy` récupère les données, crée le répertoire parent si besoin, écrit le fichier, détruit le cache. Gestion propre des caches rejetés par le driver (création d’un cache vide + LOG_WARN).
- **PSO key :** `PsoKey` + `HashGraphicsPsoKey` (renderPass, subpass, layout, colorFormat0, depthFormat) et `HashComputePsoKey` (layout, spirvWordCount) dans `PsoKey.h` / `PsoKey.cpp`.
- **Warmup list :** `PipelineCache::BeginWarmup()` / `EndWarmup()`, `RegisterWarmupKey` ; tous les `vkCreate*Pipelines` sont appelés pendant le warmup dans `DeferredPipeline::Init` ; à la fin, log du nombre de PSO créés.
- **Assert debug :** `AssertPipelineCreationAllowed()` (actif en _DEBUG/DEBUG/!NDEBUG) appelé avant chaque `vkCreateGraphicsPipelines` / `vkCreateComputePipelines` ; en gameplay (hors warmup), une création de pipeline déclenche l’assert.
