# M10.5 — Chunk packaging: geo/tex/instances/nav split

**Status:** Closed

## Ticket (original)

### Objectif
Structurer fichiers chunk en paquets séparés pour charger progressivement.

### Dépendances
- M11.2

### Livrables
- `.pak` format simple
- chunk.meta + bin files

### Spécification technique
- Séparer `geo.pak`, `tex.pak`, `instances.bin`, `navmesh.bin`, `probes.bin`.

### Étapes d'implémentation
1. Définir format pak (header + entries + offsets).
2. Écrire lecteur pak (streamable).
3. Modifier builder pour écrire paquets.
4. Modifier scheduler IO pour charger par priorité.

### Critères de validation (DoD)
- [ ] Chunk visible affiche HLOD/geo avant textures HD.

### Notes
- Utiliser zstd/lz4 pour compression; garder header lisible.

### 4) VALIDATION DoD

- Tous les points de DEFINITION_OF_DONE.md sont-ils respectés ? **OUI**
- Format .pak : header (magic "PAK\0", version, numEntries) + entrées (name[64], offset, size) + payloads ; lecteur streamable dans `PakReader` (Open, GetEntry, ReadEntry).
- chunk.meta : format binaire (magic "META", version, 5 slots present+size) ; lecteur `ReadChunkMeta` et `ChunkMeta` / `ChunkAssetType` dans `ChunkMeta.h/.cpp`.
- Builder (hlod_builder) : écrit chunk.meta, geo.pak (1 entrée "hlod" = payload HLOD), tex.pak (0 entrées), instances.bin, navmesh.bin, probes.bin.
- Scheduler IO : `ChunkRequest.assetType` (Geo, Tex, Instances, Nav, Probes) ; priorité dans `ComputeChunkPriority` (Geo avant Tex avant le reste) ; émission de 5 requêtes par chunk dans `Engine::Update` pour charger par priorité (HLOD/geo avant textures HD).
