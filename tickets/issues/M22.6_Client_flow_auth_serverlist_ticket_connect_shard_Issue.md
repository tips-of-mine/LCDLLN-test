# M22.6 — Client flow: Auth → ServerList → Ticket → Connect Shard

**Status:** Closed

## Contenu du ticket (référence)

- **Objectif** : Assembler le "vertical slice" réseau distribué: client s'auth sur Master, récupère server list, demande ticket, connecte au shard avec ticket.
- **Livrables** : Flow complet (AUTH, SERVER_LIST, REQUEST_SHARD_TICKET, CONNECT SHARD), logs et erreurs claires, client ne conserve pas le ticket après usage, timeouts/retries raisonnables.
- **Spec** : Connexion shard refusée sans ticket.
