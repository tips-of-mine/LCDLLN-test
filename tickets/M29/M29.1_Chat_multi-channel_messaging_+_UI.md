
**Objectif**
Système chat multi-channel (say, yell, whisper, party, guild, zone) avec UI scrollable et filtres.

**Dépendances**
- M13.1 (server)
- M16.1 (UI framework)

**Livrables**
- `engine/net/ChatSystem.*`
- Channels: say, yell, whisper, party, guild, zone, global
- Chat UI panel (scrollable, input field)
- Message filtering/coloring

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- Say: rayon 25m
- Yell: rayon 200m
- Whisper: direct player
- Party/Guild: tous membres
- Zone: tous dans zone actuelle
- Global: broadcast serveur (admin ou event)

**Étapes d'implémentation**
1. Messages ChatMessage(channel, sender, text, timestamp)
2. Server: routing selon channel (distance check, membership)
3. Client: receive et display dans UI
4. UI: chat panel, scrollable history (500 lines ring buffer)
5. Input field: parse prefix (/s, /y, /w name, /p, /g)
6. Coloring: say=white, yell=red, whisper=pink, party=blue, guild=green

**Critères de validation (DoD)**
- [ ] Envoyer message say visible par joueurs proches
- [ ] Whisper arrive seulement au destinataire
- [ ] Party chat reçu par tous membres
- [ ] Scroll history fonctionne
- [ ] Couleurs par channel

**Notes / pièges à éviter**
- Rate limiting: max 5 msg/sec par joueur
- Profanity filter optionnel
- Timestamps display (HH:MM format)
