
**Objectif**
Implémenter state machine (idle/walk/run/jump) avec transitions et blend entre animations.

**Dépendances**
- M25.1
- M0.4 (input)

**Livrables**
- `engine/animation/AnimationStateMachine.*`
- Blend trees (linear blend, additive)
- Transitions avec durée/conditions

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- États: idle, walk, run, jump, attack
- Transitions: conditions (speed, grounded, input)
- Blend duration: 0.1-0.3s typique
- Linear blend: poseA * (1-t) + poseB * t

**Étapes d'implémentation**
1. Définir State (animation clip, loop, speed multiplier)
2. Définir Transition (from, to, duration, conditions)
3. Update state machine: évaluer conditions, trigger transitions
4. Pendant transition: blend poses source/dest
5. Exposer parameters (speed, isGrounded, etc.) pour conditions

**Critères de validation (DoD)**
- [ ] Transitions idle<->walk<->run fluides
- [ ] Jump interrompt correctement locomotion
- [ ] Pas de "pop" visuel

**Notes / pièges à éviter**
- Éviter cycles de transitions infinies
- Prévoir blend masks (upper/lower body) phase 2
