# M43.7 — Éditeur de tables de loot (`game/data/loot/loot_tables.txt`)

**Priorité :** Haute  
**Périmètre :** `engine/editor/EditorLootPanel.cpp/.h` · `engine/editor/EditorMode.cpp/.h`  
**Dépendances :** M43.4 (ImGui intégré) · M14.3 (loot bag / ServerApp parsing)

---

## Objectif

Ajouter dans le mode `--editor` un panneau ImGui permettant de **lire, modifier et sauvegarder** le fichier `game/data/loot/loot_tables.txt` sans édition manuelle de texte brut.

---

## Contexte

Le fichier actuel est un format texte tabulaire :
```
# source_archetype_id item_id quantity visibility
100 2001 1 owner
100 2002 3 owner
```

Le parser dans `ServerApp.cpp` lit 4 colonnes par ligne : `sourceArchetypeId`, `itemId`, `quantity`, `visibilityToken` (`owner` | `public`). Les lignes commençant par `#` sont des commentaires.

Ce format est fragile : une faute de frappe sur la colonne 4 provoque un `LOG_ERROR` et un refus de lancer le serveur. Un panneau ImGui avec Combo pour la visibilité élimine cette classe d'erreur.

## Changements requis

### Nouveaux fichiers

**`engine/editor/EditorLootPanel.h`**
```cpp
#pragma once
#include "engine/core/Config.h"
#include <cstdint>
#include <string>
#include <vector>

namespace engine::editor
{
    enum class LootVisibility : uint8_t { Owner = 0, Public = 1 };

    struct LootEntry
    {
        uint32_t       sourceArchetypeId = 0;
        uint32_t       itemId            = 0;
        uint32_t       quantity          = 1;
        LootVisibility visibility        = LootVisibility::Owner;
    };

    class EditorLootPanel final
    {
    public:
        bool Load(const engine::core::Config& config);
        bool Save(const engine::core::Config& config);
        void Draw();
        bool IsDirty() const { return m_dirty; }

    private:
        std::vector<LootEntry> m_entries;
        std::string m_loadedPath;
        bool m_dirty = false;

        void DrawToolbar();
        void DrawTable();
        static const char* VisibilityLabel(LootVisibility v);
    };
}
```

**`engine/editor/EditorLootPanel.cpp`**

`Load()` :
- Construire le chemin : `config.GetString("paths.content", "game/data") + "/loot/loot_tables.txt"`
- Parser le fichier ligne par ligne (même logique que `ServerApp`), sauter commentaires et lignes vides
- `LOG_INFO("[EditorLoot] Loaded {} entries from loot_tables.txt", m_entries.size())`

`Save()` :
- Écrire l'en-tête de commentaire + les entrées :
  ```
  # source_archetype_id item_id quantity visibility
  100 2001 1 owner
  100 2002 3 public
  ```
- `LOG_INFO("[EditorLoot] Saved loot_tables.txt ({} entries)", m_entries.size())`

`Draw()` :

**Toolbar :**
```
[+ Add Entry]   [Sort by Archetype]   Entries: 2
```

**Table ImGui (`ImGui::BeginTable`) :**
```
┌─────────────────┬──────────┬──────────┬────────────┬──────┐
│ Archetype ID    │ Item ID  │ Quantity │ Visibility │      │
├─────────────────┼──────────┼──────────┼────────────┼──────┤
│ [100          ] │ [2001  ] │ [1     ] │ [Owner ▼ ] │ [X]  │
│ [100          ] │ [2002  ] │ [3     ] │ [Owner ▼ ] │ [X]  │
└─────────────────┴──────────┴──────────┴────────────┴──────┘
```
- `ImGui::InputInt` pour sourceArchetypeId et itemId (min 0, max 999999)
- `ImGui::InputInt` pour quantity (min 1, max 9999)
- `ImGui::Combo("##vis", ...)` avec options `{ "owner", "public" }`
- Bouton `[X]` pour supprimer la ligne (sans popup de confirmation — loot est facile à ré-ajouter)
- Bouton `[+ Add Entry]` : ajoute une ligne avec valeurs par défaut (archetypeId=0, itemId=0, quantity=1, owner)
- Bouton `[Sort by Archetype]` : trie `m_entries` par `sourceArchetypeId` croissant puis `itemId` croissant

**Validation en bas de panneau :**
```
⚠ 1 entry has sourceArchetypeId=0 (likely unset)
```
Affiché en orange si au moins une entrée a `sourceArchetypeId == 0` ou `itemId == 0`.

Chaque modification positionne `m_dirty = true`.

### `engine/editor/EditorMode.h`

Ajouter :
```cpp
#include "engine/editor/EditorLootPanel.h"
engine::editor::EditorLootPanel m_lootPanel;
```

### `engine/editor/EditorMode.cpp`

- Dans `Init()` : `m_lootPanel.Load(config);`
- Rendu ImGui :
  ```cpp
  if (ImGui::Begin("Loot Editor"))
      m_lootPanel.Draw();
  ImGui::End();
  ```
- Propager `m_lootPanel.IsDirty()` dans `m_dirty`
- `Ctrl+S` → `m_lootPanel.Save(config)` quand la fenêtre Loot est active

## Format de sortie (invariant)

Le fichier sauvegardé doit être **compatible** avec le parser de `ServerApp.cpp` sans aucune modification côté serveur. Format exact :
```
# source_archetype_id item_id quantity visibility
<uint> <uint> <uint> <owner|public>
```
Une entrée par ligne, séparateurs espace, token visibilité en minuscules.
