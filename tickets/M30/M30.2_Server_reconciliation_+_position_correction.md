# M30.2_Server_reconciliation_+_position_correction

## Objectif
Corriger position client si divergence avec serveur (smooth correction).

## Dépendances
- M30.1

## Livrables
- Server snapshot avec tick authoritative
- Client reconciliation (replay inputs si mismatch)
- Smooth correction (interpolation)

## Spécification technique
- Snapshot: position, velocity, tick
- Client: compare predicted vs server position
- If delta > threshold (0.5m): reconcile
- Reconciliation: replay inputs depuis tick server
- Smooth: lerp vers corrected position (0.1-0.2s)

## Étapes d'implémentation
1. Receive server snapshot (tick, pos, vel)
2. Find corresponding client predicted state (same tick)
3. Compare positions: delta = |predicted - server|
4. If delta > 0.5m: rewind simulation to tick, replay inputs
5. Result = corrected state
6. Smooth transition: lerp current display -> corrected (100-200ms)
7. Purge input buffer <= tick
