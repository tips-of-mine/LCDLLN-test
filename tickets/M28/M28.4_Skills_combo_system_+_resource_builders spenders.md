# M28.4_Skills_combo_system_+_resource_builders spenders

## Objectif
Système combos (builder/spender) et ressources secondaires (combo points, holy power, etc.).

## Dépendances
- M28.1

## Livrables
- Resource types étendus (combo points, chi, etc.)
- Builder skills (génèrent ressource)
- Spender skills (consomment, gros effet)
- Combo UI display (M16)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Combo points: 0-5 typique, stack per target optionnel
- Builder: cost mana, generate 1-2 combo points
- Spender: cost combo points, no mana, high damage
- Decay: reset combo on target death ou timeout (30s)

## Étapes d'implémentation
1. Étendre PlayerStats: comboPoints, maxCombo
2. Skill effects: "generateCombo": 1 ou "consumeCombo": true
3. Builder skill: on hit, add combo points
4. Spender skill: validate combo >= threshold, consume all
5. Reset combo on target change (optional) ou death
6. UI: display pips/orbs (M16)

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD

## Notes / pièges à éviter
- Combo per-target ou global selon design
