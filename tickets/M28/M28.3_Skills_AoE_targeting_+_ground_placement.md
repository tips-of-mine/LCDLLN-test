
**Objectif**
Système targeting AoE (cercle, cone, ligne) avec preview au sol et validation serveur.

**Dépendances**
- M28.1
- M27.1 (camera raycast)

**Livrables**
- AoE shapes: circle, cone, line, ring
- Client preview (decal/particles)
- Server validation (position, targets hit)

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- Circle: center, radius
- Cone: origin, direction, angle, range
- Line: origin, direction, width, length
- Preview: projected decal au sol (texture + shader)

**Étapes d'implémentation**
1. Skill JSON: ajout "aoeShape", "aoeRadius", etc.
2. Client: raycast mouse->world pour position placement
3. Afficher preview decal/particles
4. On confirm: send SkillUseRequest(skillId, targetPos, targetDir)
5. Server: validate position reachable, apply effects à tous dans AoE
6. Hit detection: distance check ou spatial query

**Critères de validation (DoD)**
- [ ] Preview cercle AoE visible au sol
- [ ] Multiple ennemis dans rayon prennent damage
- [ ] Serveur refuse si position invalide (hors range)

**Notes / pièges à éviter**
- LOS check optionnel (pillars bloquent AoE)
- Limiter targets max (10-20) pour perf
