# M15.3 — Dynamic events: phases + rewards + notifications

## Objectif
Système événements dynamiques par zone avec phases (waves/boss) et rewards.

## Dépendances
- M15.2
- M16.1

## Livrables
- Event JSON
- EventState server
- EventState msg client

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Triggers time/random; phases séquentielles; cooldown.

## Étapes d’implémentation
1. Définir schema event.
2. Server: scheduler trigger + state machine phases.
3. Spawn waves via spawner-like API.
4. Client: notifier UI (texte + progression).

## Critères de validation (DoD)
- [ ] Un event se déclenche, progresse, finit et récompense.

## Notes / pièges à éviter
- Persistance event optionnelle (cooldown).
