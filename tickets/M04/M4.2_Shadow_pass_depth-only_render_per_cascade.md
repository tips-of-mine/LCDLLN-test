# M4.2 — Shadow pass: depth-only render per cascade

## Objectif
Rendre une shadow map depth par cascade (D32) en pass dédiée.

## Dépendances
- M4.1
- M3.1

## Livrables
- FG images ShadowMap[4] (depth + sampled)
- Pipeline depth-only (VS, no PS ou PS trivial)


## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Depth bias slope + constant.
- Culling front-face optionnel pour réduire acne.

## Étapes d’implémentation
1. Créer ShadowMap resources (res configurable).
2. Passes shadow i=0..3: set viewport, render depth.
3. Brancher mêmes meshes opaques (shadow casters).
4. Vérifier layouts depth attachment → sampled.

## Critères de validation (DoD)
- [ ] Shadow maps générées, visibles en debug capture.
- [ ] Validation OK.
