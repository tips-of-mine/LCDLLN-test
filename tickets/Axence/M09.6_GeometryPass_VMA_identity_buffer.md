# M09.6 — GeometryPass : migration VMA pour l'identity instance buffer

## Objectif
Supprimer le dernier `vkAllocateMemory` direct restant dans `GeometryPass::Init`
(buffer mat4 identité, M09.3) et le remplacer par VMA, en cohérence avec le reste
du moteur (FrameGraph, AssetRegistry, AutoExposure).

## Contexte
En M09.3, `m_identityInstanceBuffer` (64 octets, host-visible + coherent) a été créé
avec `vkGetPhysicalDeviceMemoryProperties` + boucle manuelle + `vkAllocateMemory`.
Tous les autres systèmes sont passés à VMA depuis M08. Ce vestige sollicite
inutilement le quota `maxMemoryAllocationCount` et rompt la cohérence d'allocation.

## Dépendances
- M09.3 (identity instance buffer existant)
- M08.1 (VMA intégré dans le moteur)

## Livrables
- `engine/render/GeometryPass.h`
  - Signature `Init` : remplacer `VkPhysicalDevice physicalDevice` par `void* vmaAllocator`.
  - Remplacer membre `VkDeviceMemory m_identityInstanceMemory` par `void* m_identityInstanceAlloc`.
- `engine/render/GeometryPass.cpp`
  - `Init` : créer le buffer via `vmaCreateBuffer` (VERTEX, CPU_TO_GPU, MAPPED).
  - Écrire la mat4 identité dans `VmaAllocationInfo.pMappedData`.
  - `Destroy` : libérer via `vmaDestroyBuffer`.
  - Supprimer `vkGetPhysicalDeviceMemoryProperties`, la boucle de sélection et `vkAllocateMemory`/`vkFreeMemory`.
- `engine/render/DeferredPipeline.cpp`
  - Passer `vmaAllocator` à `m_geometryPass.Init(...)`.

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
```cpp
VkBufferCreateInfo bufInfo{};
bufInfo.usage = VK_BUFFER_USAGE_VERTEX_BUFFER_BIT;
bufInfo.size  = 64; // sizeof(mat4)

VmaAllocationCreateInfo vmaInfo{};
vmaInfo.usage = VMA_MEMORY_USAGE_CPU_TO_GPU;
vmaInfo.flags = VMA_ALLOCATION_CREATE_MAPPED_BIT;

VmaAllocationInfo detail{};
vmaCreateBuffer((VmaAllocator)vmaAllocator, &bufInfo, &vmaInfo,
                &m_identityInstanceBuffer,
                (VmaAllocation*)&m_identityInstanceAlloc, &detail);
// Écrire mat4 identité dans detail.pMappedData (memcpy 64 octets)
```
- Conserver `void*` opaque dans le `.h` (pas d'include `vk_mem_alloc.h` dans le header).
- Ne pas modifier `RecordInstanced` (hors scope).

## Étapes d'implémentation
1. Modifier la signature `GeometryPass::Init` : `physicalDevice` → `void* vmaAllocator`.
2. Remplacer la création/écriture manuelle par VMA.
3. Remplacer la libération dans `Destroy` par `vmaDestroyBuffer`.
4. Mettre à jour l'appel dans `DeferredPipeline::Init`.
5. Vérifier qu'aucun autre site n'utilise l'ancien prototype.

- [ ] Aucun `vkAllocateMemory` / `vkFreeMemory` / `vkGetPhysicalDeviceMemoryProperties` dans `GeometryPass.cpp`.

- [ ] Rapport final : fichiers modifiés + commandes + résultats + DoD.

## Notes / pièges à éviter
- `VMA_MEMORY_USAGE_CPU_TO_GPU` est correct : buffer écrit une fois CPU, lu chaque frame GPU.
- `VMA_ALLOCATION_CREATE_MAPPED_BIT` rend `pMappedData` valide sans `vmaMapMemory` explicite.
- Conserver `void*` opaque pour ne pas propager `vk_mem_alloc.h` dans les headers publics.
