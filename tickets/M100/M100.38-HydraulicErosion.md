# M100.38 — Hydraulic Erosion (Particle-Based Simulation)

## Résumé

Outil d'érosion hydraulique réaliste qui simule **N gouttes d'eau** (particules) parcourant la heightmap depuis des points hauts vers les points bas. Chaque goutte transporte du sédiment et applique une déposition / érosion en fonction de sa vitesse, de la pente locale et de sa charge de sédiment. Après convergence, le terrain présente des vallées creusées, des deltas alluvionnaires, des reliefs adoucis — ce qui complète le rendu naturaliste obtenu par M100.35 (chaînes de montagnes générées) et M100.36 (réseau hydrographique).

L'outil **réutilise la heightmap consolidée et l'algorithme D8** de M100.36 (factoring partagé) pour distribuer intelligemment les points d'apparition des gouttes : pondération par flow accumulation, pour concentrer le calcul sur les zones où l'eau s'écoulerait naturellement.

`HydraulicErosionTool` est enregistré dans la toolbar M100.35. La simulation est **offline-feel** (modale de progression, 2-20 s selon le nombre de gouttes). L'application produit une seule `ICommand` réversible portant les `SparseChunkDeltas` cumulés.

**Aucun nouvel exécutable n'est créé.** Code dans `engine_core`, binaire éditeur unique `world_editor_app` (`lcdlln_world_editor.exe`).

## Contexte critique à respecter

### 1. Factoring depuis M100.36

Ce ticket **dépend du factoring** des structures de M100.36 dans `WatershedSimulation.h` :

- `ConsolidatedHeightGrid` (lecture-seule consolidée sur tous les chunks chargés) : promu dans `src/world_editor/terrain/ConsolidatedHeightGrid.h/.cpp`.
- `D8FlowDirection`, `FlowAccumulation`, `PathSimplifyDouglasPeucker` : déjà en files séparés sous `src/world_editor/water/` (cf. M100.36). M100.38 inclut ces headers tels quels.

Si M100.36 a livré sans extraire `ConsolidatedHeightGrid` (resté privé interne), ce ticket **doit le faire** dans son premier livrable : 1 fichier `.h/.cpp` créé, M100.36 mis à jour pour l'utiliser, tests M100.36 inchangés.

### 2. Cohabitation avec M100.36 (rivières) et M100.37 (océan)

- Si une `RiverNetworkCommand` (M100.36) a déjà été appliquée, l'érosion **modifie la heightmap** sous les splines de rivière existantes mais **ne touche pas** aux instances `RiverInstance` dans `WaterDocument`. Le mappeur peut donc avoir l'impression visuelle que la rivière « flotte » légèrement au-dessus du nouveau lit. Recommandation utilisateur affichée en banner de Tool Properties : « ré-exécutez la simulation watershed après l'érosion pour aligner les rivières ».
- Si un océan (M100.37) existe, les gouttes qui descendent en-dessous de `seaLevelMeters` **s'évaporent instantanément** (pas d'érosion sous-marine, déposent leur sédiment puis disparaissent).

### 3. Invariant terrain visible (rappel)

- Pas de bascule caméra.
- La preview est un overlay couleur 2D ImGui draw-list (rouge = érosion, vert = déposition), pas de render pass.
- Pas de touche à `m_noUserTextures` ni au frustum cull bypass.
- L'outil reste fonctionnel quand le terrain est en fallback orange.

### 4. Une seule application

Pas de nouvel exécutable hors la suite `hydraulic_erosion_tests`.

### 5. Arbo cmangos-style

Code sous `src/world_editor/terrain/erosion/` (nouveau sous-dossier dédié), `src/world_editor/ui/`, `src/world_editor/tests/`.

#### Partie C — Algorithme cœur (par goutte)

Pseudo-code synthétique :

```
Pour chaque goutte (parallélisable) :
  pos = position initiale (selon distribution)
  velocity = (0, 0)
  water = 1.0
  sediment = 0.0

  Pour step in 0..maxLifetime :
    h0, gradient = BilinearSampleHeightsAndGradient(pos)
    velocity = velocity * (1 - inertia) + gradient * (gravity * dt)
    pos_new = pos + normalize(velocity) * cellSize

    Si pos_new hors grid ou under sea level : break

    h1 = BilinearSampleHeights(pos_new)
    delta_h = h1 - h0

    capacity = max(-delta_h, minSlope) * |velocity| * water * sedimentCapacity

    Si sediment > capacity :
      // Déposition : on relâche du sédiment
      amount_deposit = (sediment - capacity) * depositionRate
      BilinearDistributeDelta(pos, +amount_deposit)
      sediment -= amount_deposit
    Sinon :
      // Érosion : on creuse
      amount_erode = min((capacity - sediment) * erosionRate, -delta_h * 0.5)
      BilinearDistributeDelta(pos, -amount_erode)
      sediment += amount_erode

    water *= (1 - evaporation)
    Si water < 1e-4 : break

    pos = pos_new
```

`BilinearDistributeDelta(pos, value)` distribue le `value` sur les 4 cellules voisines de `pos` en fonction des poids bilinéaires, et accumule le tout dans `SparseChunkDeltas` (thread-safe via tableau atomic float ou réduction par thread + merge final).

Borne de sécurité : à chaque écriture, `delta_cumulé[cell] = clamp(delta_cumulé[cell] + ajout, -maxDeltaPerCell, +maxDeltaPerCell)` pour éviter les runaways.

#### Partie D — Distributions d'initialisation

Trois modes de distribution des positions initiales des gouttes :

1. **Uniform** : tirage aléatoire uniforme dans `[0, kZoneSize]²`. Coût additionnel : nul.
2. **Weighted altitude** : tirage pondéré par `altitude(pos)` (les sommets émettent plus de gouttes). Coût : précalcul d'un cdf 1D en O(N), tirage en O(log N).
3. **Weighted flow accum** : tirage pondéré par `flowAccumulation(pos)` (les têtes de bassin versant émettent plus). Réutilise le `FlowAccumulation` de M100.36. Coût : pré-calcul flow accum en ~250 ms si pas déjà cached, tirage en O(log N).

Le `noiseSeed` (RNG) est paramétrable pour reproductibilité.

#### Partie E — Preview interactive

À la fin de la simulation, une preview overlay s'affiche sur le viewport :

- Cellules érodées (delta négatif) : surlignées en **rouge translucide**, intensité ∝ `|delta|`.
- Cellules déposées (delta positif) : surlignées en **vert translucide**, intensité ∝ `delta`.
- L'overlay est rendu via `ImGui::GetForegroundDrawList()` (additif 2D), pas de render pass.
- Toggle dans `ToolPropertiesPanel` : « Afficher preview érosion » (par défaut activé).

Re-tweaker un paramètre n'invalide pas le résultat ; il faut cliquer **Re-simulate** pour relancer.

#### Partie F — Multi-chunks & couture

- Pendant la simulation, chaque écriture passe par `BilinearDistributeDelta` qui écrit dans une map `SparseChunkDeltas` indexée par `GlobalChunkCoord` + `cellIndex`.
- Les cellules de frontière partagées entre 2 chunks reçoivent les deltas via leur position monde (cell_index identique des deux côtés vu de leur chunk respectif) → couture identique bit-à-bit, vérifiée par test.
- Au commit, `EnqueueLodRegen` est appelé pour chaque chunk touché.

#### Partie G — Parallélisation (optionnelle)

- Les gouttes sont **indépendantes** : on peut les paralléliser trivialement via `engine::core::jobs::JobSystem`.
- Le buffer `SparseChunkDeltas` est écrit en concurrent : implémentation par **réduction par thread** (chaque worker accumule dans son propre `SparseChunkDeltas` local, fusion finale au join). Évite les atomics.
- Seuil d'activation par défaut : si `dropCount >= 50000`, parallèle, sinon single-thread.
- Param de config (`config.json`) : `editor.world.hydraulic_erosion.parallel_threshold` (défaut 50 000).

#### Partie H — Commit unique

`Apply` pousse une seule `HydraulicErosionCommand` qui contient :
- les `SparseChunkDeltas` cumulés,
- un `HydraulicSimulationParams` snapshot (pour info debug, pas pour rejouer).

Undo restaure octet-à-octet.

#### Partie I — Tests

Suite `hydraulic_erosion_tests` couvrant : convergence, érosion sur pente, déposition en plat, stop sous sea level, déterminisme RNG, parallélisation correcte (= single-thread), undo complet, couture inter-chunks, distributions de seeding.

## Spécification fonctionnelle

### Workflow

1. Cliquer **Hydraulic** dans la toolbar.
2. Configurer paramètres dans `ToolPropertiesPanel` (généralement défauts OK).
3. Choisir la distribution d'initialisation (« Weighted flow accum » recommandée si M100.36 a déjà tourné).
4. Cliquer **Simulate** → modale de progression :
   - « Consolidation heightmap (1/5) »
   - « Distribution sources (2/5) »
   - « Simulation des gouttes (3/5) [barre %] »
   - « Reduction multi-thread (4/5) » (si parallèle)
   - « Préparation preview (5/5) »
5. Preview overlay rouge/vert apparaît sur le viewport, stats remplies dans le panneau.
6. **Re-simulate** avec d'autres paramètres si nécessaire → écrase le résultat précédent.
7. **Apply** → `HydraulicErosionCommand` poussée. Chunks touchés → `dirty` + LOD regen.
8. **Cancel** → abandon, preview supprimée.

```cpp
namespace engine::editor::world::erosion
{
    /// Distribue un delta sur les 4 cellules voisines de `pos` selon les
    /// poids bilinéaires. Écrit dans `deltas` (modifié in-place, thread-local).
    /// Effet de bord : modification de deltas, pas de heightmap directe.

    /// \param value  Delta à distribuer.
```

Garantie de couture : pour `pos` exactement sur la frontière partagée entre 2 chunks, la fonction écrit dans **les deux chunks** les mêmes deltas aux cellules de frontière correspondantes.

## Critères d'acceptation

### Simulation

- [ ] Une sim sur heightmap plate (gradient nul partout) produit `deltas` vides (aucune cellule modifiée).
