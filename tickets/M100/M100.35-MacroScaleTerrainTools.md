# M100.35 — Macro-Scale Terrain Tools (Mountain Ranges & Valley Chains)

## Résumé

Deux nouveaux outils éditeur pour authorer des structures de terrain à grande échelle qui traversent plusieurs chunks de façon cohérente :

- **MountainRangeTool** — l'utilisateur dessine une polyline 2D (chemin XZ projeté sur le sol) ; le système génère une crête continue le long de cette polyline avec paramètres par-vertex (largeur, hauteur, bruit, asymétrie), interpolés linéairement entre les vertices.
- **ValleyChainTool** — même mécanisme, profil inversé ; carve une vallée le long de la polyline.

Chaque brushstroke produit une seule `ICommand` réversible, potentiellement multi-chunks (jusqu'aux 400 chunks d'une zone). Le commit déclenche la régénération LOD asynchrone via le hook M100.8. Les coutures inter-chunks sont préservées par rasterisation en espace monde global (gather + scatter dans le même run, pattern hérité de M100.6).

Ce ticket **étend la phase 2 « Terrain »** de M100 ; il ne modifie ni ne renumérote les tickets `Done` (M100.5–8).

## Dépendances

- **M100.2** — Command Stack & Undo/Redo (`ICommand`, `CommandStack`).
- **M100.5** — Heightmap Data Structure (`TerrainChunk`, accès cellule).
- **M100.6** — Terrain Sculpting Brushes (raycast écran→sol réutilisé).
- **M100.7** — Terrain Stamps & Procedural Generators (`smoothstep` / `simplex2D` réutilisés depuis `ProceduralStampGenerators`).
- **M100.8** — Terrain LOD Regeneration (hook `OnCommit` pour enqueue worker LOD).

## Portée

### Inclus

- Outil `MountainRangeTool` : pose de polyline par clics successifs, mode `Open` ou `Loop`, raccourci **M**.
- Outil `ValleyChainTool` : identique, raccourci **V**, profil inversé.
- Paramètres par vertex : `widthMeters`, `heightMeters`, `noiseAmplitude`, `asymmetry` (–1..+1). Interpolation linéaire le long de chaque segment.
- Paramètres globaux par polyline : `PolylineMode` (Open/Loop), `FlankProfile` (Smoothstep/Linear/Exp), `noiseSeed`, `noiseFrequency`.
- Algorithme partagé `RasterizeMacroPolyline` qui calcule les deltas de hauteur sparse par chunk.
- Itération limitée au bounding box élargi de la polyline (`max(widthMeters) * 1.5`), pas de balayage de zone entière.
- Multi-chunk : une seule commande peut écrire sur N chunks ; deltas stockés sparse par cellule modifiée pour limiter la mémoire d'undo.
- Régénération LOD : commit déclenche `TerrainDocument::EnqueueLodRegen(coord)` pour chaque chunk touché.
- Preview overlay : pendant la pose, géométrie semi-transparente (ambre pour Mountain, bleu pour Valley) recalculée à chaque modification de paramètre (debounce 50 ms).
- Édition post-pose : drag d'un vertex repositionne et réévalue la preview.
- Cancel par **Esc**, retour à l'outil précédent.
- Tests unitaires couvrant rasterisation, parité couture, déterminisme bruit, undo multi-chunk.

### Hors scope

- Génération de réseaux fluviaux automatique → **M100.36** (futur).
- Erosion hydraulique → **M100.38** (futur).
- Erosion thermique / éolienne → **M100.39** (futur).
- Forking / ridge branches automatiques (l'utilisateur réutilise l'outil pour poser une seconde polyline).
- Génération procédurale « chaîne entière sans input utilisateur » (futur).
- Overhangs / falaises verticales (Phase 11 — M100.40+).
- Persistance des polylines pour re-édition après commit (le résultat est dans `terrain.bin` ; refaire = re-dessiner). Une future extension pourra sauvegarder dans `world_editor/maps/<zone_id>/macro_polylines.json`.
- Export des paramètres macros dans `map.lcdlln_edit.json` (inchangé).

## Spécification fonctionnelle

### Workflow MountainRangeTool

1. Raccourci **M** → outil actif. Si la caméra n'est pas en vue zénithale, basculement automatique en orbit top-down centrée sur la zone (réutilise `EditorCameraController::SetTopDown`).
2. Clic 1..N sur le sol → vertices posés, polyline visualisée en jaune entre les contrôles.
3. **Enter** ou double-clic → polyline figée, mode `Open`. **Ctrl+Enter** ou clic sur le premier vertex → mode `Loop` (ferme la polyline).
4. Tweak des paramètres globaux + par-vertex dans `ToolPropertiesPanel` (sélection vertex courante via menu déroulant ou click direct sur un control point).
5. Drag vertex → repositionne, preview se met à jour (debounce).
6. **Apply** → `MountainRangeCommand` pushée sur le `CommandStack`. Chunks touchés → `dirty`, hook LOD M100.8 enqueue.
7. **Esc** ou **Cancel** → abandon, preview supprimée, retour outil précédent.

### Workflow ValleyChainTool

Identique, raccourci **V**, preview bleue. Le profil radial est inversé : `h -= weight`.

### Algorithme cœur

Pour chaque cellule de heightmap dans le bounding box de la polyline élargi de `max(widthMeters) * 1.5` :

```
Pour chaque cellule (xWorld, zWorld) :
  1. Pour chaque segment de la polyline :
       distLat, t = ClosestPointOnSegment(xWorld, zWorld, segment.v0, segment.v1)
       garde le segment avec distLat min
  2. Interpolation linéaire des paramètres du vertex le long du segment retenu :
       width_local      = lerp(v0.width,      v1.width,      t)
       height_local     = lerp(v0.height,     v1.height,     t)
       noiseAmp_local   = lerp(v0.noiseAmp,   v1.noiseAmp,   t)
       asym_local       = lerp(v0.asymmetry,  v1.asymmetry,  t)
  3. Si distLat > width_local : skip (cellule hors zone d'effet).
  4. Profil radial :
       u = distLat / width_local             // 0..1
       w = profile(u) * height_local         // smoothstep(1, 0, u) par défaut
  5. Asymétrie :
       perpDir   = perpendiculaire au segment (XZ)
       segDir    = direction du segment      (XZ)
       side      = sign(cross(segDir, vec(cellPos - segCenter)))  // +1 ou -1
       w        *= (1 + asym_local * side)
  6. Bruit :
       offset    = SimplexNoise2D(xWorld * noiseFreq, zWorld * noiseFreq, seed) * noiseAmp_local
  7. Delta cellule :
       delta = w + offset                   // Mountain
       delta = -(w + offset)                // Valley
  8. Enregistrer (chunkCoord, cellIndex, delta) dans SparseChunkDeltas.
```

Le profil par défaut `Smoothstep` :
```
profile_smoothstep(u) = smoothstep(1.0, 0.0, clamp(u, 0, 1))
```

Variantes :
```
profile_linear(u)  = clamp(1.0 - u, 0, 1)
profile_exp(u)     = exp(-3.0 * u * u)     // gaussien
```

### Application au commit

```
MountainRangeCommand::Apply(TerrainDocument& doc) {
  for ((chunkCoord, cellDeltas) in m_sparseChunkDeltas) {
    TerrainChunk& chunk = doc.AcquireChunkMutable(chunkCoord);
    for ((cellIndex, delta) in cellDeltas) {
      chunk.heights[cellIndex] += delta;     // Mountain
    }
    doc.MarkDirty(chunkCoord);
  }
  doc.EnqueueLodRegenForDirtyChunks();
}

MountainRangeCommand::Undo(TerrainDocument& doc) {
  for ((chunkCoord, cellDeltas) in m_sparseChunkDeltas) {
    TerrainChunk& chunk = doc.AcquireChunkMutable(chunkCoord);
    for ((cellIndex, delta) in cellDeltas) {
      chunk.heights[cellIndex] -= delta;
    }
    doc.MarkDirty(chunkCoord);
  }
  doc.EnqueueLodRegenForDirtyChunks();
}
```

Symétrie parfaite : `Apply` puis `Undo` restaure octet-à-octet (testé).

### Structures (extrait `PolylineMacroCore.h`)

```cpp
#pragma once

#include "src/client/world/WorldModel.h"           // GlobalChunkCoord
#include "src/shared/math/Math.h"                  // Vec2, Vec3

#include <cstdint>
#include <unordered_map>
#include <vector>

namespace engine::editor::world
{
    enum class PolylineMode  : uint8_t { Open, Loop };
    enum class FlankProfile  : uint8_t { Smoothstep, Linear, Exp };

    /// Un vertex de polyline avec paramètres locaux. Les paramètres sont
    /// interpolés linéairement le long du segment vers le vertex suivant.
    struct PolylineVertex
    {
        float worldX           = 0.0f;
        float worldZ           = 0.0f;
        float widthMeters      = 250.0f;    // demi-largeur perpendiculaire (rayon)
        float heightMeters     = 400.0f;    // élévation max sur l'axe
        float noiseAmplitude   =  30.0f;    // variation simplex le long de la crête
        float asymmetry        =   0.0f;    // -1..+1
    };

    struct MacroPolylineParams
    {
        std::vector<PolylineVertex> vertices;
        PolylineMode mode            = PolylineMode::Open;
        FlankProfile profile         = FlankProfile::Smoothstep;
        uint32_t     noiseSeed       = 0;
        float        noiseFrequency  = 0.005f;   // cycles / mètre
    };

    /// (chunkCoord) → (cellIndex → deltaMeters).
    /// cellIndex = z * 257 + x (M100.5 layout).
    using SparseChunkDeltas =
        std::unordered_map<engine::world::GlobalChunkCoord,
                           std::unordered_map<uint32_t, float>>;

    /// Rasterise les deltas de hauteur pour la polyline donnée.
    /// \param params      Paramètres de la polyline.
    /// \param invert      false = Mountain (delta positif), true = Valley (delta négatif).
    /// \return            Deltas sparse par chunk. Vide si vertices < 2.
    ///
    /// Effet de bord : aucun. Pure function. Thread-safe.
    /// Itère uniquement sur les cellules dans le bounding box élargi.
    SparseChunkDeltas RasterizeMacroPolyline(const MacroPolylineParams& params,
                                             bool invert);
}
```

### Couture inter-chunk

La rasterisation calcule les deltas en **espace monde global** (`xWorld`, `zWorld` en mètres). Pour deux chunks `(0,0)` et `(1,0)` adjacents, la colonne `x=256` du chunk gauche et la colonne `x=0` du chunk droit représentent la même position monde (`xWorld = 500.0` exactement). L'algorithme leur attribuera donc le même `delta`. Le test `Test_ChunkBorderParity_TwoChunksSameValueOnFrontiers` vérifie ça à la résolution `kTerrainResolution = 257`.

### Performance attendue

Polyline de 5 km à largeur moyenne 250 m :
- Bounding box ≈ 5500 m × 750 m.
- Pas heightmap effectif : `kChunkSize / (kTerrainResolution - 1) = 500 / 256 ≈ 1.953 m/cellule`.
- Cellules à évaluer : ≈ 5500 / 1.953 × 750 / 1.953 ≈ 1 080 000 cellules.
- Coût par cellule : ~200 ns (distance segment + lerp + smoothstep + simplex).
- Total estimé : ~220 ms single-thread.

Mitigation si > 500 ms :
1. Parallélisation par segment via `engine::core::jobs::JobSystem` (chaque job traite un segment + sa bande latérale).
2. Le `JobSystem` existe déjà (M00.2). Pas de nouvelle dépendance.

### Format / parité

Aucun nouveau format binaire. Les deltas modifient `terrain.bin` (M100.5).
Aucun changement réseau, aucun changement client-runtime.

## Critères d'acceptation

- [ ] Une polyline de 3 vertices génère une crête continue entre les vertices avec largeur correcte mesurée perpendiculairement (tolérance ±2 % à 250 m).
- [ ] Mode `Loop` connecte le dernier vertex au premier sans discontinuité visible (test : hauteur sur le segment fermant identique à la formule).
- [ ] La rasterisation sur 2 chunks adjacents produit des valeurs **strictement identiques** sur les 257 cellules de frontière partagée (parité octet-à-octet).
- [ ] `asymmetry = +1.0` produit un flanc à hauteur quasi-pleine d'un côté et quasi-nul de l'autre, à `distLat = width/2` ; `asymmetry = 0` donne un profil symétrique.
- [ ] Le bruit simplex est **déterministe** : deux runs avec mêmes `noiseSeed`, `noiseFrequency`, mêmes vertices → mêmes deltas (parité octet-à-octet).
- [ ] Undo d'une `MountainRangeCommand` restaure toutes les heightmaps touchées octet-à-octet vs pré-commit. Idem `ValleyChainCommand`.
- [ ] Après commit, chaque chunk touché a son flag `dirty` set et le worker LOD M100.8 reçoit l'enqueue (vérifiable via test mock du worker).
- [ ] La preview ne modifie pas `TerrainChunk` tant qu'**Apply** n'a pas été cliqué (test : sampler les hauteurs pré-Apply == hauteurs avant pose).
- [ ] **Esc** supprime la preview sans pousser de commande sur la stack.
- [ ] Une polyline avec moins de 2 vertices est silencieusement ignorée (pas de commande poussée, pas de log error).
- [ ] Les tests `macro_terrain_tests` passent.
- [ ] Le projet configure et compile sans erreur sous preset `vs2022-x64` et `vs2022-x64-debug`.
- [ ] `lcdlln_world_editor.exe` se lance, l'onglet « Macro » apparaît dans le tab bar des outils, les raccourcis **M** et **V** activent les bons outils.
