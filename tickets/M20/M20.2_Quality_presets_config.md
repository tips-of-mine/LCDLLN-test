# M10.2 — Presets Qualité — Low/Medium/High/Ultra (config)

> **Type**: Ticket (Engineering)  
> **Priority**: High  
> **Owner**: Codex  
> **Status**: TODO  
> **Created**: 2026-02-08

## Objectif
Ajouter un système de presets qualité chargé via config/CLI afin de piloter un ensemble cohérent de paramètres.

Les presets doivent couvrir (au minimum):
- résolution interne ou scale (si présent),
- TAA (on/off + paramètres),
- SSAO (samples/radius/blur),
- ombres (cascades/PCF/size),
- bloom (threshold/strength),
- auto-exposure (on/off),
- LOD/rings world (si déjà utilisables).

Un preset doit être appliqué tôt au démarrage, avant l’initialisation render si possible.

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Code jeu** : uniquement sous `/game/...`
- **Contenu** (textures/meshes/json/audio/shaders) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- **Interdit** : créer des dossiers racine du type `/assets`, `/textures`, `/data`, `/shaders`, etc.
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json). Pas de chemins absolus.


## Livrables
- `engine/core/QualityPresets.*` (ou `game/QualityPresets.*`)
- Fichier `game/data/presets/quality.json` (ou équivalent)
- Support CLI: `--quality=Low|Medium|High|Ultra`
- Log au boot listant les valeurs appliquées

## Validation
- `--quality=Low` vs `--quality=Ultra` modifie visiblement la qualité/perf (ex: ombres, SSAO)
- Un preset inexistant => fallback propre + log warn
- Aucun chemin absolu pour charger le JSON presets

## Commandes (Windows)
- Configure: `cmake --preset win-ninja-vcpkg`
- Build: `cmake --build --preset win-build`
- Run: `./build/engine_app.exe <args>`

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD
