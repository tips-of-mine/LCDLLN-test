# M13.3 — Replication: spawn/despawn + state updates by interest

## Objectif
Envoyer spawn/despawn et updates uniquement pour entités dans interest set.

## Dépendances
- M13.2

## Livrables
- Messages Spawn/Despawn
- Snapshot entity states


## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- EntityId 64-bit; archetypeId; transform+state minimal.

## Étapes d’implémentation
1. Définir messages et sérialisation.
2. Sur entrée cell: envoyer spawns.
3. Sur sortie: envoyer despawns.
4. Snapshot: positions/rotations/vel des entités pertinentes.

## Critères de validation (DoD)
- [ ] Deux joueurs se voient bouger; entités hors range disparaissent.

## Notes / pièges à éviter
- Commencer sans delta compression; ajouter plus tard.
