# M22.3 — Heartbeat system + shard down detection

## Objectif
Mettre en place heartbeat périodique shards→master et détection shard down fiable.

## Dépendances
- M22.2
- M23 (metrics sera plus tard)

## Livrables
- Heartbeat interval/timeout (config)
- Master watchdog: timeout => mark offline
- Log + event interne “shard_down”

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Heartbeat payload: shard_id, current_load, timestamp.
- Timeout: ex 90s.
- Master is authoritative: state transition offline.

## Étapes d’implémentation
1. Timer heartbeat côté shard.
2. Handler master: update last_heartbeat + load.
3. Watchdog master: scan et mark down.
4. Tests intégration: kill shard => master détecte.

## Critères de validation (DoD)
- [ ] Master détecte shard down en ≤ timeout.
