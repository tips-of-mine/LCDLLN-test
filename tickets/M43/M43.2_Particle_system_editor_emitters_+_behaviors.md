# M43.2_Particle_system_editor_emitters_+_behaviors

## Objectif
Éditeur particules visuel pour configurer emitters, behaviors, renderers.

## Dépendances
- M17.2
- M12.1

## Livrables
- Editor UI: emitter properties (rate, lifetime, velocity)
- Modules: color over lifetime, size over lifetime, force fields
- Preview realtime (render particles in viewport)
- Save/load particle FX assets (.pfx JSON)

## Structure & chemins (verrouillé)
- Code moteur : uniquement sous `/engine/...`
- Contenu (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- Outils offline : uniquement sous `/tools/...`
- Tous les chemins de contenu sont relatifs à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Emitter: spawn rate, burst count, shape (box, sphere, cone)
- Modules: velocity, gravity, drag, color gradient, size curve
- Renderer: billboard, mesh, trail

## Étapes d'implémentation
1. UI panel: emitter properties
2. Sliders: rate (particles/sec), lifetime (sec)
3. Dropdowns: shape, blend mode
4. Modules list: add/remove modules
5. Curves editor: color/size over time (gradient/curve)
6. Preview: spawn emitter in viewport, render particles
7. Playback controls: play, pause, restart
8. Save: serialize to .pfx JSON
9. Load: parse JSON, instantiate emitter

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD

## Notes / pièges à éviter
- Performance: limit preview particles (1000 max)
- Hot reload: update assets sans restart editor
