# M24.2 — Gitea CI: build/test/artifacts (Linux)

## Objectif
Mettre en place CI Gitea Actions pour build + tests + artefacts téléchargeables des binaires serveur.

## Dépendances
- M19.10 (tests intégration)
- M23.4 (logs config)
- Toolchain CMake

## Livrables
- Workflow CI:
  - configure/build
  - unit tests
  - integration tests (Linux)
  - package artefacts (tar.gz)
- Artefacts nommés avec version semver + commit

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Caches (deps) si possible.
- Fail fast sur tests.

## Étapes d’implémentation
1. Ajouter workflow.
2. Ajouter steps tests.
3. Publier artefacts.

## Critères de validation (DoD)
- [ ] Chaque push produit un build validé.
- [ ] Artefact disponible.
