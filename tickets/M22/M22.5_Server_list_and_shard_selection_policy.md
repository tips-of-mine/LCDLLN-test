# M22.5 — Server list + shard selection policy (Master)

## Objectif
Fournir au client la liste des shards disponibles et sélectionner un shard cible selon policy (load, capacity).

## Dépendances
- M22.1
- M20 (auth session)

## Livrables
- Opcode `SERVER_LIST_REQUEST/RESPONSE`
- `SERVER_LIST_RESPONSE` contient:
  - shard_id
  - status
  - current_load
  - max_capacity
- Policy sélection:
  - lowest load ratio
  - exclude offline/degraded
- Hooks pour afficher nb persos (M20/M21 tables)

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Les clients consultent uniquement le Master.
- Le Master est source de vérité.

## Étapes d’implémentation
1. Définir payload server list.
2. Implémenter handler.
3. Implémenter policy sélection.
4. Tests integration: 2 shards -> choisir moins chargé.

## Critères de validation (DoD)
- [ ] Server list reflète registry.
- [ ] Policy choisit shard correct.
