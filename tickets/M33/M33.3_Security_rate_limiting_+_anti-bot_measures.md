
**Objectif**
Rate limiting requêtes, captcha registration, détection bots.

**Dépendances**
- M33.1

**Livrables**
- Rate limiter (IP-based, user-based)
- CAPTCHA integration (hCaptcha, reCAPTCHA)
- Bot detection heuristics (input patterns)

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- Rate limits:
  - Login: 5/5min per IP
  - Registration: 3/hour per IP
  - Chat: 10 msg/min per user
  - Skills: 20/sec per user
- CAPTCHA: on register, on too many login fails
- Bot detection: track input timing, impossible actions

**Étapes d'implémentation**
1. Implement rate limiter (token bucket ou sliding window)
2. Track attempts per IP/user in cache (Redis ou in-memory)
3. Return 429 Too Many Requests si exceeded
4. CAPTCHA: embed hCaptcha widget on register form
5. Verify captcha token server-side
6. Bot heuristics: flag accounts avec suspicious patterns
7. Auto-ban ou flag for review

**Critères de validation (DoD)**
- [ ] 6 logins rapides bloquent IP
- [ ] CAPTCHA apparaît après 3 fails
- [ ] Bot spam chat rate-limited

**Notes / pièges à éviter**
- Whitelist trusted IPs (devs, admins)
- Log rate limit events (analytics)
