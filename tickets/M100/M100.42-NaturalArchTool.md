# M100.42 — Natural Arch Tool (Phase 11 — Volumes 3D)

## Résumé

Outil spécialisé pour poser des **arches naturelles** : meshes glTF avec **deux ouvertures** (deux côtés passables) que le joueur peut traverser. Cas typique : pont naturel sur un canyon, arche taillée par l'érosion, formation rocheuse dans un désert (style Arches National Park).

Comme M100.41, ce ticket **réutilise intégralement** la fondation `MeshInsertSystem` de M100.40. Il ajoute trois spécificités :

- **Catalogue d'arches** (3 MVP : `arch_small_01`, `arch_medium_01`, `arch_canyon_01`).
- **Placement à deux points** : contrairement aux grottes (1 entrée) et overhangs (1 face ancrée), l'arche a deux pieds. L'outil demande deux clics terrain pour définir les piliers, et calcule l'orientation + scale pour que l'arche couvre la distance.
- **Détection de cohérence** : avertissement non-bloquant si la distance entre les deux piliers est très différente du `naturalSpanMeters` du catalog (ex : forcer une arche prévue pour 5 m à couvrir 20 m la rend visuellement étirée).

Catégorie `insertCategory = "arch"`.

**Aucun nouvel exécutable.** Code dans `engine_core`.

## Portée

### Inclus

#### Partie A — Bouton toolbar

- Nouveau bouton « Arch » dans `EditorToolbar`, icône `game/data/icons/editor/tools/arch.png`.
- `ActiveTool::Arch = 14`.

#### Partie B — Catalogue

`game/data/meshes/arches/catalog.json` avec 3 entrées MVP :

| id | description | `naturalSpanMeters` | usage typique |
|---|---|---|---|
| `arch_small_01` | Arche basse, accolée au sol | 4 m | Sentier de randonnée, ornement |
| `arch_medium_01` | Pont naturel, ~3 m de hauteur | 8 m | Traversée de canyon étroit |
| `arch_canyon_01` | Grande arche dramatique | 20 m | Pont sur canyon, vue spectaculaire |

Chaque entrée :
- `naturalSpanMeters` : distance entre les deux pieds de l'arche dans le mesh original (pour calcul du scale).
- `naturalHeightMeters` : hauteur du sommet de l'arche au-dessus du sol.
- `pillarA`, `pillarB` (Vec3 relatifs au pivot) : points de contact avec le sol pour chaque pied.
- `passageWidthMeters` : largeur du passage sous l'arche.

#### Partie D — Workflow

1. Cliquer **Arch** dans la toolbar.
2. Sélectionner un mesh dans le catalog.
3. **Premier clic** terrain : pose le pilier A. Marqueur jaune.
4. **Deuxième clic** terrain : pose le pilier B. Calcul automatique de la position, rotation et scale :
   - `worldPosition` = milieu (A + B) / 2.
   - `rotationY` = angle pour aligner l'axe `pillarA → pillarB` du mesh sur la ligne `A → B` réelle.
   - Si `Auto-scale to fit distance = true` : `uniformScale = distance(A, B) / naturalSpanMeters`.
   - Sinon : scale tel quel, l'arche peut chevaucher ou rentrer dans le sol.
5. Preview ghost affichée.
6. **Place** → `PlaceArchCommand` poussée.
7. **Reset points** ou **Cancel** : remet les piliers à zéro.

#### Partie E — Détection de cohérence

Si `Auto-scale to fit distance = true` :
- Calcule `ratio = distance(A, B) / naturalSpanMeters`.
- Si `ratio` hors de `[1 - tolérance, 1 + tolérance]` : affiche un **warning non-bloquant** dans `ToolPropertiesPanel` (« Étirement excessif : ratio 2.3, naturel = 1.0 ± 0.25 »).
- Le placement reste autorisé (le mappeur sait ce qu'il fait).

Si `Auto-scale = false` :
- Calcule la distance et compare au `naturalSpan * uniformScale`. Si écart > tolerance, warning.

#### Partie F — Camouflage piliers

Pour chaque pilier (A et B) :
- Patch splat circulaire « Rocher » de rayon `pillarSplatRadius` au point de contact terrain.
- Optionnellement, props rochers (réutilise `CaveCamouflage` de M100.40 avec catalogue propre).

#### Partie G — Commit

`PlaceArchCommand` regroupe :
- Nouvelle instance `MeshInsertInstance` (catégorie `"arch"`).
- 2 patch splat (un par pilier).
- 0..N props rochers par pilier (si activé).

Undo retire tout.

- Détection automatique de canyons traversables (à venir comme outil macro).

- [ ] Premier clic terrain : pilier A posé, marqueur jaune visible.

- [ ] `Auto-scale = false` : scale conservé, distance peut ne pas matcher.
- [ ] Warning ratio hors tolerance affiché dans le panel.
- [ ] Place → instance créée avec `insertCategory = "arch"`, 2 patch splat appliqués.
- [ ] Undo : instance retirée + splat restauré.
- [ ] Round-trip `mesh_inserts.bin` préserve `"arch"`.
- [ ] Reset points : pilier A et B remis à zéro sans placer.
