# M21.2 — Versioned SQL migrations + checksum tracking

**Status:** Closed

## Objectif
Mettre en place le système de migrations SQL versionnées avec checksum et ordre strict.

## Dépendances
- M21.1

## Livrables
- `db/migrations/` (ex: `0001_init.sql`, `0002_add_x.sql`)
- Convention nommage + ordre
- Calcul checksum (sha256) par fichier
- Règles:
  - migrations immuables (pas de modification après merge)
  - nouvelle migration pour tout changement

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Chaque migration s'applique dans une transaction si possible.
- Stocker dans `schema_version` le checksum du fichier appliqué.
- Détecter mismatch checksum => fail startup (sécurité).

## Étapes d'implémentation
1. Définir convention migrations.
2. Implémenter checksum calc (outil ou runtime).
3. Mettre 0001_init.sql = schema.sql ou équivalent.

## Critères de validation (DoD)
- [ ] Détection migrations pending fonctionne.
- [ ] Mismatch checksum déclenche arrêt contrôlé.
