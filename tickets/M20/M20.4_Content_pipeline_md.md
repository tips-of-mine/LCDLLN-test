> **Type**: Ticket (Engineering)  
> **Priority**: High  
> **Owner**: Codex  
> **Status**: TODO  
> **Created**: 2026-02-08

## Objectif
Écrire une documentation “pipeline contenu” (sans implémenter encore les tools) qui fixe:
- arborescence `/game/data` (textures, meshes, shaders, zones, presets),
- conventions de nommage,
- formats acceptés (ex: png, hdr, gltf),
- règles de packaging (futur .pak),
- versioning (hash, manifest) (brouillon).

C’est un document contractuel pour éviter la dérive de structure.

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Code jeu** : uniquement sous `/game/...`
- **Contenu** (textures/meshes/json/audio/shaders) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- **Interdit** : créer des dossiers racine du type `/assets`, `/textures`, `/data`, `/shaders`, etc.
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json). Pas de chemins absolus.


## Livrables
- `CONTENT_PIPELINE.md` à la racine
- Table des dossiers: `game/data/textures`, `game/data/meshes`, `game/data/zones`, `game/data/shaders`, `game/data/presets`
- Section “Do/Don’t” (ex: pas d’assets à la racine)

## Validation
- Document clair et exploitable pour un artiste/tech artist

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants
- [ ] Exécutable lancé (smoke test) et pas de crash immédiat
- [ ] Aucun dossier racine non autorisé créé
- [ ] Rapport final: fichiers modifiés + commandes + résultats + DoD
