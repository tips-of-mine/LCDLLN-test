**Status:** Closed

## Objectif
Fournir fichiers systemd pour déployer Master et Shard proprement (Linux).

## Dépendances

## Livrables
- `deploy/systemd/master.service`
- `deploy/systemd/shard@.service` (template multi-shard)
- Docs `docs/systemd.md`

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- User dédié (non-root)
- WorkingDirectory, Restart=on-failure
- EnvironmentFile optionnel

2. Documenter installation.

### 2) COMMANDES WINDOWS À EXÉCUTER

Ce ticket ne produit pas de binaires Windows. Les unités systemd sont utilisées sous Linux.

Sous Linux (Debian/Ubuntu) après déploiement des binaires et de la config :
- `sudo systemctl daemon-reload`
- `sudo systemctl start master` / `sudo systemctl start shard@1`
- `sudo systemctl status master` / `sudo systemctl status shard@1`
- `sudo systemctl stop master` / `sudo systemctl restart master` (idem pour shard@1).
