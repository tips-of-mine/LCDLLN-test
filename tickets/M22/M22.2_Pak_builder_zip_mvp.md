# M12.2 — Packaging — Packer MVP (zip) + loader runtime

> **Type**: Ticket (Engineering)  
> **Priority**: High  
> **Owner**: Codex  
> **Status**: TODO  
> **Created**: 2026-02-08

## Objectif
Créer un packer MVP basé sur zip (pour éviter un format propriétaire trop tôt):
- outil offline: pack `game/data` en `game.pak.zip` avec manifest inclus,
- runtime: charger des fichiers depuis le pak (read all bytes/text), avec fallback vers filesystem.

Cela prépare le patching/streaming sans tout réécrire.

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Code jeu** : uniquement sous `/game/...`
- **Contenu** (textures/meshes/json/audio/shaders) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- **Interdit** : créer des dossiers racine du type `/assets`, `/textures`, `/data`, `/shaders`, etc.
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json). Pas de chemins absolus.


## Livrables
- `tools/pak/pack_zip.py` (ou .cpp tool)
- `engine/platform/VirtualFileSystem.*` (VFS)
- VFS: Open/ReadAllBytes/Exists, support zip pak + filesystem fallback
- Config: `paths.pak` optionnel

## Validation
- Lancer avec pak présent lit un asset depuis pak (log)
- Sans pak, fallback filesystem OK

## Commandes (Windows)
- Configure: `cmake --preset win-ninja-vcpkg`
- Build: `cmake --build --preset win-build`
- Run: `./build/engine_app.exe <args>`

## Definition of Done (DoD)
- [ ] Build Windows OK (configure + build) via presets existants

- [ ] Aucun dossier racine non autorisé créé
