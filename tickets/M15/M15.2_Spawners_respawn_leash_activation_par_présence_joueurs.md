# M15.2 — Spawners: respawn, leash, activation par présence joueurs

## Objectif
Spawners data-driven activés seulement si joueurs dans cells proches; gèrent respawn et leash.

## Dépendances
- M13.2
- M14.2

## Livrables
- Spawner JSON
- Spawner system server
- Mob spawn/despawn


## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Activation par interest/presence; respawnSec; count.

## Étapes d’implémentation
1. Définir schema spawner.
2. Server: charger spawners par zone.
3. Activer si players dans rayon cells; spawn count.
4. Respawn timers; despawn si zone vide (option).

## Critères de validation (DoD)
- [ ] Quand un joueur arrive, mobs spawn; quand il part, charge serveur baisse.

## Notes / pièges à éviter
- Éviter respawn pendant combat; gérer occupancy.
