# M17.2 — Particles MVP: CPU emitters + billboard renderer

## Objectif
Système particules MVP: emitters CPU, billboards, atlas, sorting simple.

## Dépendances
- M03.4

## Livrables
- Particle component
- Particle renderer pass
- Asset format .pfx or json

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Update CPU; render transparent after opaques; blending standard.

## Étapes d’implémentation
1. Définir emitter params (rate, lifetime, velocity).
2. Simuler particules CPU (pool).
3. Rendu billboards (camera-facing).
4. Optional: sort approximatif par distance.

## Critères de validation (DoD)
- [ ] Explosion simple visible et stable à 60fps.
