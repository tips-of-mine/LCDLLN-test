# M19.11 — Network counters & minimal observability (pre-M23)

## Objectif
Ajouter des compteurs et logs réseau exploitables dès M19, avant l’observabilité complète (M23).

## Dépendances
- M19.3
- M19.4
- M19.5
- M00.1

## Livrables
- Counters:
  - connections active/total
  - handshake success/fail
  - bytes in/out
  - packets in/out
  - packets dropped
  - disconnect reasons histogram
- Logs contrôlés (throttle si nécessaire)
- Option CLI debug `--net.stats` (ou équivalent)

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Préférer aggregation per-thread puis merge périodique.
- Ne pas logguer par paquet en prod.

## Étapes d’implémentation
1. Struct stats global + per-thread.
2. Instrumenter connect/accept/handshake/read/write/disconnect.
3. Ajouter dump périodique en debug.
4. Ajouter commande/flag pour snapshot.

## Critères de validation (DoD)
- [ ] Stats cohérentes sur 1000 connexions.
- [ ] Logs restent lisibles (pas de spam).
