# M20.2 — Argon2 parameters and versioning

## Argon2 v1 parameters (engine::auth::Argon2Params)

| Parameter     | Meaning              | Default (example) | Notes                    |
|--------------|----------------------|-------------------|--------------------------|
| `time_cost`  | Number of iterations | 2                 | ARGON2_MIN_TIME..MAX     |
| `memory_kib` | Memory in KiB        | 65536 (64 MiB)    | ARGON2_MIN_MEMORY..MAX   |
| `parallelism`| Lanes/threads        | 1                 | ARGON2_MIN_LANES..MAX    |
| `hash_len`   | Output hash bytes    | 32                | ARGON2_MIN_OUTLEN..MAX   |

- **Variant**: Argon2id (recommended; hybrid, side-channel and GPU resistant).
- **Encoding**: Standard argon2id string (e.g. `$argon2id$v=19$m=...,t=...,p=...$salt$hash`) for storage and verify.

## Versioning for future upgrades

- Parameters are stored or versioned so that future upgrades (e.g. higher time_cost or memory_kib) can be applied without breaking existing stored hashes.
- When introducing new parameter sets, keep backward compatibility: verify using the params embedded in the stored encoded string; new hashes can use a new param set and still be verified later.
