# M11.3 — Navmesh bake per chunk + portals

**Status:** Closed

## Objectif
Baker navmesh par chunk avec portals vers voisins (Recast/Detour).

## Dépendances
- M11.2

## Livrables
- navmesh.bin
- portals.bin

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Navmesh stitch runtime via portals; bake offline.

## Étapes d'implémentation
1. Intégrer Recast dans tool.
2. Générer navmesh par chunk (walkable).
3. Extraire portals (edges frontières).
4. Écrire outputs.

## Critères de validation (DoD)
- [x] Agent peut pathfind à travers 2-3 chunks chargés (données navmesh + portals chargées par le moteur pour les chunks ; pathfinding utilisable sur ces données).

## Notes / pièges à éviter
- Attention à la cohérence hauteur terrain/props.
