# M14.3 — Loot bag + pickup + inventory updates

## Objectif
Système loot minimal : drop entité LootBag, pickup request, update inventaire.

## Dépendances
- M14.1

## Livrables
- LootTable
- LootBag entity
- PickupRequest/InventoryDelta msgs


## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Loot instancié (owner) ou public; range check pickup.

## Étapes d’implémentation
1. Définir loot tables data.
2. À la mort: générer loot et spawn LootBag.
3. PickupRequest: valider range/ownership.
4. Ajouter items à inventory, despawn bag, envoyer delta.

## Critères de validation (DoD)
- [ ] Kill->loot->pickup met à jour inventaire client.

## Notes / pièges à éviter
- Anti-dupe: tout côté serveur.
