# M19.2 — Protocol v1: C++20 serialization library (strict + tested)

**Status:** Closed

## Objectif
Créer une librairie C++20 d'encodage/décodage binaire conforme à `protocol_v1.md`, **strictement bornée** (pas de crash, pas d'OOB), et testée.

## Dépendances
- M19.1
- M00.1

## Livrables
- `ByteWriter` / `ByteReader` little-endian
- `PacketBuilder` (écriture header+payload) + `PacketView` (lecture/validation)
- Helpers: strings, arrays
- Validation stricte:
  - `size >= header_size`
  - `size <= 16KB`
  - bounds checks sur toutes les lectures
- Suite de tests unitaires encode/decode

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Écrire/relire champ par champ (éviter `reinterpret_cast` sur struct packée).
- Toute fonction `read_*` retourne un status (ok/fail) + ne lit jamais hors buffer.
- En decode:
  - si `size` annonce > bytes disponibles → attente (framing côté réseau) ou fail si paquet complet mais invalide
  - si `size` > 16KB → fail + policy (ERROR/disconnect)

## Étapes d'implémentation
1. Primitives LE: `write_u16/u32/u64`, `read_u16/u32/u64`.
2. Strings/arrays: `write_string`, `read_string`, `write_array`, `read_array`.
3. `PacketBuilder`: réserve header, écrit payload, patch `size`.
4. `PacketView`: parse header, expose payload slice + validate.
5. Tests:
   - roundtrip header
   - string/array roundtrip
   - oversize
   - buffer tronqué
   - opcode inconnu (à traiter plus tard via M19.8)

## Critères de validation (DoD)
- [ ] Aucun crash sur buffers aléatoires / tronqués.
- [ ] 16KB enforce dans builder et view.
- [ ] Tests passent en local + CI.

## Notes / pièges à éviter
- Ne pas coupler à TLS/epoll ici; rester pure serialization.

### Livrables réalisés

- **ByteWriter / ByteReader** : little-endian, champ par champ ; `WriteU16/U32/U64`, `ReadU16/U32/U64`, `WriteString`/`ReadString`, `WriteArrayCount`/`ReadArrayCount` ; contrôle des bornes sur toutes les lectures/écritures.

- **PacketView** : `Parse(data, bytesAvailable, out)` → `Ok` / `Incomplete` / `Invalid` ; validation size ≥ 18, size ≤ 16 KB ; exposition du payload en slice.
- **Tests** : roundtrip header, roundtrip string/array count, roundtrip PacketBuilder→PacketView, oversize (builder et view), buffer tronqué (Incomplete), size &lt; 18 (Invalid), buffers aléatoires (pas de crash).
