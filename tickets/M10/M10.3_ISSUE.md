# M10.3 — Cache: CPU LRU + GPU deferred free

**Status:** Closed

## Ticket (original)

### Objectif
Ajouter cache RAM (blobs décompressés) et destruction GPU différée (fences).

### Dépendances
- M10.1
- M00.2

### Livrables
- LRU cache
- DeferredDestroy queue

### Spécification technique
- Cache 1-4GB configurable; GPU destroy après fence signaled.

### Étapes d'implémentation
1. Implémenter cache LRU keyed par asset/chunk file.
2. Intégrer dans IO stage (hit cache -> skip disk).
3. Implémenter deferred free list (resource + fenceValue).
4. Collect lors de BeginFrame.

### Critères de validation (DoD)
- [ ] Ré-entrer dans une zone récemment quittée évite re-IO massif.

### Notes
- Mesurer: hit rate et memory usage.

### Détail implémentation

- **LruCache** : clé string, valeur blob (vector<std::byte>), liste LRU + map ; Get/Put/Clear, SetCapacityBytes, éviction quand taille > capacité. Config `streaming.cache_size_mb` (défaut 2048, borné 1024–4096).
- **DeferredDestroyQueue** : Push(device, fence, callback) ; Collect(device) parcourt les entrées, vkGetFenceStatus ; si signalé, exécute le callback et retire l’entrée.
- **Intégration IO** : StreamingScheduler::SetCache(LruCache*), GetCachedBlob(key) ; Engine initialise le cache et appelle SetCache ; l’étage IO peut appeler GetCachedBlob avant lecture disque pour éviter re-IO.
- **BeginFrame** : si device valide, m_deferredDestroyQueue.Collect(m_vkDevice.Device()).
