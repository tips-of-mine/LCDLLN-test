## Ticket content (M01.5b) — référence

# M01.5b — ShaderHotReload : file watcher event-driven (remplacement du poll 16 ms)

## Objectif
Remplacer le polling par sleep(16 ms) dans le worker thread du `ShaderHotReload` par
un mécanisme event-driven natif :
- **Windows** : `ReadDirectoryChangesW` (réaction immédiate sur modification fichier).
- **Linux / macOS** : fallback `inotify` / `kqueue` ou conservation du poll si non disponible.

## Contexte
Dans `ShaderHotReload.cpp`, le worker thread dort 16 ms entre chaque vérification de
timestamp (`std::this_thread::sleep_for(16ms)`). Sur un projet avec 30+ shaders et un
éditeur de matériaux ouvert, le délai de réaction est variable (0–16 ms) et ne peut
pas descendre sous 16 ms, même si le shader est prêt à l'instant T.

## Dépendances
- M01.5 (ShaderHotReload existant)

## Livrables
- `engine/render/ShaderHotReload.h`
  - Interface inchangée pour l'appelant (pas de breaking change).
  - Ajouter `class FileWatcher` (privé ou `engine/platform/FileWatcher.h`).
- `engine/platform/FileWatcher.h` + `engine/platform/FileWatcher.cpp`
  - `FileWatcher::Init(const std::string& directory)`.
  - `FileWatcher::WaitForChange(uint32_t timeoutMs) -> bool` (retourne `true` si changement détecté, `false` si timeout).
  - `FileWatcher::Destroy()`.
  - Implémentation Windows via `ReadDirectoryChangesW` + `OVERLAPPED`.
  - Implémentation fallback (Linux/Mac ou non-Windows) : `inotify_add_watch` ou poll existant.
- `engine/render/ShaderHotReload.cpp`
  - Worker thread : remplacer `sleep_for(16ms)` par `m_watcher.WaitForChange(500)`.
  - Sur `true` : parcourir les shaders modifiés (vérification mtime inchangée).
  - Sur `false` (timeout) : vérification périodique de sécurité (unchanged).
- `CMakeLists.txt` : ajouter `engine/platform/FileWatcher.cpp`.

## Spécification technique
### Windows
```cpp
// FileWatcher::Init
m_hDir = CreateFileW(dir, FILE_LIST_DIRECTORY,
    FILE_SHARE_READ|FILE_SHARE_WRITE|FILE_SHARE_DELETE,
    nullptr, OPEN_EXISTING,
    FILE_FLAG_BACKUP_SEMANTICS | FILE_FLAG_OVERLAPPED, nullptr);
// Lancer ReadDirectoryChangesW en OVERLAPPED

// FileWatcher::WaitForChange(timeoutMs)
// WaitForSingleObjectEx(m_hEvent, timeoutMs, FALSE)
// Si WAIT_OBJECT_0 : relancer ReadDirectoryChangesW, retourner true
// Si WAIT_TIMEOUT : retourner false
```
### Fallback (non-Windows)
- Conserver le poll `mtime` avec `sleep_for(16ms)` encapsulé dans `WaitForChange`.
- Permettre la compilation croisée sans `#ifdef` dans `ShaderHotReload.cpp`.

## Notes / pièges à éviter
- `ReadDirectoryChangesW` surveille un répertoire, pas un fichier : filtrer par extension `.glsl`/`.vert`/`.frag`/`.comp`.
- Toujours relancer `ReadDirectoryChangesW` après chaque notification (mode OVERLAPPED).
- Éviter les `#ifdef _WIN32` dans `ShaderHotReload.cpp` : isoler dans `FileWatcher.cpp`.
- Sur Windows, fermer le handle IOCP dans `Destroy` pour débloquer le thread worker.
