
**Objectif**
Système multi-currency (gold, silver, copper, gems, badges) avec wallet tracking.

**Dépendances**
- M14.4 (persistence)
- M16.1 (UI)

**Livrables**
- Currency types config (id, name, max, conversion)
- Wallet per player (DB)
- Transactions (add, subtract, transfer)
- UI wallet display

**Structure & chemins (verrouillé)**
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

**Spécification technique**
- Currencies: gold (main), honor, badges, premium_currency
- Max: gold=999,999,999 (uint32), honor=100,000
- Conversion: 100 copper = 1 silver, 100 silver = 1 gold (optional)
- Transactions atomic (DB transaction)

**Étapes d'implémentation**
1. Define currency types (JSON config)
2. DB: player_wallet (player_id, currency_id, amount)
3. API: AddCurrency(playerId, currencyId, amount)
4. Validate: check max cap, overdraft prevention
5. Log transactions (player_currency_log for audit)
6. UI: display wallet (gold icon + amount)
7. Events: CurrencyChanged -> update UI

**Critères de validation (DoD)**
- [ ] Ajouter gold fonctionne
- [ ] Dépenser gold décrémente
- [ ] UI affiche currencies
- [ ] Cap max respecté

**Notes / pièges à éviter**
- Signed overflow: use unsigned types
- Negative amounts check (server validation)
