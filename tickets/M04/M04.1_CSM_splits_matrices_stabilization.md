# M04.1 — CSM: splits + matrices + stabilization

## Objectif
Calculer cascades (4) avec practical split et matrices lightViewProj stables (snap texel).

## Dépendances
- M03.2

## Livrables
- CPU code compute cascades + split distances
- Uniform buffer `Cascades` (matrices + splits)

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- Practical split λ=0.7, 4 cascades.
- Stabilisation: snap ortho center à worldUnitsPerTexel.

## Étapes d’implémentation
1. Implémenter calcul splits near/far.
2. Calculer 8 coins frustum slice en world space.
3. Construire light view, AABB, ortho proj.
4. Appliquer stabilization snap.
5. Uploader matrices + splits.

## Critères de validation (DoD)
- [ ] Matrices valides, cascades couvrent frustum.
- [ ] Shimmering réduit (test caméra orbit).

## Notes / pièges à éviter
- Conserver lightDir normalisé; choisir distance backoff suffisante.
