# M03.2 — Deferred: lighting pass fullscreen (PBR GGX)

## Objectif
Ajouter pass lighting qui lit GBuffer + Depth et produit `SceneColor_HDR`.

## Dépendances
- M03.1

## Livrables
- FG image SceneColor_HDR (R16G16B16A16_SFLOAT)
- Pipeline fullscreen lighting (triangle/quad)

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.

## Spécification technique
- PBR metallic/roughness (UE4-like): GGX + Schlick + Smith.
- Reconstruction position via depth + inverse VP.

## Étapes d’implémentation
1. Créer SceneColor_HDR resource.
2. Shader lighting: read gbuffer textures + reconstruct pos.
3. Appliquer 1 directional light + ambient constant (IBL plus tard).
4. Output HDR.

## Critères de validation (DoD)
- [ ] Lumière réagit à normal/rough/metal.
- [ ] SceneColor_HDR visible via tonemap simple.

## Notes / pièges à éviter
- Mettre constants en uniform (dir light, color).
