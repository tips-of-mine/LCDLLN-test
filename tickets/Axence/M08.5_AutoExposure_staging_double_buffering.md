# M08.5 — AutoExposure : double-buffering du staging readback

## Objectif
Corriger un défaut de synchronisation dans `AutoExposure` : le staging buffer de
readback luminance est unique, ce qui crée un risque de race GPU/CPU lorsque le
moteur tourne avec 2 frames-in-flight. Passer à un staging + luminance buffer par
frame slot garantit la correctness sans surcoût visible.

## Contexte
Dans M08.3, `AutoExposure::Update()` lit `m_stagingBuffer` après `vkWaitForFences`.
Avec 2 frames-in-flight, la fence attendue appartient au slot courant, mais la passe
compute du slot précédent peut écrire dans le même staging au même moment :

```
Frame N   GPU : compute → m_stagingBuffer   ← écriture en cours
Frame N   CPU : Update  ← m_stagingBuffer   ← lecture simultanée (RACE)
```

## Dépendances
- M08.3 (AutoExposure existant)

## Livrables
- `engine/render/AutoExposure.h`
  - `static constexpr uint32_t kAESlots = 2;`
  - Scalaires `m_luminanceBuffer/Alloc` et `m_stagingBuffer/Alloc` → tableaux `[kAESlots]`.
  - `m_descriptorSet` scalaire → `m_descriptorSets[kAESlots]`.
  - `Record(VkDevice, VkCommandBuffer, Registry&, ResourceId, VkExtent2D, uint32_t frameIndex)`.
  - `Update(VkDevice, float dt, float key, float speed, uint32_t frameIndex)`.
- `engine/render/AutoExposure.cpp`
  - `Init` : boucle `for i in [0, kAESlots)` — créer buffers VMA + descriptor set par slot.
  - `Record(frameIndex)` : utilise slot `frameIndex % kAESlots`.
  - `Update(frameIndex)` : lit slot `(frameIndex + 1) % kAESlots` (résultats frame N-1).
  - `Destroy` : libère tous les slots.
- `engine/Engine.cpp` / `engine/render/DeferredPipeline.cpp`
  - Transmettre `m_currentFrame % 2` à `Record` et `Update`.

## Spécification technique
| Slot | Rôle frame N | Rôle frame N+1 |
|---|---|---|
| `N % 2` | Compute écrit ici | CPU lit ici (résultat N-1) |
| `(N+1) % 2` | CPU lit ici (résultat N-2) | Compute écrit ici |

- Le descriptor set doit pointer vers `m_luminanceBuffer[slot]` correspondant.
- Délai 1 frame sur l'exposition : standard AAA, imperceptible.

## Étapes d'implémentation
1. Ajouter `kAESlots` et transformer les membres scalaires en tableaux dans `.h`.
2. Mettre à jour `Init` : boucle de création (buffers + descriptor sets).
3. Mettre à jour `Record(frameIndex)` : sélection slot + mise à jour descriptor set.
4. Mettre à jour `Update(frameIndex)` : lecture slot décalé.
5. Mettre à jour `Destroy` : libération de tous les slots.
6. Adapter les call-sites dans `Engine.cpp` / `DeferredPipeline.cpp`.

## Notes / pièges à éviter
- Ne pas tenter de lire le slot courant avant la fin du compute de ce même slot.
- Le descriptor set compute lie `luminanceBuffer` : dupliquer un descriptor set par slot.
- Ne pas passer à `framesInFlight` dynamique : 2 slots hardcodés suffisent.
