# M21.5 — Backup strategy: daily dump + rotation + restore script

## Objectif
Automatiser backups MySQL (quotidiens) avec rotation et script de restauration documenté.

## Dépendances
- MySQL 8.0

## Livrables
- `scripts/backup_mysql.sh` (mysqldump --single-transaction + gzip)
- `scripts/restore_mysql.sh`
- Rotation (garder N jours)
- `docs/recovery_db.md`

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Backups: horodatés, compressés.
- Stockage configurable (path).
- Restore: vérifier schema_version après restore.

## Étapes d’implémentation
1. Script backup + options config.
2. Script restore.
3. Doc runbook.

## Critères de validation (DoD)
- [ ] Backup créé et rotaté.
- [ ] Restore fonctionne sur DB vide.
