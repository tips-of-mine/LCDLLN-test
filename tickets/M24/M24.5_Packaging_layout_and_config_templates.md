# M24.5 — Packaging layout + config templates

## Objectif
Définir la structure de package serveur et fournir templates de config (server_config.ini, DB config, TLS paths).

## Dépendances
- M24.2
- M23.4

## Livrables
- Arborescence package:
  - `bin/`
  - `config/`
  - `logs/`
  - `db/` (migrations)
- Templates:
  - `server_config.ini`
  - `tls.ini` (cert/key/fingerprint policy)
  - `db.ini`
- Docs `docs/deployment.md`

## Structure & chemins (verrouillé)
- **Code moteur** : uniquement sous `/engine/...`
- **Contenu** (textures/meshes/json/audio) : uniquement sous `/game/data/...`
- **Outils offline** : uniquement sous `/tools/...`
- Tous les chemins de contenu sont **relatifs** à `paths.content` (config.json).  
  Exemple : charger `textures/brick_albedo.png` depuis `${paths.content}/textures/brick_albedo.png`.
- ❌ Interdit : créer un dossier racine `/assets` ou utiliser des chemins absolus.


## Spécification technique
- Aucun chemin absolu dans templates.
- Config override via CLI si besoin.

## Étapes d’implémentation
1. Définir layout.
2. Ajouter templates.
3. CI produit tar.gz conforme.

## Critères de validation (DoD)
- [ ] Package exécutable sur machine cible.
